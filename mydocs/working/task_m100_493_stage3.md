# Task M100-493 Stage 3 — 표/셀 속성 대화상자 크기 안정화

- 이슈: #493
- 브랜치: `local/task_m100_493`

- 선행 커밋: `e7c84130 task 493: 셀 보호 입력 차단 UX 구현`

## 1. 작업 목표

한컴오피스의 `표/셀 속성` 대화상자처럼 탭을 이동해도 모달 외곽 크기가 변하지 않도록 한다.

사용자 확인 요구:

- `기본`, `여백/캡션`, `테두리`, `배경`, `표`, `셀` 탭 이동 시 대화상자 크기가 일정해야 한다.
- 탭 내용이 적은 경우에도 모달 높이가 줄어들지 않아야 한다.
- 탭 내용이 많은 경우에는 전체 모달이 커지는 대신 본문 영역 안에서 처리되어야 한다.

## 2. 설계 방향

- `TableCellPropsDialog` 전용 class를 모달 루트에 부여한다.
- 표/셀 속성 대화상자는 고정 폭/높이의 flex 레이아웃으로 만든다.
- 탭 패널 컨테이너는 고정된 본문 영역을 차지하게 하고, 각 탭 패널은 동일 높이를 사용한다.
- 내용이 긴 탭은 패널 내부 스크롤로 처리한다.

## 3. 검증 계획

- `npm run build` (`rhwp-studio`)
- Playwright로 `표 속성...` 대화상자를 열고 각 탭 클릭 전후 `.dialog-wrap` 크기가 동일한지 확인
- 보호 셀의 `셀 속성...` 대화상자도 탭 이동 시 크기가 일정한지 확인

## 4. 구현 결과

- `TableCellPropsDialog` 루트에 `tcp-dialog` 전용 class를 부여했다.
- 표/셀 속성 본문에 `tcp-dialog-body`, 탭 패널 영역에 `tcp-panel-container`를 추가했다.
- 표/셀 속성 다이얼로그만 고정 폭/높이 flex 레이아웃으로 동작하도록 CSS를 제한 적용했다.
- 탭 패널은 동일 높이를 유지하고, 긴 내용은 패널 내부 스크롤로 처리한다.

## 5. 검증 결과

- `npm run build` (`rhwp-studio`) 통과
- 작업지시자 시각 검증 완료:
  - `표 속성...` 대화상자에서 탭 이동 시 모달 외곽 크기 유지 확인
