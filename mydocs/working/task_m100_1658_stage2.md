## 2. 디버그로 규명한 실패 원인

### (a) 수렴 루프 진동 (RHWP_BOTTOM_DEBUG)
PC 케이스 pi6 예약 페이지가 pass 마다 **page1 ↔ page2 진동**, 미수렴:
- both-on-page1 → page1 예약 549.6 → 가용 440 → pi6 앵커가 다음 페이지로 밀림 → pi6 page2
- pi6 page2 → page1 예약 247 → 가용 743 → pi6 앵커 page1 복귀 → 다시 both-on-page1 …
- **하단 표의 예약이 자기 자신의 앵커를 밀어내는** 불안정 fixpoint.

### (b) 2-고정-pass 도 미해결 (close-call tip-over)
pass0 에서 페이지 고정(진동 차단) 후 pass1 적용해도 PC=2 유지:
- PC: 본문(pi0 표 222 + pi4 TAC 145 + 텍스트) ≈ 415px, 하단예약 549.6 → 가용 440.
