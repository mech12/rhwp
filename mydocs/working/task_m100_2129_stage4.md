### 시각판정 피드백 #1 반영 (작업지시자, 2026-07-10) — x 슬롯 중앙 배치

작업지시자 1차 시각판정에서 발견: 한컴은 라인 점을 **카테고리 슬롯 중앙**에 배치

고정해 가로로 꽉 차게 렌더 — 한컴 부정합. 수행계획서 Non-goals의 "x 배치 불변"은
작업지시자 지시로 **범위 보정**되어 본 단계에서 수정.

- 근거: 샘플 XML `c:crossBetween val="between"` (슬롯 중앙 배치 신호) + 정답지/편집기
  실측. 카테고리 라벨이 이미 쓰는 슬롯 중앙 공식과 일치시킴.
- 수정: `x = px + (pw/cat_count) × (i + 0.5)` (종전 `px + (pw/(n-1)) × i`).
  마커는 동일 좌표를 공유하므로 자동 반영. 비누적 기존 2종(꺽은선형·표식이있는꺽은선형)도
  함께 한컴 정합으로 개선됨.
- TDD: `test_line_points_at_category_slot_centers` (첫/끝 점 x == 카테고리 라벨 x) —
  RED 확인 후 수정, GREEN. 모듈 72/72 + 통합 6/6 유지.
