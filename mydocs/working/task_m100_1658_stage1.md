## 근본 난점 (왜 국소 수정이 실패하는가)

페이지 하단 고정 표의 올바른 배치는 **"어느 페이지의 하단에 들어가는지"** 를 알아야 하는데,
그 페이지 배정은 flow 결과에 의존한다(닭-달걀). 단일 pass 에서:
- 표를 만나기 전 콘텐츠는 reserve 미반영 상태로 이미 배치됨 → 하단 표가 본문 중간/다페이지일 때
  reserve 가 늦게 적용되어 오정렬.
- fit 검사(`current_height+reserve>available`)를 넣으면 PC(2개 하단표) 는 다시 분리되고
  (둘째 표 reserve 시 잔여 부족 오판), 단일표 문서는 reserve 가 flow 와 미세 어긋나 −1.
- 즉 **하단 예약은 본문 flow fit/페이지 배정과 강결합** → 정확하려면 2-pass(페이지 배정 확정 후
  하단 표를 해당 페이지 하단에 재배치)가 필요.

- 정확한 해법 = **2-pass 페이지 배정**(또는 #1658 의 저장 vpos 하이브리드). 영향·난도 큼(대형).
