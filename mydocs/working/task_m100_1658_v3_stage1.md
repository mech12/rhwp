# Task #1658 v3 1단계 완료 보고 — 케이스 고정·한글 실태 확인·모델 수정

## 결론 (요약)

한글 PDF 실측으로 **정답 모델을 수정**한다: 페이지 하단 고정 표는 "하단 스택"이
아니라 **하단 절대배치(겹침 허용) + 본문 하단 배타 영역(union = 최대 높이)** 이다.
그리고 rhwp 결함은 typeset 회계 1곳이 아니라 **2곳** — (a) typeset flow 소비,
(b) 렌더 경로의 Page·Bottom 절대배치 미적용.

## 대표 케이스 속성 (dump/XML 고정 — 36389312 관악 화재발생 알림)

- PAGE·BOTTOM 표 2개, 모두: `treat_as_char=false, wrap=자리차지(TopAndBottom),
  vert=쪽(Page) offset **0**, valign=Bottom, holdAnchorAndSO="1"`, 1×1 컨테이너
  ("틀" 필드, 내부 중첩 표), outer margin 0.
- pi5: 665.2×**247.3px** / pi6: 665.2×**357.2px** (합 604.5px).
- 동형: PC 셧다운 계열(36378545 등, PAGE·BOTTOM 1~2개), 디지털도시국 36398366(2개).

## 한글 실태 (COM PDF, 권위 근거)

- **한글 = 1쪽** (rhwp 2쪽).
- 두 표 높이 합(604.5px)이 본문 높이(1084.7px)와 본문 텍스트(saved-vpos 기준
  ≈577.4px)를 동시에 수용 불가 → **스택 모델 반증**.
- 성립 모델: 두 표 모두 vertOffset=0·Bottom → **본문 하단에 절대배치(서로 겹침)**.
  내용 사이가 성긴 틀이라 시각적으로 인터리브(PDF 하단 605~813pt 에 두 틀 내용 공존).
  본문 텍스트는 **배타 영역(=max(357.2px)) 위**까지만: 1084.7−357.2=727.5 ≥ 577.4 ✓.
- Stage 2b(75→55 회귀) 재해석: 일괄 out-of-flow(소비 0)는 **배타 영역 미적용**이라
  본문이 틀 영역까지 침범·과밀집(under 12→34). 정답은 소비 0 + **하단 배타 예약**.

## rhwp 현재 결함 2종 (동일 케이스 실측)

1. **typeset flow 소비**: page1 `used=852.0px vs hwp_used≈577.4px (diff +274.7px
   = pi5 247.3px + 마진)` — pi5 를 문서순 위치에서 소비, pi6(357.2px)가 page2 로 밀림.
2. **렌더 절대배치 미적용**: rhwp p2 에서 pi6 내용이 **상단(커서 y, 59~271pt)** 에
   렌더 — `compute_table_y_position`(table_layout.rs:2209~)의 Page·Bottom 절대배치
   (`ref_y+ref_h−h−v_off`)가 이 경로(1×1 "틀" 컨테이너)에서 적용되지 않음.
   정확한 우회 지점은 Stage 2 계측으로 특정.

## Stage 2 구현에의 함의 (구현계획서 보정)

1. 판별: `!treat_as_char ∧ wrap=TopAndBottom ∧ vert_rel_to=Page ∧ vert_align=Bottom`
   (+ Paper 제외, TAC 제외). `holdAnchorAndSO` 는 판별에 불사용(보수적).
2. typeset: 해당 표를 flow 미소비로 배치하되 **페이지 하단 배타 높이
   `bottom_fixed_exclusion = max(표높이+v_off)`** 를 `available_height()` 차감
   (각주 예약과 동일 패턴, 페이지 전환 시 리셋). 스택 산정 불필요(겹침 허용).
3. 렌더: PageItem 표식 또는 기존 절대배치 경로 활성화로 **하단 절대 y** 적용 —
   1×1 컨테이너 경로가 어디서 절대배치를 우회하는지 계측 후 최소 수정.
4. 페이지 이월: 잔여 본문 공간(현재 소비 높이)과 배타 높이가 겹치면(=본문이 이미
   배타 영역 아래까지 소비) 표를 다음 페이지 하단으로 이월 — 한글 동작 확인 필요
   시 케이스 발굴(범위 밖이면 문서화).
