# Task M100-493 Stage 2 — 셀 보호 입력/선택 UX

## 1. 작업 목표

한컴 도움말의 셀 보호 동작에 맞춰 보호 셀에는 텍스트 커서가 들어가지 않게 하고,
보호 셀도 표/셀 선택 경로를 통해 선택한 뒤 셀 속성에서 보호를 해제할 수 있게 한다.

사용자 확인 요구:

- 표 자체가 선택되어야 한다.
- 선택한 표의 개체 속성에 들어갈 수 있어야 한다.
- 셀 보호된 셀 위에 마우스를 올리면 진입할 수 없다는 표시가 보여야 한다.
- 보호된 셀을 클릭하면 커서를 넣지 않고 보호 셀을 선택해서 셀 서식으로 진입할 수 있어야 한다.
- 셀 보호 기능이 실제로 입력/커서 동작에 반영되어야 한다.

## 2. 설계 방향

- Stage 1에서 노출한 `CellProperties.cellProtect`를 프런트엔드 입력 처리의 기준으로 사용한다.
- 일반 셀 클릭은 기존처럼 셀 내부 커서 진입을 유지한다.
- 보호 셀 클릭은 셀 내부 커서 진입을 막고, 해당 셀을 셀 블록/선택 대상으로 전환한다.
- 보호 셀 hover는 기존 편집 커서 대신 진입 불가 시각 표시를 제공한다.
- 보호 셀도 선택 상태에서는 기존 표/셀 속성 다이얼로그를 열 수 있어야 한다.
- 표 선택과 표 개체 속성 진입 경로가 보호 셀 클릭과 충돌하지 않도록 유지한다.

## 3. 조사/구현 대상

- 입력/히트 테스트:
  - `rhwp-studio/src/engine/input-handler.ts`
  - 커서/셀 컨텍스트 관련 타입과 WASM bridge
- 셀 선택 표시:
  - `rhwp-studio/src/engine/cell-selection-renderer.ts`
  - 선택 상태 저장/명령 dispatch 경로
- 표/셀 속성 명령:
  - `rhwp-studio/src/command/commands/format.ts`
  - `rhwp-studio/src/ui/table-cell-props-dialog.ts`
- 필요 시 Rust API:
  - 보호 셀 판별을 위한 셀 bbox 또는 hit result 확장 여부 검토

## 4. 검증 계획

- 단위/통합:
  - 보호 셀 클릭 시 커서가 셀 내부로 이동하지 않는지 확인
  - 보호 셀 클릭 후 셀 선택 상태와 셀 속성 진입 가능 여부 확인
  - 보호 해제 후 일반 셀처럼 커서 진입 가능 여부 확인
- 로컬:
  - `npm run build` (`rhwp-studio`)
  - 관련 Rust 테스트 재실행
  - 필요 시 Browser/Playwright로 샘플 문서에서 hover/click 동작 확인

## 5. 구현 결과

- 보호 셀 hover:
  - `cellProtect` 셀 위에서는 편집 커서 대신 진입 불가 표시(`×`)와 `not-allowed` 커서를 표시한다.
  - 문서 변경/편집 후 보호 셀 판별 캐시와 hover 표시를 초기화한다.
- 보호 셀 클릭:
  - 보호 셀에는 텍스트 캐럿을 넣지 않고 셀 선택 상태로 전환한다.
  - 보호 셀 선택 상태에서 일반 문자 입력은 차단하고, 셀 선택 하이라이트를 유지한다.
  - 보호 셀 선택 상태에서도 우클릭 메뉴의 `셀 속성...`이 활성화되어 셀 보호를 해제할 수 있다.
- 표 객체 선택/속성:
  - 표 바깥 테두리 클릭은 리사이즈가 아니라 표 객체 선택으로 처리한다.
  - 표 객체 선택 상태의 우클릭 메뉴에서 `표 속성...`을 열 수 있도록 `table:cell-props` 실행부가 `selectedTableRef`를 사용한다.
  - 표 외곽 클릭 탐색은 페이지 레이아웃의 실제 table control index를 우선 사용한다.
