# Task M100-2020 Stage 3 — 복학원서 접수증 도장 원 위치 보정

## 목표

Stage 2 커밋 `8e12d33c6` 이후 남은 복학원서 1쪽 하단 접수증 도장 원 차이를 처리한다.

작업지시자가 지적한 증상:

- rhwp 출력은 도장 원이 `㊞` 글자 중심에 맞춰져 작고 위쪽에 놓인다.
- 한컴 출력은 도장 원이 더 크고 오른쪽/아래쪽으로 분리되어 있으며, `㊞` 글자는 원 중심이 아니라 왼쪽 안쪽에 놓인다.
- 따라서 Stage 2의 "`㊞`은 빨간 도장 원 중심과 맞아야 함" 회귀 가드는 잘못된 기대값이다.
- 도장 원 오른쪽 아래의 짧은 검은 선이 rhwp 출력에서 보이지 않는다.

## 기준 자료

- 원본: `samples/복학원서.hwp`
- 기준 PDF: `pdf/issue2020/복학원서-2022.pdf`
- Stage 2 산출물: `output/issue2020/stage2_bokhak_probe/`

## 조사 순서

1. 기준 PDF 1쪽 하단 접수증 영역을 래스터화해 빨간 도장 원 bbox를 측정한다.
2. rhwp render-tree/SVG에서 도장 원 shape bbox와 `㊞` 텍스트 bbox를 분리해 측정한다.
3. 원본 HWP의 해당 Shape common attr, anchor, margin, rel-to, width/height를 확인한다.
4. 문서명 기반 예외 없이 구조 가드로 위치/크기 산식을 보정한다.
5. Stage 2의 잘못된 중심 정렬 테스트를 한컴 기준 bbox/상대 위치 테스트로 교체한다.

## 완료 기준

- 복학원서 1쪽 접수증 빨간 도장 원 bbox가 한컴 PDF 기준과 시각적으로 맞는다.
- `㊞` 글자는 원 중심 정렬이 아니라 한컴 기준처럼 원 내부 왼쪽에 남는다.
- Stage 2에서 복원한 `-------------(인)------------` 날인선은 유지된다.
- 표 뒤 line 1에 남는 `U+F081C U+F081C` 한컴 PUA 선문자는 짧은 검은 선으로 보인다.
- line 0의 `U+F081C` prefix filler는 계속 폭 0/출력 숨김으로 유지한다.
- 기존 `issue_937` PUA 치환 및 U+F081C filler 미출력 정책을 깨지 않는다.
- `cargo fmt --check`, `git diff --check`, 관련 테스트, 복학원서 visual sweep을 통과한다.

## 추가 조사: 도장 오른쪽 아래 선

`samples/복학원서.hwpx`와 HWP 덤프를 대조한 결과, 누락 표식은 언더스코어 문자가 아니라
문단 16의 TAC 표 뒤에 남는 `U+F081C` 두 글자다.

- line 0: `U+F081C` 99개 prefix filler. 표를 body left에 놓기 위한 filler이므로 계속 폭 0/미출력.
- line 1: 표 컨트롤 뒤, 타원 컨트롤 앞의 `U+F081C U+F081C`. 한컴 화면에서는 도장 원 오른쪽 아래의 짧은 선으로 보인다.

정정은 문서명이나 날짜 행 문구가 아니라 접수증 TAC 표 구조와, 그 표 컨트롤 뒤/다음 컨트롤 앞의
1~2개 `U+F081C` marker 조건으로 제한한다.
