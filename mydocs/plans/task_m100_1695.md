# task m100 #1695: HWP3 LINE_SEG vpos reset/rewind 처리 검토

## 배경

- 이슈: https://github.com/edwardkim/rhwp/issues/1695
- 대상 샘플: `samples/SO-SUEOP.hwp`
- 기준 PDF: `pdf/SO-SUEOP-2024.pdf`
- HWP3 `LINE_SEG`의 `vpos` reset/rewind가 페이지/단 경계 힌트로 쓰이는지 확인하고, 최소 규칙으로 반영한다.

## 현재 기준

- `devel` 기준 SO-SUEOP HWP 출력 페이지 수는 46페이지다.
- `--respect-vpos-reset` 옵션을 적용해도 현재 Typeset 경로 출력은 기본 출력과 동일하다.
- 본문 `pi=179`, `pi=634`, `pi=748`에서 내부 `vpos` reset/rewind와 하단 overflow 로그가 함께 관찰된다.
- #1743에서 #1695 증상 일부는 완화되었지만, 일반 `LINE_SEG` reset/rewind 규칙은 아직 검증되지 않아 이슈가 open 상태다.

## 작업 방침

1. HWP3 전용 원인과 공통 renderer 적용 필요 범위를 분리한다.
2. 가능한 경우 HWP3 parser 경계(`src/parser/hwp3/`)에서 의미를 정규화한다.
3. 공통 Typeset 로직 수정이 필요하면 HWP3 전용 하드코딩 없이 `LINE_SEG` 기반 일반 규칙으로 제한한다.
4. SO-SUEOP의 페이지 수, overflow 로그, visual sweep 결과를 함께 확인한다.
5. 회귀 방지를 위해 #1695 전용 테스트를 추가한다.

## 완료 기준

- SO-SUEOP HWP 출력 페이지 수가 기준 46페이지를 유지한다.
- `pi=179`, `pi=634`, `pi=748` 관련 하단 overflow가 개선된다.
- 기준 PDF와의 visual sweep에서 페이지 단위 검토 결과가 PR 설명에 기록 가능하다.
- `cargo test --test issue_1695` 및 필요한 로컬 검증이 통과한다.
