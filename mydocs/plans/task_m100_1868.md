## 1. 배경 / 현황 조사

- rhwp CLI 는 export-svg/pdf/png/text/markdown 을 지원하나 **HWP→HWPX 변환 명령이 없다**.
- 기존 `convert` 명령은 **포맷 변환이 아니다** — 배포용(distribution) 문서 해제 +
  `export_hwp_with_adapter()` 로 **항상 .hwp 출력**. #1879 fidelity 하니스도 HWPX→HWP
  방향으로만 사용.
- 코어에는 `export_hwpx_native()` 가 이미 구현·검증돼 있다(#1532/#1533 HWPX 직접 저장,
  studio `exportHwpx` 사용 중, hwpx_roundtrip_baseline 게이트 보유). **CLI 노출만 하면 된다**
  — 이슈의 "비교적 간단할 것" 판단과 일치.

## 2. 목표

```
rhwp export-hwpx <입력.hwp|입력.hwpx> [출력.hwpx]
```

- HWP5/HWP3/HWPX 입력을 파싱해 HWPX 로 직렬화 저장 (파서는 포맷 자동 감지, 코어 동일 경로).
- 출력 생략 시 `<입력 stem>.hwpx` (입력과 같은 폴더).
- 출력 확장자가 `.hwpx` 아니면 경고(진행은 함 — export-svg 등 기존 관례 정합).
- help(`print_usage`)에 등재.

## 3. 설계 선택 — 새 명령 vs convert 확장

| 안 | 장점 | 단점 |
|---|---|---|
| **A. `export-hwpx` 신설 (권장)** | 이슈 요청 그대로, export-* 계열 명명 일관, convert(배포용 해제)와 의미 혼동 없음 | 명령 표면 +1 |
| B. `convert` 를 출력 확장자 분기로 확장 | 명령 표면 유지 | convert 의 현 의미(배포용 해제→HWP)와 충돌, #1879 하니스 등 기존 사용처 회귀 위험 |

A 안을 권장한다. #1638(convert --verify 게이트)은 별개 이슈로 존속.
