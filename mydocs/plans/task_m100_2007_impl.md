## ⚠️ 원인 가설 반증 (Stage 1 선행검증 결과)

**keep_with_next 가설은 직접 검증으로 반증됨.** 코딩 전 dump keep 플래그 확인 결과:

- 관세사 발산점 헤딩 pi36(제1절)/pi37/pi39/pi45: **`keep_with_next=false keep_lines=false widow_orphan=false page_break_before=false`** (attr1=0x00009680, bit16-19 전부 0).
- 즉 이 문서의 heading 은 **keep 플래그가 문서에 설정되어 있지 않다.** 한글은 keep 플래그 **없이도** 헤딩 경계에서 더 일찍 개행한다 → **한글의 기본(암묵) 동작**이지 문서 플래그가 아니다.

**파생 사실 (여전히 유효)**:
- `style_resolver.rs:863-865` 는 keep 플래그를 파싱하나 `typeset.rs` 는 `page_break_before` 만 소비(미사용은 사실). 단, **본 문서엔 keep 플래그가 없어** keep 적용 구현으로는 −9 가 해소되지 않는다(no-op).

**결론**: keep_with_next 구현은 본 케이스에 **무효**. 실제 −9 기전 = **한글 기본 heading/orphan 동작 또는 미세 분산 밀도(라인/문단 간격) 차이**. 근거 없는 keep 구현 착수 금지(반증 완료).

## 다음 RCA (재조정)

1. **라인 pitch 정밀 대조**: 발산 구간 dense body 페이지에서 rhwp render-tree 라인 pitch vs 오라클 PDF(fitz) pitch 대조 → 미세 라인높이 과소(#1842/#1898 축)인지, 순수 break-timing 인지 판별.
2. pitch 일치면 → 한글 기본 heading orphan/keep 동작 모사 필요(플래그 무관, 최고 회귀위험, 신중).
3. pitch 과소면 → 라인높이 계열(#1842/#1898) 국소 수정 가능성.

**라인 pitch 대조 결과(수행)**: rhwp 라인 gap 중앙 21.40px vs 오라클 25.87px(비 0.83). **그러나 혼란 신호로 판정** — page3가 정확 일치(diff+0.0)했으므로 전역 17% 라인높이 과소는 성립 불가. 이 비율은 rhwp/오라클 라인 모집단(표셀/헤딩/본문 혼합 비율) 차이의 아티팩트이며 clean 라인높이 버그 증거가 아니다.

**현 판정**: 3개 구체 가설 모두 기각/불확정 —
1. 표 행높이 과소 → **기각**(표0 852.5=852.4).
2. keep_with_next/keep_lines → **기각**(발산점 플래그 OFF).
3. 라인높이 과소 → **불확정**(page3 정확일치와 모순, 모집단 혼란).

→ 잔여 −9/−16 = 한글 기본 heading/orphan 동작 또는 미세 분산 밀도 차이로, **확증된 안전한 수정 지점 없음**. 저심각 롱테일. 무근거 hot-path 수정 금지 원칙 적용 — **본 세션 구현 보류**, 정밀 per-paragraph 오라클 대조(발산점 문단의 rhwp vs 오라클 배치 좌표 1:1)를 향후 진입점으로 남긴다.

## 최종 국소화 (per-page 오라클 대조 수행)

rhwp render-tree ↔ 오라클 PDF 페이지별 텍스트 정렬 결과:
- rhwp p1 표지 / **p2 빈 페이지**(오라클도 빈 페이지, 정합) / **p3~5 목차(목차·표목차·그림목차)**.
- 첫 발산 pi33-38 은 **목차(TOC) 구역 내부**. rhwp 가 한글보다 목차 항목을 페이지당 약간 더 담아 목차가 1쪽 적다.
- 목차 항목 문단: pi33/34 = **빈 문단(ps_id=22, line=110%)**, pi36 = 절 항목(before=1600, line=163%), pi37 = 소항목(before=200/after=200, line=150%).

**최종 판정**: −9 의 첫 축은 **목차 구역의 빈 문단 높이 / 항목 간격(spacing_before·line%) 렌더 밀도 차이**(rhwp 가 약간 조밀). 이는 **전역 문단 간격/라인높이/빈문단 경로**(#1836/#1880 빈-앵커 억제 계열 포함)라 목차만 국소 수정 불가 → 광역 회귀 위험. 안전한 좁은 수정 지점 없음.

**결론**: #2007/#2006-HIV 는 저심각(−9/126, −16/130) 분산 밀도 롱테일로, 현 근거로 **안전한 수정이 없다**. 정식 수정은 (a) 목차/빈문단/간격 렌더를 한글 오라클에 맞추는 광역 캘리브레이션 + (b) 코퍼스 pi-page 회귀게이트를 전제로 한 대형 독립 타스크로만 가능. 본 세션에서는 무근거 수정 대신 **완전한 RCA·오라클·재분류·진입점**을 확보하는 것으로 마무리.
