# 수행계획서 — Task #1646

## 1. 목표

루트 `README.md` 상단 배지 영역에 Chrome Web Store, Microsoft Edge Add-ons, Firefox Add-ons 배지를 추가한다.
기존 CI/Demo/npm/VS Code/License/Rust/WebAssembly 배지 줄은 그대로 유지하고, 브라우저 확장 스토어 배지는 바로
아래 별도 가운데 정렬 줄에 배치한다.

## 2. 범위

- 대상 파일: `README.md`
- 검토 파일: `README_EN.md`
- 제외: 확장 패키지 코드, manifest, 배포 설정, 버전 값 직접 변경

`README_EN.md`는 같은 상단 배지 구조를 갖고 있으므로 구현 시 구조 차이만 확인한다. 이번 이슈의 완료 기준은 루트
`README.md`이므로, 별도 지시가 없으면 `README_EN.md`에는 변경을 적용하지 않는다.

## 3. 배지 방침

- Chrome Web Store: Shields.io `chrome-web-store/v` 동적 버전 배지를 사용하고 `logo=googlechrome&logoColor=white`
  옵션을 붙인다.
- Microsoft Edge Add-ons: Shields.io의 안정적인 전용 동적 버전 배지가 없으므로 정적 `Edge Add-ons | Store`
  배지를 사용한다.
- Firefox Add-ons: Shields.io `amo/v` 동적 버전 배지를 사용하고 `logo=firefoxbrowser&logoColor=white`
  옵션을 붙인다.
- Edge의 비공식 `dynamic/json` 우회 배지는 README 안정성을 위해 사용하지 않는다.

## 4. 구현 계획

1. `README.md`의 기존 상단 배지 `<p align="center">` 문단은 내용과 순서를 유지한다.
2. 기존 배지 문단 바로 아래에 새 `<p align="center">` 문단을 추가한다.
3. 새 문단에 Chrome, Edge, Firefox 순서로 확장 스토어 배지를 배치한다.
4. 각 배지의 링크가 공개 스토어 URL로 이동하는지 문자열 기준으로 확인한다.
5. `README_EN.md`의 상단 구조를 확인해 향후 동일 적용 시 충돌 지점이 없는지 점검한다.

- `README.md` 상단에서 기존 배지 줄이 보존된다.
- 기존 배지 줄 아래에 Chrome Web Store, Edge Add-ons, Firefox Add-ons 배지 줄이 별도 문단으로 추가된다.
- Chrome/Firefox 배지는 로고가 포함된 버전 배지 URL을 사용한다.
- Edge 배지는 로고 없는 정적 Store 배지 URL을 사용한다.

  - Chrome: `https://chromewebstore.google.com/detail/pgakpjflombjmehnebnbpnalhegaanag`
