## 1. 배경

C1a·C1b로 OOXML 차트 25종(stock 2종 제외)의 렌더 커버리지(placeholder 0건)와 데이터·기하 정합은
확보됐으나, 한컴 2022 정답지(`pdf/chart/`) 대비 **스타일 4갭**이 전 종류 공통으로 남아 있다:

① 제목 미렌더 ② 팔레트 녹색-우선(한컴 파랑-우선) ③ 범례 하단(한컴 우측) ④ Y축 max가 데이터에 붙음(한컴 headroom).

④는 C1b 최종보고서 §5에서 작업지시자 결정(2026-06-29)으로 본 작업에 명시 이관된 항목이다
(공용 `nice_range` 정책 갭 — 막대/선/콤보/scatter 일괄 보정).

## 2. 사실 검증 (소스 수정 전 확인 완료)

핵심 발견: **샘플 XML(27종 전수)에 스타일 값이 사실상 없다(범례 제외)** → 본 작업은 XML 파싱이
아니라 **한컴 기본 스타일 재현**이다.

| 항목 | 확인 결과 |
|---|---|
| ① XML `c:title` | 요소는 있으나 텍스트(`c:tx`/`a:t`) 없음 + `autoTitleDeleted=0`. 파싱(`parser.rs:263,368-377`)·렌더(`renderer.rs:91,110-117`) 코드는 이미 존재하나 title=None이라 미발동 |
| ① 한컴 실측 | 자동 제목 placeholder **"차트 제목"** 상단 중앙, regular weight (PDF 라스터 확인) |
| ② XML 시리즈 색 | `srgbClr`/`schemeClr` 전무. `ho:hncChartStyle colorIndex="0"` 인덱스만 존재. theme1.xml 없음 → 항상 `DEFAULT_PALETTE`(`renderer.rs:10-12`, 녹색 0x70AD47 우선) 폴백 |
| ② 한컴 실측 팔레트 | **#6183D7(파랑) → #FE813B(주황) → #B0B0B0(회색) → #FCD801(노랑)** — 정답지 PDF 픽셀 히스토그램 실측 (막대 3시리즈 + 원형 4슬라이스). Office 표준(#4472C4…)이 아닌 한컴 고유 변형. 코퍼스 최대 4시리즈라 5번째 이후는 실측 불가 |
| ③ XML 범례 | 27종 전부 `<c:legend><c:legendPos val="r"/>` **명시** — 유일하게 XML로 해결 가능한 갭. 현재 파서 미파싱, 렌더러 하단 중앙 고정(`renderer.rs:92-96,120-131,153-160`, `render_legend:909` 가로 배치만) |
| ④ XML 축 | `c:max`/`c:min`/`c:majorUnit` 없음(auto). `nice_range`(`renderer.rs:234-254`)가 `new_max=ceil(max/step)*step`이라 데이터 max가 step 경계면 headroom 0. `scatter_range` 주석에 "C1c 후속" 명시 |
| ④ 한컴 실측 앵커 3점 | 막대 max 5.0→축 0~6 **라벨 0,2,4,6**(성긴 라벨) / scatter Y max 4.0→0~5(1 간격) / scatter X max 2.6→0~3(**0.5 간격 유지**) |
| 변경 봉인성 | 축/범례/팔레트 함수 전부 `renderer.rs` private, `OoxmlChart::parse`/`render_svg` 소비자는 `shape_layout.rs:1703,1724` 두 곳뿐 → 파급이 `src/ooxml_chart/` 안에 봉인 |
| 레거시 경로 | `src/ole_chart/`(svg_renderer.rs)는 완전 독립 코드 — 본 작업 범위 밖 (#1251 후속) |

## 3. 목표

렌더되는 전 차트 종류(막대/라인/원형/콤보/scatter, hwp+hwpx)에서 4갭을 한컴 정답지와 시각 정합
(합리적 수준 — 픽셀 parity 아님, #1251 정책)시킨다.

## 4. 범위 (설계 요지)

대상 파일: `src/ooxml_chart/{mod,parser,renderer}.rs` + 신규 통합 테스트 1파일.

- **② 팔레트**: `DEFAULT_PALETTE` → `[0xFF6183D7, 0xFFFE813B, 0xFFB0B0B0, 0xFFFCD801, …]`
  (앞 4개 실측, 이후 유추 — 주석에 구분 명기). `scheme_color`(parser.rs:438)는 **의도적 무변경**
  (schemeClr는 문서 테마 참조 의미라 Office 기본값이 타당, 코퍼스 미사용, 변경 시 기존 테스트가 근거 없이 깨짐).
- **④ 축**: `nice_range` → `nice_axis` — `(min, max, step)` 반환. 데이터 max가 step 경계에 걸리면
  +1 step 확장 후 step 재계산(ceil-nice), 아니면 step 유지(X축 0.5 간격 보존에 필수).
  실측 앵커 3점 동시 재현을 수치 검증 완료: 5.0→(0,6,2) / 4.0→(0,5,1) / 2.6→(0,3,0.5).
  `render_value_grid`는 5등분 고정 → step 기반 정수 루프. percentStacked는 (0,100,20) 명시 전달
  (현행 출력과 동일 = 무회귀). 격자 전용 함수 분리 금지 — 막대 기하가 같은 (vmin,vmax) 공유.
  알려진 편차: 콤보 보조축 max 10→축 0~15 (50% headroom, 실측 앵커 없음).
- **① 자동 제목**: 모델에 `has_title_elem`/`auto_title_deleted` 플래그 추가(파서 arm 2개).
  `chart.title`은 명시 텍스트 전용 유지 → `parser.rs:85` 조기 반환 가드 무변경(시리즈 없는 XML이
  fallback으로 새는 회귀 원천 차단). 렌더러 `effective_title = title.or("차트 제목" if 플래그)`,
  weight 600→400.
- **③ 범례**: `enum LegendPos`(default=**Bottom** — 모델 직접 구성하는 기존 단위 테스트 보호) +
  `legendPos` 파서 arm. Right → 우측 세로 스택(`legend_w = clamp(최장라벨×10+26, 50, w×0.30)`,
  plot_w 축소, 수직 중앙, pie 경로 동일 분기), 그 외 → 현행 하단 가로 폴백.
  `<g class="hwp-chart-legend">` 래핑(통합 테스트 위치 검증용).

**회귀 예측(실코드 대조)**: 확실히 깨지는 기존 테스트는 `test_render_scatter_decimal_axis_labels`
1개(`">2.4<"`→`">2.5<"`, 의도된 스펙 변경)뿐. percent 라벨·기하 distinct·콤보 범례 텍스트·통합
placeholder 가드는 설계 선택(가드 무변경, LegendPos default, scheme_color 무변경)으로 보호됨.
