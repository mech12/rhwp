# Task #667 계획 — ingest 스키마 진화

## 배경

Issue #667은 `build-from-ingest` 입력 스키마가 시험지의 공유 지문, `<보기>` 박스,
반복 머리말/꼬리말 정보를 구조적으로 표현하지 못하는 문제를 다룬다.

선행 이슈 #665, #666은 완료되었고, 현재 `build-from-ingest`는 텍스트 중심 시험지 HWPX 생성과
roundtrip smoke 검증이 가능한 상태다.

## 범위

- ingest schema v1을 backward-compatible하게 확장한다.
- Rust serde 모델에 다음 필드를 추가한다.
  - top-level `passages`
  - `Question.passage_ref`
  - `StemBlock::Boxed`
  - top-level `header_text`, `footer_text`, `form_label`
- `exam_paper` builder가 새 구조를 문서 IR로 매핑한다.
- `.claude/skills/rhwp-exam-ingest/SKILL.md`, `cli_commands.md`, JSON schema/sample을 갱신한다.
- 기존 v1 JSON은 계속 파싱되고 같은 출력 의미를 유지해야 한다.

## 범위 밖

- Vision/OCR 자동 구조화 품질 개선
- 실제 이미지/Picture 직렬화 완성
- 복잡한 Table/Frame 기반 `<보기>` 레이아웃 완전 복원
- 한컴 기준 PDF와의 원본 시험지 시각 정합 보장

## 구현 방향

1. 공유 지문은 `passages[].id`와 `Question.passage_ref`로 연결한다.
   builder는 같은 passage가 처음 참조되는 문제 앞에 한 번만 출력한다.
2. `boxed` stem block은 border/fill이 있는 문단 모양으로 최소 구현한다.
   다중 paragraph/table/frame 수준의 정밀 박스는 후속으로 분리한다.
3. `header_text`/`footer_text`는 실제 Header/Footer control로 매핑하고,
   `form_label`은 머리말 텍스트에 함께 표시한다.
4. 기존 `version: "1"` 입력을 유지하며, 새 필드는 모두 optional/default 처리한다.

## 검증

- parser unit test: 기존 minimal JSON 호환 + 새 필드 파싱/roundtrip
- builder unit test: shared passage 중복 방지, boxed para shape, header/footer control 생성
- CLI smoke: `build-from-ingest` sample 생성 후 `export-text`, `dump`, `export-svg`
- 최소 focused test: `cargo test parser::ingest document_core::builders::exam_paper`
