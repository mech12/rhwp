# 수행계획서 — Task M100 #2230: placeholder 선택 + 그림 삽입 편집 기능

- 이슈: #2230 (#2225 후속, 작업지시자 지시) / 브랜치: `local/task2230` / 작성일: 2026-07-12
- 목표: 한컴 편집기처럼 그림 미지정 placeholder(#2225 MissingPicture)를
  **선택**하고 **그림을 삽입(지정)** 할 수 있게 한다. 검증 문서: 36389312
  "심볼" 필드.

## 재사용 인프라 (정찰 완료)

- `PlaceholderNode.control_ref: Option<ObjectControlRef>` — OLE 선택이 이미
  이 경로로 동작 (cursor_rect hit-test :1195, kind=="ole" 필터).
- `insert_picture_native` — BinData 추가·storage id 규칙 포함 완전 삽입
  커맨드 존재. #2230은 그 축소 변형("기존 Picture 컨트롤에 이미지 지정")이
  필요.
- studio 입력 메뉴의 그림 삽입 파일 선택 흐름 (재사용 조사 — 구현계획서에서
  확정).

## 단계 구성 (구현계획서에서 상세화)

1. **선택 가능화 (Rust)**: layout이 MissingPicture placeholder에
   control_ref(kind="picture", 섹션/문단/컨트롤 + cell_path)를 채우고,
   hit-test 의 kind 필터를 확장 — studio 클릭 선택 성립.
2. **지정 커맨드 (Rust)**: `assign_picture_image_native(...)` — BinData
   append + `picture.image_attr.bin_data_id` 갱신 + 재조판/캐시 무효화 +
   undo 스택 정합. wasm API 노출.
3. **studio UI (TS)**: placeholder 선택 상태에서 그림 지정 진입(더블클릭)
   → 파일 선택 → 커맨드 호출 → 재렌더. 선택 표시(기존 개체 선택 UI 재사용).
4. **저장 왕복 + 게이트**: 지정 후 HWPX/HWP 저장 → 재열기에서 그림 유지
   (BinData 직렬화·매니페스트). 표적 테스트(코어 커맨드 왕복) + 전수 +
   studio npm + 판정(작업지시자 실사용: 선택→삽입→저장→재열기).

## 절차

기능 규모(3층 관통)이므로 본 수행계획서 승인 후 **구현계획서
(task_m100_2230_impl.md) 작성 → 승인** 게이트를 거쳐 구현에 착수한다.

- HWPX BinData 매니페스트: insert_picture_native의 기존 규칙 재사용으로
  왕복 리스크 최소화.
- 편집 뷰 전용 기능 — export 억제(#2225)와의 상호작용 없음(지정 후에는
