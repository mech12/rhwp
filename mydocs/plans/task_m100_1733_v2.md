# Task #1733 v2 계획 — 국제고속선기준 잔여 over-pagination 재추적

## 배경

#1736에서 `samples/task1725/text_footnote_tail_overpagination.hwpx`는 250쪽에서 245쪽으로 완화됐고,
기준 PDF(`pdf/text_footnote_tail_overpagination-2024.pdf`)는 242쪽이다.

2026-07-03 현재 `upstream/devel` 기준으로 다시 측정하면 다음과 같다.

| 파일 | 현재 rhwp | 기준 |
|---|---:|---:|
| `samples/task1725/text_footnote_tail_overpagination.hwpx` | 250쪽 | PDF 242쪽 |
| `samples/task1725/text_footnote_tail_overpagination.hwp` | 249쪽 | PDF 242쪽 |

즉 기존 #1733 부분 완화 코드는 남아 있지만, 후속 레이아웃 변경 이후 페이지 수가 다시 증가한 상태다.

## 목표

1. #1736 당시 245/244쪽 완화가 현재 devel에서 왜 사라졌는지 확인한다.
2. PDF 기준 242쪽에 가까워지도록 잔여 near-empty/over-pagination 원인을 좁힌다.
3. 완화 조건은 국제고속선기준 계열 tail/vpos-reset 패턴에 한정하고, 다른 문서의 하단 겹침 회귀를 피한다.

## 초기 검증 명령

```bash
cargo build --bin rhwp
./target/debug/rhwp dump-pages samples/task1725/text_footnote_tail_overpagination.hwpx | rg -c '^=== 페이지'
./target/debug/rhwp dump-pages samples/task1725/text_footnote_tail_overpagination.hwp | rg -c '^=== 페이지'
pdfinfo pdf/text_footnote_tail_overpagination-2024.pdf | rg '^Pages:'
```

## 분석 순서

1. 현재 devel의 near-empty 페이지와 마지막 배치 item을 덤프한다.
2. #1736 완화 커밋 이후 `src/renderer/typeset.rs` 변경이 tail-before-vpos-reset 조건에 준 영향을 비교한다.
3. 남은 후보를 PartialParagraph, PartialTable, 비각주 tail, vpos-reset 직전/직후 컷으로 분류한다.
4. 최소 조건 가드와 회귀 테스트를 추가한다.

## 검증 기준

- 대표 샘플 HWPX/HWP page count가 개선되어야 한다.
- 기존 tail 계열 회귀 샘플은 page count 무회귀여야 한다.
- `cargo test`는 focused test부터 수행하고, PR 준비 시 전체 검증은 별도 승인 후 진행한다.
