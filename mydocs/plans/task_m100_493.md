# Task M100-493 수행계획서 — 셀 보호/필드 이름/양식 모드 셀 속성 보존

- 이슈: <https://github.com/edwardkim/rhwp/issues/493>
- 작업 브랜치: `local/task_m100_493`
- 작성일: 2026-06-19
- 모드: 기여자 모드

## 1. 배경

#493은 `loadFile`로 기존 서식 파일을 열 때 표 셀의 셀 보호와 셀 속성의 필드 이름이 누락되어,
기안기에서 셀 필드 이름을 키로 값을 주입하기 어렵다는 요청이다.

PR #1401에서 HWPX `hp:tc/@name` 파싱 누락은 수정되었다. 다만 PR 본문과 검토 문서 모두
셀 보호(`protect`)는 잔여 범위로 남겼고, 이번 샘플 확인 결과 `editable`도 같은 셀 속성 묶음으로
다뤄야 한다.

## 2. 권위 자료와 샘플

- 한컴 도움말: `표/셀 속성: 셀`
  - 셀 보호: 보호 셀에는 커서가 들어가지 않는다.
  - 필드 이름: 외부 프로그램에서 한/글 문서 안의 자료를 읽고 쓸 수 있도록 셀에 이름을 부여한다.
  - 양식 모드에서 편집 가능: 양식 모드에서도 해당 셀을 편집할 수 있도록 허용한다.
- 제공 샘플:
  - `samples/셀보호.hwp`
  - `samples/셀보호.hwpx`
  - `pdf/셀보호.pdf`

## 3. 현재 상태

### 이미 해결된 부분

- HWPX `hp:tc/@name`은 `Cell::field_name`으로 파싱된다.
- HWP5 셀 필드 이름은 `raw_list_extra`에서 추출되어 `getFieldList` 가상 필드로 노출된다.
- 셀 필드 값 설정 경로는 `setFieldValueByName`에서 셀 첫 문단 텍스트를 교체한다.

### 남은 문제

- HWPX 파서가 `hp:tc/@protect`, `hp:tc/@editable`을 읽지 않는다.
- HWPX serializer가 `protect="0"`, `editable="0"`을 고정 방출한다.
- `hwpx-roundtrip samples/셀보호.hwpx`는 IR diff 0건으로 통과하지만, 실제 출력 `셀보호.rt.hwpx`에서는
  원본의 `protect="1"` 3개와 `editable="1"` 1개가 모두 `"0"`으로 바뀐다.
- 셀 `field_name` 기반 가상 필드는 `properties: 0`으로 만들어져, 양식 모드 편집 가능 여부를
  `getFieldList()`의 `editableInForm`에 반영할 수 없다.
- 표/셀 속성 UI에는 필드 이름과 양식 모드 편집 가능 컨트롤이 있으나 disabled 상태다.

## 4. 구현 목표

1. HWPX 로드 시 셀 보호와 양식 모드 편집 가능 속성을 보존한다.
2. HWPX 저장/라운드트립 시 `hp:tc/@protect`, `hp:tc/@editable`을 원본 의미대로 방출한다.
3. HWP5와 HWPX가 같은 내부 셀 속성 bit 계약을 쓰도록 맞춘다.
4. `getCellProperties`와 `setCellProperties`에서 필드 이름과 양식 모드 편집 가능을 조회/수정할 수 있게 한다.
5. 셀 필드 기반 `getFieldList()` 가상 필드의 `editableInForm`을 셀 속성과 맞춘다.
6. 제공 샘플을 회귀 테스트로 고정한다.

## 5. 비범위

- 양식 모드 전체 입력 제한 정책을 새로 설계하지 않는다.
- 셀 보호가 설정된 셀의 커서 진입 차단 UX를 새로 구현하지 않는다.
- PDF 시각 정합 개선은 이번 범위가 아니다.
- GitHub 이슈 close와 remote push는 별도 승인 후 수행한다.
