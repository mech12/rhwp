# Task #1768 수행계획서 — 배포용 문서 HWP 저장 시 일반 문서 강하

## 배경
배포용(distribution) HWPX 를 HWP5 로 직렬화하면 FileHeader 배포용 플래그는 복사되지만
`DISTRIBUTE_DOC_DATA` 레코드를 기록하지 않아 재로드가
`InvalidFile("암호 오류: DISTRIBUTE_DOC_DATA 레코드 없음")` 으로 실패 (이슈 #1768,
hwpdocs 5,000건 render-diff LOAD_FAIL 4건 전수).

## 수정 방향 (이슈 후보 중 강하 채택)
직렬화 시점의 IR 은 이미 복호화된 평문이고 rhwp 는 ViewText/DISTRIBUTE_DOC_DATA 를
생성하지 않으므로, **일반 문서로 강하**가 정합: `serialize_hwp` 에서 배포용(0x04)·
암호화(0x02) 플래그를 클리어(불리언 + flags 비트 + raw_data[36..40] 패치).
DISTRIBUTE_DOC_DATA 재생성(재암호화)은 별도 기능 영역으로 범위 밖.

## 구현 (소규모 — 단일 지점)
1. 재현 샘플 동결: `samples/task1768/distribution_doc.hwpx` (36381620).
2. `serializer/cfb_writer.rs` `serialize_hwp`: 강하 로직 (helper).
3. 테스트: 배포용 hwpx → serialize → `parse_document` 재로드 성공 + 플래그 클리어 단언.

## 게이트
- 재현: 변환 산출물 재로드 성공 (수정 전 InvalidFile).
- `cargo test --lib` + issue_852 계열(HWPX→HWP contract) + hwp5 roundtrip 관련 무회귀.
- 일반 문서(비배포) 경로 무변경 (강하 조건 미충족).
