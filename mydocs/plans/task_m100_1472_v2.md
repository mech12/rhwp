---

## 1. 목표

- IR 의 `ParaShape.indent` 를 **정답(full HWPUNIT, HWPX 일치)** 로 유지한다.
- 동시에 모의고사군 미주(endnote) **한컴-PDF 페이지네이션 parity** 를 회귀 없이 보존한다.
- 즉, 현재 `parser/mod.rs:328 ps.indent /= 2` 가 수행하던 보정을 **값 훼손이 아닌, 미주 레이아웃
  엔진 측의 국소 보정**으로 옮긴다.

## 2. 확정된 근본 원인 (요약)

- HWP5 바이너리는 indent 를 full 로 저장. `-1608` 은 `is_hwp3_variant` 분기의 `indent /= 2` 산출물.
- `is_hwp3_variant` 는 summary 의 옛 연도(예: 1997 템플릿)로 오탐되며, 모던 HWP5(3-11월/3-09월)와
  진짜 HWP3 변환본(hwp3-sample16)이 **파일 내부 신호 동일** → 탐지로 구분 불가(옵션 A 폐기).
- **load-bearing 사실(실증)**: `indent /= 2` 만 제거해도 미주 parity 테스트가 깨진다:
  - `issue_1082` 3-09'24 미주 페이지-높이 초과 29.3 → 113.5px (페이지 수 23=한컴 PDF 일치)
  - `issue_1139`(3-09'22), `issue_1256`(3-09'22), `issue_1284`(3-09'24-미주사이20) parity 회귀
- 따라서 indent /2 는 단순 버그가 아니라 미주 페이지네이션을 한컴에 맞추는 **경험적 보정**이었다.

## 3. 메커니즘 가설 (Stage 1 에서 정밀 확정)

- **본문**: 렌더러는 IR 의 `line_segs` 를 그대로 사용(재줄나눔 없음) → indent 는 줄별 X offset
  (`effective_margin_left = margin_left + line_indent`)만 바꾸고 줄 수/높이 불변
  (`equation_tac_flow.rs:151`, `paragraph_layout.rs:1936/2133`).
- **미주**: 미주 문단은 문서 끝에서 미주 단으로 재배치되며, indent 가 미주 영역의
  높이/누적 vpos/단 분할에 영향(실증상 미주 para 초과가 변함). 정확한 진입 지점(미주 단 폭
  기반 재측정/recompose 여부, 또는 vpos_adjust 의 indent 의존 경로)을 Stage 1 에서 코드+덤프로 확정.
- 후보 보정 지점(Explore 조사): `height_cursor.rs::vpos_adjust()`(146-356),
  `compact_between_notes_gap_px()`(54-69), `vpos_corrected_end_y()`. 표 셀 reflow 패턴
  `recompose_for_cell_width` 가 미주 단에도 적용되는지 확인.

## 4. 재설계 방향 (Stage 1 결과로 택1·구체화)

- **(가) 미주 단 indent 정규화**: 미주 영역 레이아웃 진입 시, 변환본/모의고사 케이스에서
  미주 문단 폭/높이 계산에 한해 indent 를 한컴 정합 스케일로 보정(전역 IR 값은 full 유지).
  → 보정을 미주 경로에 국한, 본문/HWPX/serializer 는 full 사용.
- **(나) 미주 high-fidelity 재줄나눔**: 미주 단 폭 + full indent 로 한컴처럼 재줄나눔하여
  높이를 한컴과 정합(근본적이나 #1184 영역, 범위 큼).
- 선택 기준: 회귀 0(4개 parity + sample16 PDF + 미주 회귀 가드) + 본문/serializer/HWPX 무변경.
