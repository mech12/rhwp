# 수행계획서 — Task #1858

## 이슈

[#1858](https://github.com/edwardkim/rhwp/issues/1858) — 페이지-절대 앵커(vert=용지/쪽)
개체 배치 오차: flow 스택 폭발(3143097) + 하단 valign=Bottom 세로 오프셋(36389312)

## 조사 요약 (코드 맵 + 실측)

`typeset_block_table`(src/renderer/typeset.rs)에 페이지-절대 앵커 처리가 이미 성숙하게
존재한다(Task #321 Paper, #1611/#1624/#1658 Page+Bottom). 두 발현 모두 이 튜닝된 코드의
**잔여 오차**이며 근본 원인이 서로 다르다.

### 발현 1 — Paper flow 폭발 (3143097): 근본 원인 확정

절대배치 가드(typeset.rs `is_paper_topbottom_block`, ~11827)는
`target_y = host 문단 vpos` 기준 `target_y > current_height && target_y + pre_lines_h <= available`
일 때만 절대배치(0 flow)한다. 같은 host 문단(pi=2)에 co-anchored 된 Paper 상자 22개는
target_y 가 **모두 동일(host vpos=17287)** 이라 첫 상자만 통과하고, 나머지 21개는
current_height 가 이미 sync 되어 가드 실패 → generic flow 경로(~11997)에서 각자
table_total 만큼 flow 소비 → 페이지 폭발(한컴 1쪽 vs rhwp 4쪽, #1853 후 3쪽).

### 발현 2 — Page+Bottom 세로 오프셋 (36389312): 조사 결과 잔여, 별도

pi=5/pi=6(vert=쪽 valign=Bottom)은 `is_page_bottom_fixed_float` 제외 경로(11893)로 올바르게
배치되어 **페이지 수는 정합(1쪽, #1611/#1658 이 이 문서로 튜닝)**. 다만 하단 블록 전체가
한컴 대비 세로 ~34px 어긋난다(오라클 현장대응단장 헤딩 y=806.4 vs rhwp ~772). 원인 후보는
pi=6(12×41 발신 grid)의 높이 측정 과대 또는 valign=Bottom 기준 좌표. 성숙 코드(#1611/#1658)의
높이/기준 세부라 회귀 위험이 높고 근본 원인 확정에 추가 조사 필요.

## 방침 — 발현 1 우선, 발현 2 분리

발현 1은 근본 원인이 명확하고 surgical 검증이 가능하므로 이번 타스크에서 수정한다.
발현 2는 성숙 코드의 높이/기준 잔여로 회귀 위험이 높아 별도 후속으로 분리한다
(이슈 #1858 본문의 "조사 후 필요 시 하위 분리" 방침).

## 수정 (발현 1)

절대배치 가드에서 같은 host 문단에 선행 Paper 자리차지 표가 있으면(= co-anchored 그룹의
2번째 이후) 후속 상자도 절대배치(0 flow)한다. 첫 상자는 기존대로 host vpos sync,
단독 Paper 상자(선행 없음·sync 불가)는 기존대로 flow.
