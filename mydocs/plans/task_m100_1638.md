# Task M100 #1638 수행 계획

## 배경

이슈 #1638은 PR #1366의 `hwp2hwpx --verify` / `--verify-pages` UX 중 검증 게이트만 현재 `devel`
기준으로 재구성하는 작업이다. 현재 CLI에서 HWPX 출력은 `export-hwpx`, HWP 출력은 `convert`가
담당하므로 옛 `hwp2hwpx` 명령은 되살리지 않는다.

## 범위

- `export-hwpx`에 `--verify`와 `--verify-pages`를 추가한다.
- `convert`에 `--verify`와 `--verify-pages`를 추가한다.
- `convert --verify`는 HWPX 입력에서 어댑터가 문서를 정규화할 수 있으므로, 변환 전 원본 IR이 아니라
  HWP 저장 직전의 어댑터 적용 후 IR과 재파싱 IR을 비교한다.
- CLI help와 `mydocs/manual/cli_commands.md`를 현재 동작에 맞게 갱신한다.

## 검증 방향

- `export-hwpx --verify-pages`: 변환 전/후 페이지 수를 비교하고 불일치 시 종료 코드 4.
- `export-hwpx --verify`: 변환 후 산출물을 재파싱하여 IR diff를 수행하고 차이 발생 시 종료 코드 3.
- `convert --verify-pages`: 저장 전/재파싱 후 페이지 수를 비교하고 불일치 시 종료 코드 4.
- `convert --verify`: 저장 후 산출물을 재파싱하여 어댑터 적용 후 IR과 재파싱 IR을 비교하고 차이 발생 시
  종료 코드 3.
- CLI 바이너리 integration test로 `export-hwpx`와 `convert` 검증 게이트 경로를 모두 확인한다.

## 주의 사항

- 신규 명령은 추가하지 않는다.
- 검증 실패 종료 코드는 PR #1366 검토에서 유용하다고 판단된 `3`/`4`를 유지한다.
- `convert --verify`는 원본 IR 직접 비교가 아니라 저장 직전 IR과 재파싱 IR 비교라는 의미를 명확히 유지한다.
