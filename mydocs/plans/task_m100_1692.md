# Task #1692 계획 - HWP3 글색상 손실

## 배경

- 이슈: <https://github.com/edwardkim/rhwp/issues/1692>
- 부모 이슈: #1689 SO-SUEOP HWP3 렌더링 차이 분리
- 증상: HWP3 원본의 파란 필기/주석 글색이 SVG에서 검정으로 렌더링된다.

## 확인 자료

- `samples/SO-SUEOP.hwp`: HWP3 원본 샘플
- `samples/SO-SUEOP.hwpx`: 같은 문서의 HWPX 변환 기준 샘플
- `pdf/SO-SUEOP-2024.pdf`: 한컴 PDF 기준 파일, 46쪽

## 원인 가설

- `Hwp3CharShape`는 `text_color`를 읽지만 `convert_char_shape`에서 `CharShape.text_color`로 전달하지 않는다.
- HWP3 글자색은 1바이트 기본 8색 인덱스이고, rhwp 내부 `ColorRef`는 `0x00BBGGRR` 형식이다.

## 완료 기준

- HWP3 `text_color` 인덱스가 `CharShape.text_color`로 보존된다.
- SO-SUEOP HWP3 원본 파싱 결과에 파란 글자색 `0x00FF0000`이 존재한다.
- HWPX 기준 샘플도 동일한 파란 글자색을 갖는지 함께 검증한다.
- 기존 검정 글자색 렌더링은 유지한다.
