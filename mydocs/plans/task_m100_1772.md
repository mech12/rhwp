# Task M100 #1772 수행 계획서 — HWPX 파서 표 outMargin → common.margin 동기화

## 배경

Issue #1772 (HWPX/HWP5 파스 경로별 saved-vpos 신뢰 차이 — 11.36px 이동 군집 50건+).
조사 결과 신뢰 로직 차이가 아니라 **HWPX 파서의 IR 계약 위반**으로 원인이 확정되었다.
상세 조사 기록: `mydocs/troubleshootings/hwpx_table_out_margin_common_margin_sync.md`.

## 원인 요약

- HWPX 파서(parse_table)가 `<hp:tbl><hp:outMargin>` 을 `table.outer_margin_*` 에만 기록,
  `table.common.margin` 은 0 으로 방치.
- 레이아웃의 쪽 고정 자리차지 표 본문 예약(`calc_shape_bottom_y`)과 HWPX→HWP 어댑터
  (`materialize_table_outer_margin`)는 `common.margin` 기준으로 동작.
- 결과: HWPX 직파스 문서에서만 표 바깥 아래 여백 누락 → 본문이 한컴 저장 lineseg 위치보다
  11.36px(3mm) 위로 렌더. HWP5 재파스본이 한컴 정합(정답).

## 수정 내용

1. `src/parser/hwpx/section.rs` parse_table 종료 시
   `table.common.margin ← table.outer_margin_*` 동기화 (어댑터의 materialize 와 동일 규칙).
2. 재현 샘플 `samples/task1772/table_outer_margin_common_sync.hwpx` (36381023, PII 방침 A).
3. 회귀 테스트 `tests/issue_1772_table_outer_margin_sync.rs`:
   - IR 계약: 모든 표 `common.margin == outer_margin_*`
   - 렌더: 본문 첫 줄 y ≈ 306.7px (저장 lineseg 정합, 수정 전 결함값 295.4)
