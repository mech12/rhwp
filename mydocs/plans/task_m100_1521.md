## 현상

브라우저 확장의 HWP/HWPX 링크 hover 미리보기에서 링크에 마우스를 올린 뒤, 카드 표시 지연 시간보다 빨리 링크를 벗어나면 미리보기 카드가 뒤늦게 나타나 화면에 남을 수 있다.

재현 흐름은 다음과 같다.

1. HWP/HWPX 링크에 `mouseenter`
2. content-script가 hover card 표시를 지연 타이머로 예약
3. 카드가 표시되기 전 링크에서 `mouseleave`
4. 기존 show 예약이 취소되지 않음
5. 예약된 `showHoverCard(anchor)`가 뒤늦게 실행
6. 이미 마우스가 링크/카드 밖에 있으므로 후속 leave 이벤트가 없어 카드가 유지

## 사전 분석

### Chrome / Firefox

`rhwp-chrome/content-script.js`와 `rhwp-firefox/content-script.js`는 show 예약과 hide 예약을 하나의 `hoverTimeout`으로 관리한다.

- `mouseenter`: 기존 `hoverTimeout`을 clear하고 `showHoverCard(anchor)`를 300ms 뒤 예약
- `mouseleave`: 기존 show 예약을 먼저 clear하지 않고 `hideHoverCard()`를 200ms 뒤 예약
- `showHoverCard(anchor)`: 실행 직전에 anchor가 여전히 hover 대상인지 검증하지 않음

따라서 `mouseleave`가 발생해도 pending show가 남을 수 있고, show와 hide 예약이 같은 변수에 덮여 상태 추적이 흐려진다.

### Safari

`rhwp-safari/src/content-script.js`는 `activeAnchor`를 별도로 갖고 있지만 show/hide 타이머는 여전히 단일 `hoverTimeout`이다.

- `mouseleave`에서 기존 show 예약을 clear하는 경로는 있으나 show 실행 시점의 active/hover 검증이 부족하다.
- 카드 썸네일 비동기 응답은 현재 카드 생존 여부 검증이 Chrome/Firefox보다 약하다.
- Safari 전용 구현도 같은 lifecycle 모델로 정렬해야 플랫폼별 동작 차이를 줄일 수 있다.

## 작업 목표

링크 hover가 끝난 뒤에는 예약된 hover card 표시가 실행되지 않도록 한다.

정정 후 기대 상태:

- 카드 표시 지연 시간 전에 링크를 벗어나면 카드가 생성되지 않는다.
- 링크를 충분히 hover하면 기존처럼 카드가 표시된다.
- 링크에서 카드로 자연스럽게 이동하면 카드가 유지된다.
- 카드에서 벗어나면 카드가 닫힌다.
- 빠르게 여러 HWP/HWPX 링크를 지나갈 때 이전 링크의 카드가 늦게 표시되지 않는다.
- Chrome/Firefox/Safari content-script의 hover lifecycle 의미가 같은 구조로 정리된다.

## 검증 게이트

| 검증 | 기준 |
|------|------|
| 빠른 leave | show 지연 시간 전에 링크를 벗어나면 카드가 생성되지 않음 |
| 정상 hover | 지연 시간 이상 hover하면 카드가 표시됨 |
| 링크→카드 이동 | 링크에서 카드로 이동할 때 카드가 유지됨 |
| 카드 leave | 카드에서 벗어나면 hide 지연 후 카드가 닫힘 |
| 빠른 링크 전환 | 이전 링크의 stale show 타이머가 새 상태를 덮지 않음 |
| JS 문법 검사 | 수정한 content-script 모두 `node --check` 통과 |
| 확장 빌드 | Chrome/Firefox `npm run build` 통과, Safari는 가능한 범위 명시 |
| 보안 회귀 | hover card DOM API/textContent 흐름 유지, innerHTML 재도입 없음 |
