## 범위 (이슈의 1~2단계)
1. **측정 하니스** `tools/metric_drift_survey.py` 신설:
   - 파일별: 한글 PDF 생성(pyhwpx) → PyMuPDF 로 페이지별 텍스트 baseline(줄 pitch) +
     `find_tables` 표 bbox / rhwp `export-svg`(baseline) + `export-render-tree`(표 bbox).
   - 표 매칭: 같은 페이지에서 중심점·크기 근접(정규화 IoU) — 폼 무테 결재란 미검출은
     "매칭 없음" 정직 표기(object_visual_regression 컨벤션).
   - 산출 TSV: 페이지 단위(pitch delta) + 표 단위(h/w delta) + 파일 요약.
2. **1차 서베이**: razor-thin 확정 사례(S1/S2 잔여) + MATCH 표본 일부(대조군)에 실행 →
   delta 분포·클러스터 분석 → 지배 원인 상위군 보고.

**범위 밖**: 3단계(지배 원인별 수렴 수정)는 서베이 결과에 따라 개별 이슈 분리.
