## 3. 구현 순서

### Stage 1 — 복학원서 잔여 차이

우선순위 1순위다. 자동 후보가 `line_order_overlap` 하나로 좁혀졌고, render tree 후보도
`pi=16` 의 PUA 선 문자 문단과 하단 접수증 표, 글앞 타원 shape 조합으로 좁혀졌다.

목표:

- 복학원서 p1 의 `line_order_overlap` 플래그를 해소하거나, 플래그가 false positive 라면 검출/회귀 게이트를
  더 정확히 만든다.
- 원형/도형 위치가 기준 PDF 방향으로 이동하는지 visual sweep 으로 확인한다.
- `tests/issue_2020.rs` 에 복학원서 하단 표/도형 배치 회귀 가드를 추가한다.

### Stage 2 — 여권신청서 RowBreak/TAC 표 흐름

여권신청서는 `pi=0` 의 문서 전체 크기 35x13 RowBreak/TAC 표에 거의 모든 내용이 들어 있다.
표 내부 LineSeg, 셀 높이, 문단 흐름, 하단 용지규격 문구가 함께 얽혀 있어 Stage 1보다 회귀 위험이 크다.

목표:

- HWP 2022 기준 PDF 와 공식 PDF 중 어떤 기준을 1차 oracle 로 삼을지 다시 분리 기록한다.
- `210mm×297mm[백상지 120g/㎡]` 하단 문구 overflow 후보를 회귀 테스트로 고정한다.
- 공식 PDF 기준 line/column drift 는 표 내부 셀 높이와 행 높이 계산 후보를 별도로 좁힌다.

### Stage 3 — FSC 폰트/굵기/자간

FSC HWP/HWPX p1 은 자동 플래그 0이나 visual proxy 는 낮다. 이는 layout 붕괴보다는 폰트/metrics/fallback
환경 차이일 가능성이 높다.

목표:

- 현재 시스템 폰트 기준과 기준 PDF의 차이를 문서화한다.
- font alias/metrics 쪽의 실제 코드 결함인지, 한컴 전용 폰트 부재에 따른 환경 차이인지 분리한다.

## 5. 완료 기준

- #2020 본문 증상별 잔여 항목이 코드/테스트/문서로 재분류되거나 해결된다.
- 해결한 항목은 자동 회귀 테스트 또는 visual sweep 대표 asset 으로 남긴다.
- 한컴 전용 폰트가 없는 공개 기본 검증 환경에서는 pixel-level glyph/font fidelity 를 #2020 close 기준으로 삼지 않는다.
- #2020 close 기준은 페이지 수, 표/문단 흐름, 객체 bbox, overlap, PUA marker, advance, visual sweep 구조 플래그이다.
- 한컴 폰트가 설치된 환경의 추가 글꼴 fidelity 개선은 별도 선택 검증/장기 축으로 분리해 PR 본문과 이슈 코멘트에 명확히 기록한다.
