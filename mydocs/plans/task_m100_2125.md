다음 결정은 선행 리뷰에서 이미 확정됐으므로 다시 선택지로 열지 않는다.

- canonical source root는 저장소 공통 `assets/fonts`다.
- `rhwp-studio/public/fonts`는 canonical이 아니라 runtime public path를 제공하는 소비자다.
- Chrome/Firefox/Safari/VS Code도 canonical source의 소비자이며 별도 원본을 소유하지 않는다.
- runtime URL `fonts/...`, extension `dist/fonts`, VS Code `dist/media/fonts`는 공개·배포 경로이므로
  source directory 이름과 구분한다.
- font fallback, 파일 구성, 폰트 내용, manifest/CSP/WAR, `/web` legacy 삭제는 이번 작업에 섞지 않는다.

## 1. 목적

현재 실사용 font binary와 license 문서는 legacy `web/fonts`가 소유한다. 이 때문에 Studio public
symlink, 브라우저 확장 build, VS Code webpack, metrics/test/tooling과 CI path filter가 legacy `/web`에
결합돼 있다.

이번 작업은 font asset의 source ownership만 공통 `assets/fonts`로 옮기고 모든 소비자가 명시적인
copy/link 계약으로 같은 파일을 받도록 정리한다. 성공 기준은 디렉터리 이동 자체가 아니라 다음 세 가지다.

1. canonical source와 runtime/distribution 경로가 구분된다.
2. 모든 소비자가 이전과 동일한 36개 font와 동일한 runtime URL을 받는다.
3. 향후 `assets/fonts`만 바뀌는 PR도 frontend/Render Diff gate가 자동 실행된다.

## 3. 수행 범위

### 3.1 canonical ownership

- `assets/fonts`를 font binary와 해당 license/inventory 문서의 canonical source로 만든다.
- 현재 36개 WOFF2의 이름과 내용은 변경하지 않는다.
- `FONTS.md`와 Source Han Serif Korean OFL 근거를 binary와 같은 canonical root에 둔다.
- 저작권 보호 대상 로컬 font의 `.gitignore` 경로도 새 canonical root로 옮긴다.
- 동일 binary를 여러 source directory에 복제하지 않는다.

### 3.2 소비자 source path

- `rhwp-studio/public/fonts`가 새 canonical root를 가리키도록 한다.
- Chrome/Firefox build가 `assets/fonts` 전체 WOFF2를 복사하도록 한다.
- Safari가 Chrome dist를 통해 동일 asset을 받는 계약을 검증한다.
- VS Code의 기존 11개 subset 정책을 유지하고 source root만 바꾼다.
- metrics, extension dist test, CanvasKit font E2E, subset 생성 도구가 새 canonical root를 사용한다.
- `npm/editor`와 관련 사용자 문서의 self-hosting 설명을 새 ownership에 맞춘다.

### 3.3 CI와 변경 감지

- `.github/workflows/ci.yml`의 push/PR `paths-ignore: assets/**`를 비런타임 asset 하위 경로로 좁혀
  `assets/fonts/**` 변경에서 workflow 자체가 시작되게 한다.
- `.github/workflows/ci.yml`의 frontend 영향 판정에 `assets/fonts/`를 명시한다.
- `.github/workflows/render-diff.yml`이 `assets/fonts/**` 변경을 감지하게 한다.
- rename PR에서는 `previous_filename=web/fonts/...`로 우연히 gate가 실행되는 것에 의존하지 않는다.
- 향후 font-only 수정도 fresh WASM frontend package gate와 Render Diff 검토 대상으로 남긴다.

### 3.4 legacy `/web` 호환

`web/editor.html`은 상대 URL `fonts/...`를 사용한다. `/web` 삭제는 범위 밖이므로 canonical 이동 때문에
기존 legacy runtime을 암묵적으로 깨뜨리지 않는다.

구현 계획서는 다음 우선순위로 호환 방식을 확정한다.

1. 중복 source 없이 `web/fonts` compatibility link로 기존 상대 URL을 보존할 수 있는지 검증한다.
2. symlink가 build/checkout 환경에서 안전하지 않다면 명시적인 legacy serve/copy adapter를 검토한다.
3. legacy URL 변경이나 `web/editor.html` 제거가 필요하면 이번 PR에 넣지 않고 별도 이슈로 분리한다.

## 4. 범위 제외와 금지 목록

### 4.1 범위 제외

- `/web` JS/HTML/CSS 전체 삭제 또는 archive
- font fallback chain, substitution, `font-display`, unicode range 변경
- 새 font 추가, 기존 font 삭제, 파일명 변경, subset 재생성
- CDN font 정책 또는 `disableExternalWebFonts` 의미 변경
- `InputHandler`, dialog, `WasmBridge`, `diff-engine`, `core/types.ts` 해체
- Rust/WASM API 또는 renderer/layout 동작 변경
- runtime UI framework 도입

### 4.2 확정 금지

- `@rhwp/editor` runtime dependency 또는 iframe/MessageChannel public contract 변경
- extension CSP 완화, inline script 추가, `web_accessible_resources` 확대
- Chrome/Firefox `publicDir: false` 우회
- Safari를 제외한 2-browser 결과만으로 asset copy 계약 완료 선언
- VS Code 11개 subset의 임의 확대·축소
- 저작권 보호 대상 한컴/MS font를 Git 또는 배포 artifact에 포함
- source move와 font binary 내용 변경을 같은 commit에 혼합
- #2124 baseline snapshot을 새 경로 기준으로 다시 쓰기
- `/web` 삭제와 canonical 이전을 같은 PR에 혼합

React/Vue/Svelte 등 runtime UI framework 금지는 #2023 v2의 프론트 전체 guardrail로 계속 적용한다.
이번 이슈에는 UI 구현 자체가 없으므로 관련 dependency 변경은 허용하지 않는다.

## 5. 소비자별 목표 계약

| 소비자 | canonical source | 유지할 배포/runtime 계약 | 필수 검증 |
|--------|------------------|--------------------------|-----------|
| Studio | `assets/fonts` | public `fonts/...` URL 유지 | symlink target, build, browser font load |
| Chrome | `assets/fonts` 전체 36개 | `dist/fonts`, WAR `fonts/*` 유지 | build 후 이름·개수·hash |
| Firefox | `assets/fonts` 전체 36개 | `dist/fonts`, WAR `fonts/*` 유지 | Chrome과 exact parity |
| Safari | Chrome dist 경유 | `dist/fonts`, 더 엄격한 WAR 유지 | Chrome inheritance, Safari build/manifest |
| VS Code | `assets/fonts` 11개 subset | `dist/media/fonts`, webview URI/CSP 유지 | compile 후 exact subset·hash |
| npm/editor | 직접 font bundle 없음 | Studio/self-hosting 안내와 zero dependency 유지 | package test, 문서 경로 |
| legacy `/web` | canonical 아님 | `web/editor.html`의 `fonts/...` 접근 보존 | compatibility path smoke |
| metrics | `assets/fonts` inventory | #2124 snapshot은 불변 | 현재 inventory와 baseline delta 설명 |

source path를 바꾸더라도 `font-loader.ts`, CanvasKit fetch, manifest의 `fonts/*`처럼 runtime URL을
표현하는 문자열은 변경 대상이 아니다. source path와 runtime URL을 기계적으로 일괄 치환하지 않는다.

## 6. 예상 단계와 승인 관문

### Stage 1. 현재 inventory와 dependency graph 고정

1. `e750e02f` 기준 36개 파일의 이름, bytes, SHA-256과 license file을 migration manifest로 기록한다.
2. #2124 snapshot과 비교해 #2190 한 파일 외의 미설명 delta가 0인지 확인한다.
3. tracked source, symlink, build, test, tool, workflow, documentation의 `web/fonts` 참조를 전수 분류한다.
4. `web/editor.html` compatibility 방식과 Windows/symlink 위험을 구현 계획서에서 확정한다.

관문: 현재 source-of-truth와 모든 소비자가 표로 설명되지 않으면 이동을 시작하지 않는다.

### Stage 2. canonical root와 ownership 이전

1. 36개 WOFF2와 canonical license/inventory 문서를 `assets/fonts`로 이동한다.
2. 이동 전후 filename, bytes, SHA-256의 1:1 대응을 검증한다.
3. Studio와 legacy `/web`의 link/compatibility path를 설정한다.
4. 보호 font `.gitignore` 규칙을 새 canonical root에 맞춘다.

관문: 파일 추가·삭제·내용 변경이 0이고 canonical binary source가 한 곳이어야 한다.

### Stage 3. build·test·tool·CI source path 갱신

1. Chrome/Firefox build와 VS Code webpack source path를 갱신한다.
2. metrics, extension dist test, CanvasKit font E2E, subset tool의 canonical path를 갱신한다.
3. frontend impact detector와 Render Diff path filter에 `assets/fonts`를 반영한다.
4. runtime URL, fallback, manifest CSP/WAR에는 의미 변화가 없음을 diff로 확인한다.

관문: `web/fonts` 직접 source 의존은 승인된 legacy compatibility 경로 외에는 남지 않아야 한다.

### Stage 4. 소비자별 artifact와 보안 계약 검증

1. fresh WASM을 생성하고 #2183 frontend package gate와 같은 package build/test를 실행한다.
2. Studio에서 대표 local font와 #2190 기호 coverage를 browser E2E로 확인한다.
3. Chrome/Firefox dist의 36개 font가 canonical source와 동일한지 검증한다.
4. Safari inheritance/build와 VS Code 11개 subset을 검증한다.
5. CSP, WAR, inline script, `publicDir: false`, editor zero dependency에 delta가 없음을 확인한다.

관문: 필수 소비자 한 곳이라도 누락·404·hash 불일치가 있으면 문서 단계로 넘어가지 않는다.

### Stage 5. 문서·metrics·종료 정리

1. ownership/copy matrix, `FONTS.md`, `THIRD_PARTY_LICENSES.md`, npm/editor, extension/VS Code 관련 문서를 갱신한다.
2. #2124 공식 snapshot을 비교 입력으로만 사용해 frontend metrics를 실행한다.
3. 코드 복잡도는 path-only 변경에서 0-delta를 기대하며 차이가 있으면 원인을 설명한다.
4. `/web` legacy 삭제 가능 여부와 후속 이슈 필요성을 근거와 함께 판단한다.
5. 단계 보고서와 최종 보고서에서 #2125 체크리스트를 근거 파일·gate에 연결한다.

관문: 후속 이슈 생성, GitHub 코멘트, PR 생성은 초안을 작업지시자에게 먼저 제시하고 승인 후 수행한다.

## 10. SOLID·복잡도 적용 방식

이번 작업은 대형 함수 해체가 아니라 asset ownership 정리다. 따라서 SOLID 총점을 새로 계산하거나
기존 미채점 상태를 점수로 승격하지 않는다.

- OCP/DIP 관점의 목표는 새 소비자가 legacy `/web` 내부 구조가 아닌 canonical asset contract를
  의존하게 하는 것이다.
- source path의 통이동 자체를 SOLID 개선 점수로 계상하지 않는다.
- metrics의 complexity/LOC는 0-delta가 기본 기대다.
- #2130과 #2124에서 고정한 Total CC, Top 20 합, CC>25 합, stable function diff를 함께 비교한다.
- CI detector나 test code 변경으로 함수 지표가 달라지면 위 총량과 함수별 원인을 공개한다.
- Phase B hotspot 우선순위나 #2124 공식 baseline 수치를 이번 이슈에서 재정의하지 않는다.

## 12. 완료 기준

- `assets/fonts`가 유일한 canonical font source로 문서화된다.
- 현재 36개 WOFF2의 이름·내용·총 크기가 이동 전후 동일하다.
- #2190의 현재 Noto Sans KR hash가 migration 기준으로 보존된다.
- Studio, Chrome, Firefox, Safari, VS Code, npm/editor, legacy `/web`의 수신 계약이 명확하다.
- Studio/extension runtime URL과 VS Code webview 경로가 유지된다.
- frontend CI와 Render Diff가 향후 `assets/fonts` 변경을 감지한다.
- extension CSP/WAR/inline script/publicDir guardrail에 변화가 없다.
- font fallback, 파일 구성, public API, 기능·보안 정책에 변화가 없다.
