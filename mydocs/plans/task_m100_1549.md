# Task #1549 수행계획서 — visible host 제목이 co-anchored float 표 아래로 밀리고 표/제목 간격이 어긋남

## 1. 이슈

- GitHub: <https://github.com/edwardkim/rhwp/issues/1549>
- 제목: `[HWP/렌더링] visible host 문단 텍스트(제목)가 양수 offset co-anchored float 표 아래로 밀려 렌더됨 (#1510 계열, #1535와 별개)`
- 상태: OPEN
- 관련: #1510(PR #1518), #1535(PR #1548 — 본 작업이 그 위에 스택)

## 2. 재현 (확정)

작업지시자 제공 한글 양식(작업일지류, 비공개)과 issue1510 구조를 narrow 한 합성 fixture로 확정.

- visible host 문단(섹션 제목 텍스트 있음)에 양수 offset co-anchored TopAndBottom float 표가 있을 때,
  host 제목이 표 *아래*로 밀려 렌더된다(한컴은 제목을 문단 앵커=표 위에 둠).
- render tree 실측(나린뜰_일일작업일지.hwp): 제목 "완제품 생산" y=193.9‥212.5, 관능검사 표 y=193.9‥393.2
  → 제목이 표 상단과 같은 y 에서 시작해 겹침/아래로 밀림.
- 트리거: 선행 float exclusion(예: 작성/승인 박스 [116,193.9])이 제목·표를 같은 y 로 밀어, 표의 작은 양수
  offset(5.4mm)이 흡수되어 둘이 붙음. 큰 offset(#1510 A=60mm)은 표가 제목 한참 아래라 안 겹쳐 정상.

## 3. 근본 원인 (런타임 계측 확정)

`src/renderer/typeset.rs` `place_table_with_text` 에서 visible-para float 의 host 제목은 pre-text 가 아니라
표들 뒤 **post-text**(PageItem `[표…, 제목]`)로 emit 되고, `src/renderer/layout.rs` 의 exclusion consume 가
**자기 문단 표가 만든 zone 까지** 적용해 제목을 zone 하단으로 점프시킨다. 이어 제목·표가 선행 float
exclusion 으로 같은 y 까지 밀리며 offset 이 흡수돼 붙는다. 또한 표의 외곽여백(`outer_margin_top`/`_bottom`)이
exclusion/배치에 반영되지 않아 제목↔표·표↔다음 섹션 간격이 0 이 된다.

- #1535(PR #1548)는 float 표가 선행 float 점유영역을 consult 하게 했으나 가드가 `is_current_visible_para_float`
  라 빈-host float 을 건너뛰고, host 제목 줄·외곽여백은 다루지 않는다.

## 4. 수정 방향 (작업지시자 승인: B = #1548 위 스택, 한컴 PDF 정답지 기준 정밀 매칭)

`layout.rs` 만 수정. #1535 의 `table_y_start` exclusion 클램프 위에 다음 4 가지를 올린다.

1. `VisibleFloatExclusion` 에 `owner_para` 추가 → consume 시 같은 문단 텍스트는 자기 표 zone skip(제목이
   자기 표 아래로 안 밀림).
2. #1535 클램프 가드를 `is_para_topbottom_float` 로 확장(빈-host float 도 선행 밴드 아래로). 빈-host 는 zone
   하단 아래로 자기 offset 만큼 복원해 표-표 간격을 만든다(visible host 는 3 이 처리).
3. visible-host 양수 float 표를 host 제목 줄(선행 exclusion 아래 흐름 + 제목 줄높이 + `outer_margin_top`)
   아래로 클램프 → 제목/표 겹침 해소 + 제목↔표 간격. 큰 offset 은 max 라 무영향, v_off==0 보존(#1510).
4. float exclusion 하단을 `outer_margin_bottom` 만큼 확장 → 표↔다음 섹션 제목 간격.

리스크: 모두 visible_float_exclusion 클램프 계열이라 #1510/#1535 회귀 위험 → 회귀 테스트로 가드, 전체
스위트로 검증. 코어 렌더 트리 좌표 계산 외 다른 모듈 미변경.
