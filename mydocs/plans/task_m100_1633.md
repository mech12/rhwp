## 문제 요약

`samples/대각선샘플.hwp`에서 우하단 굵은 X 대각선 셀을 선택한 뒤
rhwp-studio의 `셀 테두리/배경 > 대각선` 탭을 열면, 한컴오피스와 달리
현재 셀의 대각선 상태가 UI와 미리 보기에 반영되지 않는다.

사용자가 제공한 한컴 화면에서는 선택 셀의 `\` 대각선과 `/` 대각선, 굵기,
색이 대화상자에 반영된다. 반면 rhwp-studio는 기본값처럼 `실선`, `0.1mm`,
검정, 대각선 미선택 상태를 표시한다.

## 현재 분석

#1623 작업에서 대각선/중심선 필드는 이미 `getCellProperties` JSON과
rhwp-studio 모달에 연결됐다.

이번 샘플의 우하단 X는 셀 자체 `border_fill_id`가 아니라 표의 `cellzone`
overlay로 적용된다.

- `zone[1] row=2..2 col=2..3 bf=11`
- `bf=11`은 slash/backSlash 대각선 정보를 갖는다.
- 현재 `get_cell_properties_native`는 `cell.border_fill_id`만 사용한다.

따라서 UI 초기화가 실제 화면에 적용된 effective border fill이 아니라 셀 고유
border fill만 읽는 것이 1차 원인이다.

## 작업 범위

1. `getCellProperties`가 선택 셀을 덮는 `cellzone` border fill을 effective 값으로
   반환하도록 보강한다.
2. 겹치는 zone이 있을 경우 렌더 순서와 맞춰 뒤쪽 zone을 우선한다.
3. `samples/대각선샘플.hwp` 기준으로 우하단 cellzone 대각선 값이
   `getCellProperties`에 반영되는 focused 테스트를 추가한다.
4. rhwp-studio 모달 코드는 필요한 경우에만 최소 보정한다.

## 제외 범위

- cellzone overlay 자체를 편집/삭제하는 새 UX는 이번 범위에 넣지 않는다.
- 한컴 UI 전체의 픽셀 단위 복제는 다루지 않는다.
- 저장 정책 변경은 focused 테스트로 필요한 범위에서만 다룬다.
