# Task #666 수행계획서 — Neumann ingest roundtrip 검증 재개

## 배경

- 이슈: https://github.com/edwardkim/rhwp/issues/666
- 선행: #660/#668 에서 `rhwp build-from-ingest` CLI, ingest schema/parser, 시험지 빌더, `rhwp-exam-ingest` skill 골격이 들어왔다.
- #666 은 기존 PDF/SVG visual sweep 도구를 새로 만드는 작업이 아니라, ingest pipeline 으로 생성한 HWPX 의 roundtrip 성격을 확인하는 후속 검증 이슈다.

## 현재 확인

- 기존 visual sweep 인프라는 `scripts/task1274_visual_sweep.py` 와 `mydocs/manual/visual_sweep_guide.md` 로 이미 존재한다.
- #666 의 핵심 DoD 는 visual sweep 자체가 아니라 `Vision/ingest.json -> build-from-ingest -> dump/export-text/export-svg` 경로에서 텍스트 보존과 결함 분류를 확인하는 것이다.
- `tools/rhwp-ingest/schema/sample_minimal.json` 으로 현재 devel 의 `build-from-ingest` 를 실행하면 HWPX 직렬화가 실패한다.
  - 실패: `XML 쓰기 실패: 미등록 ID 참조 발견: charPrIDRef: [0]`
  - 원인 후보: `build_exam_paper()` 가 `Document::default()` 에 문단 `char_shape_id=0` 을 참조시키지만, `doc_info.char_shapes`/style pool 을 등록하지 않는다.

## 작업 방침

1. 새 visual sweep 도구는 만들지 않는다. 필요 시 기존 visual sweep 가이드를 사용한다.
2. 먼저 `build-from-ingest` 최소 샘플이 HWPX 직렬화와 `export-text` 를 통과하도록 빌더의 기본 문서 속성을 보정한다.
3. 보정은 샘플명/페이지/임의 계수가 아니라, HWPX 직렬화 계약상 필요한 `DocInfo` 기본 style pool 등록에 근거한다.
4. 이후 #666 의 4종 시험지 검증은 별도 ingest 산출물이 필요하므로, 본 단계에서는 기반 명령의 회귀 복구와 검증 방법을 명확히 한다.
5. 대표 PDF 4종은 전체 텍스트 레이어를 ingest 로 구성해 `build-from-ingest -> export-text -> dump -> export-svg` 경로를 검증하고, Vision 기반 구조화 산출물 전수 검증은 후속으로 분리한다.

## 완료 기준

- `tools/rhwp-ingest/schema/sample_minimal.json` 이 `rhwp build-from-ingest` 로 HWPX 생성된다.
- 생성 HWPX 에 대해 `rhwp export-text` 가 성공하고 문항/선택지 텍스트가 보존된다.
- 필요 시 `dump` 또는 `export-svg` 최소 확인을 수행한다.
- #666 이 “기존 visual sweep으로 대체되어 close 가능”한 상태가 아니라, ingest roundtrip 보정/검증 이슈로 계속 유효하다는 판단을 문서화한다.
