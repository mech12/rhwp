# Task #1716 수행계획서 — 반복 제목행 overhead 누적 폭주 수정

- 이슈: edwardkim/rhwp#1716 (M100)
- 브랜치: `local/task1716` (devel 기반)
- 유형: 표 레이아웃 버그 (렌더링 페이지수/PI 정합)

## 1. 배경 / 문제

hwpdocs 표본 3000개 PI↔페이지(한글 OLE) 검증에서 최악 아웃라이어:
`admrul_downloads/국토교통부/3031803_[별표 2] 건설공사 품질시험기준(…).hwpx`
→ rhwp **173쪽** vs 한글 **52쪽** (+121).

표 pi=12(183행×7열, RowBreak)가 행 60부터 **페이지당 1행씩** 배치되어 폭증.

## 2. 근본 원인 (진단 확정)

반복 제목행(repeat-header) overhead를 **`cursor_row`(렌더러는 `start_row`) 아래에 있는 모든
`is_header` 행 높이의 합**으로 계산한다. "제목행은 표 상단에만 있다"는 가정이나, 이 문서는
`<hp:tc header="1">`가 4265셀 중 364개로 **본문 행 전반에 흩어져** 있다(파서는 정상 반영).

cursor_row 전진 시 누적 대상 is_header 행이 계속 늘어 overhead가 단조 증가:
```
cursor 6   header_oh=20.9   avail_for_rows=988   (정상)
cursor 40  header_oh=589.5  avail_for_rows=419.6
cursor 76  header_oh=1350.4 avail_for_rows=0.0   ← 페이지 초과
```
`avail_for_rows=0` → 매 반복 `end_row=cursor+1`(1행)만 배치 → 173쪽.
(RHWP_TABLE_DRIFT 계측으로 확증. 상세: `output/poc/pi_page_hwpdocs/A_TYPE_DIAGNOSIS.md`)

### 관련 이력 (주의)
Task #1022가 "행 0만" → "cursor 아래 모든 is_header 행"으로 **의도적으로 확대**했다.
사유: 다중 머리행 표(pi=111 75x10, rs≥2 헤더)에서 행 0만 계산하면 본문 초과.
따라서 단순 롤백은 #1022 회귀를 부른다. **상단 연속 제목행 블록**만 반영하는 방식으로
#1022(연속 다중 머리행)와 #1716(흩어진 하위 header) 둘 다 만족시켜야 한다.

## 3. 목표

- 대표 파일이 한글과 정합(≈52쪽, ±수쪽)하도록 페이지당 1행 폭주 제거.
- Task #1022 회귀(다중 머리행 표) 없음.
- 표본/샘플 회귀 게이트 무회귀.

## 4. 수정 대상 (페이지네이터 ↔ 렌더러 동기 필수)

두 지점이 **동일 로직**(start_row/cursor_row 이전 모든 is_header 행 반복)이라 함께 고쳐야 desync(오버플로) 없음:

1. 페이지네이터: `src/renderer/typeset.rs` ~11320 `header_overhead`
2. 렌더러: `src/renderer/layout/table_partial.rs` ~166, ~308 `header_rows` 수집

## 5. 접근 방향 (구현계획서에서 세분)

핵심 정의: **반복 제목행 = 표 상단(행 0)부터 연속으로 제목행인 블록 `rows 0..H`**.
- `H` = 행 0부터 "그 행에 header 셀이 존재하는" 상태가 연속되는 최대 구간(중간에 끊기면 종료).
- 흩어진 하위 is_header 행(연속 블록 밖)은 반복/overhead에서 제외.
- 페이지네이터·렌더러가 **동일한 H 산출 함수**를 공유하도록 헬퍼로 단일화(재-desync 방지).
- 방어: overhead가 page_avail의 일정 비율(예: 50%)을 넘지 않도록 상한(안전망).
