# task_m100_2259 — rhwp-vscode 배율 메뉴 통합 및 맞춤(fit) 보기

- **이슈**: [#2259](https://github.com/edwardkim/rhwp/issues/2259)
- **마일스톤**: M100 (v1.0.0)
- **브랜치**: `local/task2259` (base: `upstream/devel` @ dbb2d9ffc)
- **범위**: `rhwp-vscode` 확장만. rhwp-studio / chrome / firefox / safari 는 제외.

## 1. 배경

작업지시자 요구사항:

1. 2쪽 보기 선택 시 화면에 맞춰 비율이 자동 조절되어야 한다.
2. 폭 맞춤, 쪽 맞춤(전체 보기) 옵션이 있어야 한다.
3. 이 옵션들을 배율(%)과 합쳐 하나의 메뉴로 정리한다.
4. 창 크기가 바뀌면 폭 맞춤 / 쪽 맞춤 비율도 창 크기에 따라 다시 계산되어야 한다.

## 2. 현황 분석

### 2.1 배율 상태

`src/webview/viewer.ts`

| 항목 | 현재 |
|------|------|
| 배율 상태 | `let currentZoom = 1.0` — 숫자 하나뿐, 맞춤(fit) 개념 없음 |
| 배율 변경 | `applyZoom(newZoom, anchorY?)` — 클램프(0.25~3.0) 후 레이아웃 재구성 |
| 진입점 | `stb-zoom-out` / `stb-zoom-in` 버튼, Ctrl+휠 (`ZOOM_STEP = 0.1`) |
| 표시 | `updateStatusBar()` 에서 `stb-zoom-val` 에 `${Math.round(currentZoom * 100)}%` |

### 2.2 보기 모드

| 항목 | 현재 |
|------|------|
| 상태 | `let viewMode: "single" \| "double"` |
| 전환 | `setViewMode()` — `stb-view-mode` 버튼 클릭 시 토글 |
| 레이아웃 | `buildPageLayout()` — `double` 이면 두 쪽씩 `.page-row` 로 좌우 배치 |
| **배율 연동** | **없음** — 2쪽으로 바꿔도 `currentZoom` 을 건드리지 않음 |

**→ 문제의 원인**: 기본 100% 에서 2쪽을 켜면 스프레드 가로 폭이 뷰포트를 초과해 가로 스크롤이 발생한다.

### 2.3 창 크기 변화 대응

현재 뷰포트 리사이즈를 관찰하는 코드가 **전혀 없다**. `scroll` 이벤트만 듣고 있다. 따라서 창을 넓혀도 배율은 그대로다.

### 2.4 레이아웃 여백 (맞춤 계산에 필요)

`src/hwp-editor-provider.ts` 인라인 CSS

| 요소 | 값 |
|------|-----|
| `#scroll-container` | `overflow: auto`, `padding: 12px 0` (세로만) |
| `#scroll-content` | `flex-direction: column`, `align-items: center`, `gap: 12px` |
| `.page-row` | `flex-direction: row`, `gap: 12px` |

가로 padding 이 0 이므로 맞춤 계산 시 좌우 숨쉴 여백을 별도 상수로 확보해야 한다.

### 2.5 상태 표시줄 마크업

`src/hwp-editor-provider.ts` `.stb-right` 내부:

```
[stb-view-mode "1쪽"] │ [stb-zoom-out −] [stb-zoom-val 100%] [stb-zoom-in +]
```

## 3. 설계

### 3.1 UI — 통합 배율 메뉴

`stb-view-mode` 버튼을 **제거**하고, `stb-zoom-val` 을 드롭다운 트리거 버튼으로 바꾼다.

```
[−]  [ 100% ▾ ]  [+]
          │
          ├─ 폭 맞춤
          ├─ 쪽 맞춤 (전체 보기)
          ├─ 두 쪽 맞춤
          ├──────────────────
          ├─ 50%
          ├─ 75%
          ├─ 100%
          ├─ 150%
          └─ 200%
```

- 현재 선택된 항목에 체크 표시(`✓`)를 둔다.
- 메뉴는 상태 표시줄 위쪽으로 펼친다(popup-above). 바깥 클릭 / `Esc` 로 닫는다.
- webview 안이므로 VS Code 네이티브 메뉴가 아닌 **자체 DOM 드롭다운**으로 구현한다.

### 3.2 상태 모델

`currentZoom` 단일 숫자를 다음으로 대체한다.

```ts
type ZoomMode = "manual" | "fitWidth" | "fitPage";

let zoomMode: ZoomMode = "manual";
let currentZoom = 1.0;            // 항상 "실제 적용 배율" (맞춤 모드에서도 계산된 값이 들어감)
let viewMode: "single" | "double" = "single";   // 기존 유지
```

`currentZoom` 은 계속 실제 배율을 담으므로 `makePageWrapper` / `renderPage` / 썸네일 등 **하위 렌더 경로는 수정이 필요 없다**.

### 3.3 메뉴 항목 → 상태 매핑

| 메뉴 항목 | `viewMode` | `zoomMode` |
|-----------|-----------|-----------|
| 폭 맞춤 | `single` | `fitWidth` |
| 쪽 맞춤 (전체 보기) | `single` | `fitPage` |
| 두 쪽 맞춤 | `double` | `fitPage` |
| 50 / 75 / 100 / 150 / 200% | **현재 값 유지** | `manual` |

- 맞춤 3항목이 곧 "쪽 배치 + 맞춤" 을 함께 정하는 유일한 진입점이다. 2쪽 → 1쪽 복귀는 `폭 맞춤` 또는 `쪽 맞춤` 선택으로 이루어진다.
- `%` 프리셋은 배치를 바꾸지 않는다. 즉 `두 쪽 맞춤` 상태에서 `100%` 를 고르면 2쪽 배치를 유지한 채 수동 100% 가 된다.
- `[−] / [+]` 버튼과 Ctrl+휠은 **`zoomMode` 를 `manual` 로 전환**하고 현재 배율에서 한 스텝 이동한다.

### 3.4 맞춤 배율 계산

```
availW = scrollContainer.clientWidth  − 2 × SIDE_MARGIN
availH = scrollContainer.clientHeight − 2 × 12(세로 padding)

pagesPerRow = (viewMode === "double") ? 2 : 1
contentW = maxPageWidth × pagesPerRow + ROW_GAP × (pagesPerRow − 1)
contentH = maxPageHeight

fitWidth = availW / contentW
fitPage  = min(fitWidth, availH / contentH)
```

- **기준 쪽 크기**: 문서 내 쪽 크기가 서로 다를 수 있으므로(가로/세로 혼합) **전체 쪽의 최대 폭·최대 높이**를 쓴다. 현재 보고 있는 쪽 기준으로 하면 스크롤 중에 배율이 요동치므로 채택하지 않는다.
- 계산 결과는 기존 `ZOOM_MIN`(0.25) ~ `ZOOM_MAX`(3.0) 로 클램프한다.
- `SIDE_MARGIN`, `ROW_GAP` 은 CSS 값과 일치하는 상수로 둔다(`ROW_GAP = 12`).

### 3.5 창 크기 변화 대응 (요구사항 4)

`scrollContainer` 에 `ResizeObserver` 를 붙인다.

- `zoomMode !== "manual"` 일 때만 배율을 다시 계산하고, 값이 바뀐 경우에만 레이아웃을 재구성한다.
- 재계산 시 **현재 쪽 위치를 유지**한다(`scrollToPage(currentPage)`).
- 리사이즈 폭풍 방지를 위해 `requestAnimationFrame` 으로 합친다.
- 사이드바 토글도 컨테이너 폭을 바꾸므로 같은 경로로 자동 처리된다(별도 훅 불필요).

### 3.6 세로 스크롤바 진동 위험

맞춤 배율을 키우면 세로 스크롤바가 나타나 `clientWidth` 가 줄고, 그러면 다시 배율이 줄어드는 진동이 생길 수 있다. 대응:

- `ResizeObserver` 의 `contentRect`(스크롤바 제외 폭)를 기준으로 계산한다.
- 계산된 배율이 직전 값과 **1% 미만 차이**면 무시한다(히스테리시스).

## 4. 검증

수동 검증 (F5 확장 개발 호스트):

1. 2쪽 보기(두 쪽 맞춤) 선택 → 가로 스크롤 없이 두 쪽 전체가 보인다.
2. 폭 맞춤 → 쪽 폭이 뷰포트를 꽉 채우고 세로 스크롤만 생긴다.
3. 쪽 맞춤 → 한 쪽 전체가 뷰포트 안에 들어온다.
4. 각 맞춤 모드에서 **에디터 패널 폭/높이를 바꾸면** 배율 표시(%)가 따라 바뀐다.
5. 사이드바 접기/펼치기 시에도 맞춤 배율이 재계산된다.
6. `[−]/[+]`, Ctrl+휠 사용 시 맞춤 모드가 해제되고 수동 배율로 바뀐다.
7. 세로 문서 / 가로 문서 / 쪽 크기가 섞인 문서에서 배율 진동이 없다.

빌드 검증: `npm run compile` (webpack, 타입 체크 포함) 무경고 통과.

> rhwp-vscode 에는 자동화 테스트 하네스가 없다. 따라서 본 타스크의 검증은 수동 확인 + 컴파일 통과로 한정한다.

## 5. 위험 및 완화

| 위험 | 완화 |

| 쪽 크기가 섞인 문서에서 배율 요동 | 최대 폭/높이 기준 고정 (3.4) |
