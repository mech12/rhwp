# 수행계획서 — #1994 절대위치 글뒤로 표 겹침 (church bulletin)

- 이슈: #1994 [Rendering] Text overlap on absolutely positioned textbox
- 브랜치: `fix/1994-behindtext-table-overlap` (origin/devel 기준)
- 재현: `20200830.hwp` (이슈 첨부 zip, `output/poc/issue1994/`), 참조 `issue_1994.pdf`(한글)
- 환경: 컨트리뷰터 재검증 완료(devel v0.7.17, 결함 재현). export-png/render-tree/reference PDF 대조.

## 1. 결함 (재현 확정)

가로(landscape) 2단 교회 주보 **3페이지**에서 두 콘텐츠 블록이 **같은 위치에 겹쳐** 판독 불가:
- "▶교육부 예배 / 유아부·유치부·어린이부·청소년부 예배 / ▶청년 공동체 …" (예배 스케줄)
- "목회자·전도사·집사 명단 (향린을 섬기는 이들)" (교역자 명단)

추가로 페이지 수 불일치(rhwp 5쪽 vs 한글 4쪽).

## 2. 근본원인 (조사 완료)

3페이지에 **글뒤로(wrap=BehindText) + 용지(Paper) 앵커 1×1 표** 2개:

| pi | 내용 | vert(Paper) | 쪽나눔 | rhwp 배치 |
|----|------|------------|--------|----------|
| 33 | 교역자 명단 | 용지 13.5mm | None | 절대 Y(상단) **정상** |
| 34 | 예배 스케줄 | 용지 **134.1mm** | **RowBreak** | 흐름 상단 + **2단 분할** → pi=33 위에 **겹침** |

- pi=33(쪽나눔 None)은 글뒤로 용지앵커로 절대 Y(13.5mm)에 정상 배치.
- **pi=34(쪽나눔 RowBreak)는 용지앵커(134.1mm)인데도 RowBreak 흐름 처리 경로를 타서** 절대 Y를
  무시하고 흐름 상단에 배치되고 2단으로 분할(dump-pages: 단0·단1 양쪽 PartialTable)된다 → pi=33 겹침.
- 즉 **RowBreak 속성이 Paper-앵커 글뒤로 표를 흐름(컬럼분할·flow Y) 처리시켜 절대위치를 무력화**하는 결함.
  (#2019 ⑤ = Paper-앵커 개체 절대배치 미준수 클래스와 동류이나, 여기선 BehindText+RowBreak 표로 국소.)

## 3. 수정 방향 (초안 — 구현계획서에서 확정)

- **Paper-앵커(vert=용지) + 글뒤로(BehindText) 표는 RowBreak 여부와 무관하게 절대 Paper Y 에 통째
  배치**하고 흐름 컬럼분할 대상에서 제외한다(pi=33 경로와 동일 취급).
- 수정 지점 후보: 표 배치 분기(typeset.rs / pagination에서 RowBreak 표 vs 절대앵커 표 판별). RowBreak
  분할 진입 전에 "글뒤로 + 용지앵커" 게이트로 절대배치 경로로 라우팅.
- 페이지 수(5→4) 영향은 겹침 해소 후 재확인(절대배치 시 흐름 소비 0 → 후속 콘텐츠 페이지 정합 개선 기대).
