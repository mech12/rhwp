# Task M100 #1516 계획: Firefox 공통 다운로드 관찰자 상태 머신 적용

## 배경

#1515에서 Chrome 다운로드 관찰자에 브라우저 API 비의존 상태 머신을 도입했다. Firefox 다운로드 관찰자는 아직 `seen`/`handled`/`workerStartedAt`을 파일 전역 메모리로 직접 관리하므로, Firefox background event page가 unload/reload되는 상황에서 추적 상태가 사라질 수 있다.

## 목표

- Firefox 다운로드 관찰자가 #1515의 `rhwp-shared/sw/download-observer-state.js`를 재사용한다.
- `browser.downloads.onCreated` / `browser.downloads.onChanged` 이벤트를 얇은 Firefox adapter로 연결한다.
- `browser.storage.session`이 있으면 다운로드 상태를 임시 저장하고, 없으면 메모리 fallback을 사용한다.
- 과거 다운로드 기록 재오픈, 같은 download id 중복 오픈, filename 확정 전후 재판정 누락을 테스트로 고정한다.

## 범위

### 포함

- `rhwp-firefox/sw/download-interceptor.js` 상태 추적 로직 교체
- Firefox용 `download-observer-state.js` symlink 추가
- Firefox 다운로드 인터셉터 테스트 추가
- Firefox build/check/test 검증

### 제외

- Chrome/Edge 변경
- Safari 다운로드 상태 머신 적용
- PR #1480 이전의 Chrome `onDeterminingFilename` 재도입

## 설계 원칙

- 상태 판단은 공통 상태 머신에 둔다.
- Firefox 파일에서는 `browser.*` 호출, 설정 조회, 뷰어 열기만 담당한다.
- HWP 판정은 기존 공유 `shouldInterceptDownload()`를 계속 사용한다.
- `autoOpen=false`일 때도 이미 판단한 download id는 handled 상태로 기록해 같은 항목 재이벤트가 탭을 열지 않게 한다.
