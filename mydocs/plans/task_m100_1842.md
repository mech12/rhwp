## 판별 결과

- 셀 내부 사다리는 **p[0](텍스트 없는 tac 묶음 전용 문단, 저장 lh=3401HU=45.3px)
  한 곳만** 저장 대비 −55.6px. p[6]/p[9] 내부표 host 라인은 저장 lh 사용 정상.
- 원인: `paragraph_layout` 의 `has_tac_shape` 라인높이 축소 분기("Shape 와
  텍스트의 baseline 정렬")가 **max_fs=0**(빈 문단 — 텍스트 없음)에서
  `raw_lh > 0*1.5` 항상 참 → `font_lh = 0*1.2 = 0` 퇴화. 저장 lh 소실 →
  후속 블록 전체 −33pt.
- 본문(sample16 pi=71, 같은 기하)은 reserved/skip-advance 보상 기계가 축소값을
  전제로 한컴 정합을 이미 이룸(issue_1116 한컴 핀) — 전역 가드는 그 핀을
  +10.4px(ls 1회분)로 깨뜨림 → **셀 컨텍스트 한정** 가드로 정제.
