## 1. 배경

C1a(#1453)~C1d(#2129, PR #2140)로 Track C의 렌더 커버리지가 25/27종까지 확보됐고,
**stock 2종(고가저가종가·시가고가저가종가)만 "차트 (미지원)" placeholder**로 남았다.
파서 plot 요소 매치에 `stockChart` arm이 없어 `chart_type=Unknown`으로 흘러
렌더러가 fallback 박스를 그린다. 값·카테고리는 이미 파싱되지만(공용 `c:val`/`c:cat`
핸들러) 계열의 역할(시/고/저/종) 구분과 HLC 표현이 없다.

여기에 C1c/C1d에서 C2로 이관된 2D fidelity 갭 4건을 함께 처리한다:

1. **범례 순서 역전** — C1c에서 "관찰 상충(누적세로·묶은가로 역순 vs 누적가로 정순)"으로
   이관(작업지시자 합의 2026-07-04). 이번 계획 수립에서 정답지 4장 재판독으로
   **규칙 해독 완료**(아래 §2) — 상충이 아니라 기하학적 규칙이었다.
2. **범례 스와치 마커 글리프** — C1d 시각판정(2026-07-10) 발견: 정답지 스와치는
   선+마커 글리프(—◆—), 현행은 선만.
3. **scatter 시리즈별 마커 모양** — 현행 전부 원(r=3), 정답지는 계열별 ◆■ 사이클.
4. **특이케이스 0.5 축 간격** — C1c v2에서 "1카테고리 미니차트 특수 동작, 섣부른 반영은
   앵커 파손 위험"으로 기록만 했던 항목. 좁은 게이트로 반영한다.

3D 입체·ofPie 보조플롯은 C2b(#2278)로 분리 — 본 작업의 범례 글리프 인프라가
C2b 범례에도 적용되므로 C2a 선행.

## 2. 사실 검증 (소스 수정 전 확인 완료, 2026-07-15)

| 항목 | 확인 결과 |
|---|---|
| stockChart 미인식 | `parser.rs:174-225` plot arm에 `stockChart` 부재 → catch-all로 무시, `chart_type=Unknown` 유지 → `renderer.rs:90-92` fallback. `mod.rs:20` 범위 외 명시 |
| stock XML 구조 | HLC=`<c:hiLowLines/>`만 / OHLC=`<c:hiLowLines/>`+`<c:upDownBars>`. 시/고/저 계열은 `<a:ln><a:noFill/>`+`<c:marker><c:symbol val="none"/>`, 종가만 marker 래퍼(symbol 부재=auto). **계열 역할 = XML 순서 규약**(3계열=고/저/종, 4계열=시/고/저/종, `c:order` 일치) |
| stock 정답지 실측 | HLC: 검정 세로 고저선 + 종가 회색 ▲(팔레트3 `#B0B0B0`·마커 사이클 ▲ 정합). OHLC: 캔들스틱(하락=진회색 채움/상승=흰+검정 테두리) + 종가 노란 ×(팔레트4 `#FCD801`). 범례: 시/고/저 빈 스와치(텍스트 정렬 유지), 종가만 글리프. 정순 |

| 마커 사이클 4번째 | `push_line_marker`(renderer.rs:749-803) — `si%4==3`은 원 폴백. OHLC 종가(4번째 계열) 정답지 실측은 **×** → 교체 필요. issue_2129 마커 12개 테스트는 3계열이라 무영향 확인 |

## 4. 범위

### 포함 (5단계 예정)

1. **마커 인프라 + scatter 마커**
   - `push_line_marker`에서 순수 경로 생성 `marker_path(si, cx, cy, r) -> (d, stroke기반)`
     헬퍼 추출(크기 파라미터화). `si%4==3` 원 폴백 → **×**(stroke 기반, fill 없음).
   - `render_scatter` 마커: circle → `marker_path` 사이클 적용(반경은 정답지 근사,
     시각판정에서 조정).
2. **stock 2종**
   - 모델(`mod.rs`): `OoxmlChartType::Stock`(label "주식형"), `OoxmlChart.has_hi_low_lines`/
     `has_up_down_bars`, `OoxmlSeries.marker_symbol: SeriesMarker{NotSpecified/None/Auto/Named}`.
   - 파서(`parser.rs`): `stockChart`/`hiLowLines`/`upDownBars` arm + handle_end plot arm,
     `marker` 핸들러에 계열 내부 게이트 추가(`cur_series.is_some()`→Auto) + `symbol` arm
     (`val="none"`→None, 그 외 Named). 기존 plot 레벨 line_markers 게이트 불변.
   - 렌더러(`renderer.rs`): 라우팅에 `Stock => render_stock` — 계열 수(3/4)로 역할 결정
     (그 외 계열 수는 `render_line` 폴백), x=슬롯 중앙, 축=`nice_axis_no_headroom + 1 step`,
     고저선(`hwp-stock-hilow`)/캔들(`hwp-stock-candle`, gapWidth 파싱 또는 150 고정 근사,
     하락 채움색은 픽셀 실측로 확정)/종가 마커(`marker_symbol != None`인 계열만).
   - `SeriesData` 합타입(C1b 예약) **도입하지 않음**: 순서 규약으로 충분, 합타입은
     파서·콤보 전반 파급(~200줄+) 대비 코퍼스 이득 없음.
3. **범례 순서**
   - **28 PDF 전수 실측표를 단계 선두에 작성**(단계보고서 수록) — percentStacked·3D 변형
     귀속 확정.
   - `legend_items`에 단일 결정 함수로 역순 규칙 적용(pie/scatter/stock/콤보/이중축=정순
     고정). `render_bars` 가로묶음 슬롯 내 배치 반전(`bar_w*(n-1-si)`) 동일 커밋.
4. **범례 스와치 글리프**
   - `legend_items` 반환에 `SwatchKind{Square/LineOnly/LineGlyph(si)/GlyphOnly(si)/Blank}`
     추가, `push_legend_swatch` 확장: LineGlyph=14px 선+중앙 글리프(—◆—),
     GlyphOnly=글리프만(scatter 표식만/stock 종가), Blank=스와치 없이 텍스트 오프셋 유지
     (stock 시/고/저).
   - 범례 글리프는 **별도 클래스 `hwp-legend-glyph`** — issue_2129의 `hwp-chart-marker`
     카운트 오염 방지.
5. **0.5축 게이트 + line3DChart 라우팅 + 종합 회귀**
   - `render_bars` 가로·1카테고리·비누적 → value step 절반(0~5 step 0.5, 소수 라벨은
     기존 비정수 step 가드가 처리). 단일 샘플 근거임을 주석+보고서에 명기.
   - `line3DChart` → Line + `is_3d=true` 방어 라우팅(인라인 XML 유닛테스트 포함, 입체는
     C2b·2D 근사 문서화).
   - 코퍼스 28종 export-svg 총괄 + 시각판정 요청.

## 5. 검증 기준

1. **파서 단위**: stockChart→`chart_type=Stock`+플래그 2종 / marker_symbol 3상태
   (none/auto/plot 레벨 무간섭) / line3DChart→Line+is_3d. 기존 파서 테스트 무회귀.
2. **렌더러 단위**: 마커 사이클 4번째 × 형상 / scatter 마커 사이클 / 범례 역순 규칙
   (역순 3케이스+정순 4케이스 모델 픽스처) / stock 축 (0,59)→0~80 step 20 /
   0.5축 게이트(1카테고리만 발동).
3. **회귀 통합** (`render_page0_svg` + stem×{hwpx,hwp} 패턴):
   - `issue_2277_stock.rs`: placeholder 부재 + `hwp-ooxml-chart` / 축 `>80<` /
     `hwp-stock-hilow` 4개 / OHLC `hwp-stock-candle` 4개(HLC 0개) / 종가 마커 4개 /
     범례 시/고/저 스와치 부재.
   - `issue_2277_legend_order.rs`: 역순 대표 4종 + 정순 대표 4종 — 범례 그룹 내
     `계열 3` y좌표 < `계열 1` y좌표 비교.
   - 기존 접점 무회귀: issue_2129 마커 12개(3계열 — × 무관)·무마커 stems /
     issue_1882 범례 x좌표(`width="10" height="10"` 유지)·3D 축 앵커·scatter 0.5 라벨 /
     issue_1453 no-fallback / issue_1431_scatter.
4. `cargo test` 전체 통과 + `cargo clippy --all-targets -- -D warnings` 무경고.
   포맷은 수정 파일만.
5. `export-svg` → `output/poc/chart_c2a/` 산출 → `pdf/chart/` 정답지 대조표 →
   **작업지시자 시각판정 통과** (Stage 3 전수 실측표 포함).
