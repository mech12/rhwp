## 원인 (계측 확정)

행높이 50.4 = Fixed(37.8) + em(12.6) — 두 결함의 합성:

1. **행미 공백 유령 줄**: 셀 재래핑(`recompose_for_cell_width`)이 trailing 공백
   포함 폭으로 분할 판정 → 공백만의 둘째 줄 생성 ("100.0␣␣" → ["100.0","␣␣"]).
   한글은 행미 공백을 hanging 처리(폭 판정 제외).
2. **aim=true 패딩 0 무시**: `Cell::use_cell_padding_axis`의 `cell_padding != 0`
   guard 가 pad=0 을 표 패딩(510HU)으로 폴백 → c0 내부폭 40.1→26.5px 축소 →
   8자리 코드가 2줄 분할. 한글 PDF 실측: 코드 렌더 폭 37.0px > 26.5px — 폴백이면
   물리적으로 1줄 불가, **한글은 0 을 존중** (resolve_cell_padding 주석과도 일치,
   guard 는 task1443 유래·근거 기록 없음).
