## 1. 결함 요약

- 부동 **글상자(텍스트박스)** 227개 + 사각형으로 구성된 별지 서식 문서가 rhwp에서 **81페이지**로 렌더(한글 2022: **18페이지**, 4.5배 과분할). 81쪽 중 35쪽이 near-empty. 서식 격자가 조각으로 깨져 렌더됨.
- 글상자 속성: `wrap=Through(어울림)`, `treat_as_char=false`, `vert_rel_to`=`Paper(용지)`/`Para(문단)`, 빈 문단(text_len=0)에 앵커.

## 2. 근본원인 (조사 완료, 코드 근거)

활성 엔진 = `TypesetEngine`(`typeset.rs`). 페이지 배치는 문단별 stored LINE_SEG vpos를 `HeightCursor::vpos_adjust`(`height_cursor.rs:156`)로 current_height에 스냅(전방 점프)한다.

- `vpos_adjust`는 **직전 문단이 "overlay 개체 문단"이면 vpos 보정을 bypass**한다(`height_cursor.rs:172-178`). 이유: 비-TAC 부동 개체를 소유한 문단의 stored vpos에는 개체 높이가 포함되어 과대하므로, 다음 항목의 보정 base로 쓰면 안 됨.
- 그 판정 함수 **`para_has_overlay_shape`(`layout.rs:866-890`)** 는
  ```
  InFrontOfText | BehindText   (모든 vert_rel_to)
  | (TopAndBottom && vert_rel_to==Para)
  ```
  **만** overlay로 인정한다. **`TextWrap::Through`(어울림)·Square·Tight 는 누락.**
- 따라서 Through-wrap 부동 글상자를 소유한 빈 앵커 문단이 overlay로 인식되지 못하고 일반 텍스트 문단처럼 취급 → **다음 앵커 문단의 팽창된 stored vpos(예: pi=91 vpos=191010)가 전방 점프로 current_height에 누적** → near-empty 페이지 대량 생성.
- Paper 개체는 절대 offset 배치이나 **소속 페이지는 앵커 문단 페이지를 따르므로**, 앵커 문단이 잘못된 페이지로 배정되면 서식 격자가 조각남(2차 연쇄).

**단일 근본 수정점**: `layout.rs:866-890 para_has_overlay_shape` 가 Through(및 Square/Tight) 비-TAC 부동 개체를 overlay로 인정하도록 확장.

## 3. 수정 방향 (설계 초안 — 구현계획서에서 확정)

- `para_has_overlay_shape` 의 overlay 판정에 `TextWrap::Through`(어울림) 비-TAC Shape/Picture 를 추가.
  - Square/Tight 포함 여부는 구현계획서 Stage에서 회귀영향 계측 후 결정(초안: Through 우선, Square/Tight는 별도 검증).
- **회귀 위험 높음**(hot layout 경로, `vpos_adjust` bypass는 광범위 문서에 영향). 따라서:
  - 게이트를 좁게: 앵커 문단이 부동 Through 개체를 소유하며 stored vpos가 팽창한 경우로 한정하는 추가 조건 검토.
  - #2004(이미지 스택)·#2015(RowBreak 표) 수정과 같은 vpos 경로를 공유하므로 상호작용 회귀 필수 확인.
