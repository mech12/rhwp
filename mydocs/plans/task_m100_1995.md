# 수행계획서 — #1995 RowBreak 표 다수 문서 과소 페이지 (rhwp 171 vs 한글 268, −97)

- 이슈: #1995 (M100)

- 작성일: 2026-07-06
- 상태: **수행계획 (승인 대기)** — 소스 미수정

## 1. 배경 / 문제

RowBreak 표가 많은 연구·보고서 문서에서 rhwp 가 한글보다 페이지 수가 크게 부족(과소 페이지).
5차 10k 서베이(#1996 보고서)에서 발견. 대표 케이스 −97쪽.

## 2. 재현 (한글 오라클 확인 완료)

- 파일: `hwpdocs/prism_downloads/국토교통부/1613000-202200006_…청년 주거정책…연구.hwp`
- **rhwp: 171쪽 / 한글(pyhwpx PageCount): 268쪽 (−97, −36%)** ← 오라클로 확정
- 대조군 `samples/2025 행정업무운영 편람(최종).hwp`: rhwp 393 vs 한컴 PDF(Hancom PDF) 383 (+10, **정상**)
  - 편람은 RowBreak 표 274개지만 대부분 다행 격자표. → **RowBreak 자체가 원인이 아님**.

## 3. 원인 진단 (render-tree 로 증명)

### 3-1. 구조 차이

| | 청년주거(결함) | 편람(정상) |
|---|---|---|
| RowBreak 표 | 162 | 274 |
| 그중 1×1 단일셀 | **115** | 36 |
| 표 형태 | 인라인 콜아웃 박스 위주 | 다행 격자표 위주 |

### 3-2. 결함 메커니즘 — 인라인 표 높이가 텍스트 흐름에 미예약

`export-render-tree` 로 확정:

- **박스 자체는 정상 높이로 렌더**됨 (예: p17 `<연구주제 1>` 박스 = 저장 셀높이 23932 HWPUNIT = 319px 정확, 셀 내부 줄 advance 정상).
- **그러나 박스 다음 본문 문단이 박스 위에 겹쳐 배치**됨 — p17 에서 표 bbox y=215.4..534.5 인데 후속 본문 12줄이 y=213..535 에 그려짐. 텍스트 흐름/페이지네이션이 표 높이만큼 전진하지 않음.
- 한글 오라클(p16)은 같은 박스의 공간을 정상 예약(겹침 없음).
- 심한 페이지(p34)는 **표끼리도 겹침**(표2 y=499..638, 표3 y=542..854) + 본문 21줄 겹침.

### 3-3. 결함 규모

- 171개 rhwp 페이지 중 **표-본문 겹침 22페이지**(표 있는 112페이지의 20%). 이슈의 "over-full 19/171" 증거와 정합.
- host 문단 저장 line_seg 는 표 높이를 인코딩(lh=24498)하고 있으나 rhwp 흐름이 이를 반영하지 않음.
- **순수 본문 페이지는 한글과 완벽히 정렬** — p14(서론 본문) 문단 간격/줄 pitch(53px) 오라클과 동일, 차이는 장제목 블록뿐. → −97쪽은 순수 텍스트 흐름이 아니라 **인라인 표/박스의 흐름 높이 미예약**에 거의 전적으로 기인. 인라인 박스는 각자 후속 콘텐츠를 자기 높이만큼 위로 끌어올려 문서 전체를 누적 압축한다.

> 주의: 초기 "격자표 150줄 압축" 관측은 넓은 다열표를 x축 무시로 오판한 것으로, p24 셀 내부는 advance 정상. 확정 결함은 **표-본문/표-표 겹침(흐름 높이 미예약)** 이다.

## 4. 수행 범위 / 접근

- **핵심 가설**: 인라인(또는 특정 앵커) 표의 전체 높이가 문단 흐름 line-advance 및 페이지네이션에 예약되지 않아, 후속 콘텐츠가 표 위로 겹치고 페이지가 압축된다.
- **1단계(정밀 원인 확정)에서** 겹침이 (a) `treat_as_char` 인라인 표의 host 문단 line-height 미반영인지, (b) `wrap=글앞으로`/`어울림` 앵커 표의 공간 미예약인지, (c) 둘의 조합인지 코드로 특정하고, 겹침 22페이지가 −97 중 몇 쪽을 설명하는지 정량한다(잔여분은 별도 추적).
- 수정은 `src/renderer/` 흐름/페이지네이션 계층 국소 수정. HWP3 전용 분기 금지 규칙 무관(공통 렌더러).
