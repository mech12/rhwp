# Task #1775 수행계획서 — CFB 스트림 경로 구분자 테스트 Windows 정합

## 배경
`issue_852_hwpx_to_hwp_contract_streams::form_01_keeps_nine_cfb_streams` 가 Windows 에서
실패 (이슈 #1775). cfb `walk()` 의 `entry.path()` 를 `to_string_lossy()` 하면 Windows 는
`\` 구분자(`/BodyText\Section0`)를 내는데 기대값은 `/` 하드코딩.

## 수정 (테스트 전용)
스트림 경로 수집 시 `\` → `/` 정규화. 프로덕션 코드 무변경.

## 게이트
- Windows 에서 해당 테스트 5/5 통과 (수정 전 4/5).
- 정규화는 비교 문자열에만 적용 — macOS/Linux 동작 불변(치환 무해).
