# task_m100_2004 수행계획서 — 전면 이미지 다수 앵커 문단 쪽당-1장 페이지네이션

- 이슈: #2004 (v1.0.0)
- 연계: #1994(글상자 겹침), #1995(과소 페이지 −97 군집), #1937/#1842(표 행높이, 별개)
- 상태: 수행계획 (승인 대기)

## 1. 배경 및 목표

한 문단에 **전면(near-full-page) 크기 이미지가 다수** 앵커된 경우(스캔 매뉴얼·별첨 이미지 묶음), 한글은 각 이미지를 **쪽당 1장**으로 배치해 N쪽을 사용하지만 rhwp는 **모두 앵커 쪽 한 곳에 겹쳐** 그려 N쪽→1쪽으로 붕괴한다.

- 페이지 손실 임팩트가 대형 문서에서 최대 **−97쪽**(오라클 확증). #1995 과소-페이지 군집의 **지배 원인**.
- 이미지가 시각적으로 겹쳐 판독 불가(#1994 글상자 겹침과 동일 뿌리).

**목표**: 전면 이미지 다수 앵커 문단을 한글과 정합하게 **쪽당 1장(또는 실제 배치 규칙대로)** 페이지네이션하여 페이지 수·시각 정합을 회복한다.

## 2. 근본원인 (오라클 확증, #1995 코멘트 4892513397/4892773035/4893305706)

두 변종이 동일 결과("전면 이미지 다수 → 쪽당 1장")를 낸다.

| 변종 | 개체 모드 | 대표 | 한글 배치 규칙 | rhwp 미구현 |
|---|---|---|---|---|
| **부동(floating)** | tac=false, wrap=Square(어울림), **overlap=false** | #1995 −97 (pi=1004 그림 96장, 182mm, 전부 offset0) | 겹침불허 부동개체를 직전 개체 **아래로 자동 재배치** → 쪽당 1장 캐스케이드 | 겹침해소 재배치 + 페이지 캐스케이드 |
| **인라인** | **tac=true**(글자처럼), Square | 1430000 −20 (pi=2607 그림 21장, 227mm 전면) | 전면 인라인 개체가 잔여 공간에 안 들어가면 다음 쪽 → 쪽당 1장 | 오버사이즈 인라인 개체의 줄 단위 페이지 분할 |

원본 CTRL_HEADER 계측으로 offset 진짜 0 확인(파서 유실 아님) → **레이아웃 규칙 미구현**.

## 3. 현황 코드 분석

- `allow_overlap`(bit14)은 `src/parser/control/shape.rs`에서 파싱되어 `CommonObjAttr.allow_overlap`에 저장되나, **레이아웃/페이지네이션 경로(`typeset.rs`/`pagination`/`layout`)에서 소비되지 않음** (사용처는 `document_core/commands/*` 편집 연산 한정). → 부동개체 겹침해소 규칙 부재 확정.
- 부동개체: `typeset.rs`의 `visible_float_exclusions`(줄 배치 시 부동개체 영역 회피) + `square_picture_wrap_anchor_for_para`/`record_wrap_around_para`(어울림 wrap 앵커)로 **텍스트가 부동개체를 피해 흐르는** 로직은 있으나, **다수 부동개체 상호 겹침을 해소해 페이지를 늘리는** 로직은 없음.
- 인라인개체: `layout/paragraph_layout.rs`의 `line_tac_picture_or_shape_height` 등이 tac 그림을 **줄 높이**로 반영. 단일 오버사이즈 그림은 처리되나 **한 문단 내 다수 오버사이즈 그림의 줄 단위 페이지 분할**은 검증 필요(−20 사례에서 미분할).

## 4. 접근 방안

변종별로 최소 침습·계측 우선 접근.

### 4-A. 인라인 변종 (선행 — 리스크 낮음)
- 한 문단 내 tac=true 그림들이 각각 "그림 줄(picture line)"로 측정되는지 확인. 다수일 때 각 그림 줄이 `paginate_text_lines`의 줄 단위 분할로 **잔여 공간 부족 시 다음 쪽**으로 넘어가도록 보장.
- 이미 대부분의 인프라(줄 높이=그림 높이, 줄 단위 페이지 분할)가 존재 → **줄 생성/분할 경로의 결함 국소화**가 핵심.

### 4-B. 부동 변종 (본체 — 리스크 높음)
- `allow_overlap=false` 부동개체 집합에 대해 **겹침 해소 배치(anti-overlap displacement)**: 같은 앵커·같은 위치를 선언한 개체를 문서순(z-order 또는 등장순)으로 직전 개체 하단으로 밀어 배치, 페이지 경계 초과 시 다음 쪽으로 캐스케이드.
- 배치 결과가 페이지 수(pagination)에 반영되도록 typeset 상태(`current_height`/페이지 진행)와 연동.
- **범위 한정 게이트**: "다수(≥2) + overlap=false + 전면급 크기 + 동일/근접 위치" 조건에서만 발동해 일반 부동개체 문서 회귀를 차단.

## 5. 단계 계획 (승인 후 구현계획서에서 세분)

1. **계측·재현 하네스**: 두 대표 문서(−97/−20)에 오라클 pi-page 기준선 고정 + 발동 조건 판별 계측(overlap/크기/개수/위치) 추가. (소스 계측 한정)
2. **인라인 변종(4-A)**: 다수 오버사이즈 tac 그림 줄 단위 페이지 분할 수정 + 1430000 −20 해소 검증.
3. **부동 변종 게이트·배치(4-B) 설계·구현**: anti-overlap displacement + 페이지 캐스케이드, 한정 게이트.
4. **−97 해소 검증**: pi-page 오라클 대조(268±), 시각(export-svg) 겹침 해소 확인.
5. **회귀검증**: `hwpx_roundtrip_baseline` 게이트 + 부동개체/이미지 다수 문서 표본 pi-page 무회귀 + 기존 wrap/float 테스트.
6. **최종 보고서 + 정리**.

## 6. 리스크 및 회귀검증 전략

- **hot layout 회귀**: 부동개체·wrap·인라인 이미지 경로는 광범위. → (a) 발동 조건을 좁은 게이트로 한정, (b) 단계별 pi-page 오라클 표본 대조(발동/비발동 양쪽), (c) `hwpx_roundtrip_baseline` 통과 유지.
- **오라클 의존**: 검증에 한글 2022 오라클 필요(현재 복구·워밍업 상태). 재현 스크립트 `scratchpad/hwp_paramap.py`(rmtree 가드) 재사용.
- **부분 해소 허용**: 부동 변종 게이트가 보수적이면 일부 문서는 미해소로 남을 수 있음 — 회귀 0을 우선하고 커버리지는 후속 확장.
