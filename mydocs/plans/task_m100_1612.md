## 2. 발견 (조사 grounding)

### 발견 1 — 메트릭 다페이지 혼동 (진단 버그)
`compute_hwp_used_height`(rendering.rs:3700)는 마지막 항목의 **stored vpos 누적 위치**를 반환.
vpos 는 다페이지에서 누적값(36398709 page2 vpos=66568~127587 HU)이라 **per-page `used` 와
비교 시 혼동** → `diff` 가 페이지마다 ~800px 누적 증가. 페이지 시작 vpos 를 차감하면 정상
per-page 높이(872/814/736/771/626px) 도출 → **base 차감으로 정정 가능**.

### 발견 2 — 잔여 12건 실제 특성
| 군 | 건수 | 특성 |
|----|------|------|
| 단일페이지 footer | 8 | used 959~985px, footer(Page+Bottom) vpos fit, 한글 2쪽. 마지막 item=Table → 현재 메트릭 None |
| 다페이지 | 4 | 메트릭 혼동으로 과대 표시(실제 per-page 차 수십px) |

8건은 본문 누적 ~20~43px 부족(footer 가 본문 직후 fit). per-line 은 저장 LINE_SEG(한글 동일) →
inter-paragraph gap/spacing 미세차 = Task #1600 하드코어. **단일 surgical fix 부재 가능성 높음.**
