# Task M100 #1444 수행계획서 — 확장 viewer CSP 인라인 스크립트 + 다크 아이콘 404

- 이슈: #1444 "확장 viewer CSP 인라인 스크립트 차단 + 다크 아이콘 404 (다크테마 확장 회귀, 0.2.5 배포 차단)"
- 마일스톤: M100 (v1.0.0)
- 브랜치: `local/task1444` (`local/devel` 분기)
- 작성일: 2026-06-20

## 1. 현상 (확장 0.2.5 배포 테스트 로그)

크롬 확장 0.2.5 viewer 에서 두 오류. 웹앱(rhwp-studio)은 정상, **확장에서만** 깨짐
(자기 검증 ≠ 확장 환경). PR #1420(다크테마)이 확장 빌드·CSP 경로를 고려하지 않은 회귀.

### ① CSP 인라인 스크립트 차단
```
viewer.html:8 Executing inline script violates ... 'script-src 'self' 'wasm-unsafe-eval''
... The action has been blocked.
```
`rhwp-studio/index.html` `<head>` 최상단의 다크테마 FOUC 방지 인라인 `<script>`(테마 모드
즉시 적용: localStorage rhwp-settings → data-theme-* / colorScheme)가 빌드 시 viewer.html
에 인라인된 채 남는다. 확장 manifest CSP `extension_pages: "script-src 'self'
'wasm-unsafe-eval'"` 는 `unsafe-inline` 이 없어 인라인을 차단 → 테마 초기화 미실행.

### ② 다크 아이콘 404
```
icon_small_ko_dark.svg:1  Failed to load resource: net::ERR_FILE_NOT_FOUND
```
`base.css:197` 이 다크 모드에서 `/images/icon_small_ko_dark.svg` 참조. 확장 빌드
`rhwp-chrome/build.mjs:94` 는 `icon_small_ko.svg`(라이트)만 복사 → 다크 아이콘 누락.

## 2. 근본 원인

- ① 인라인 스크립트는 FOUC 방지를 위해 `<head>` 최상단 동기 실행이 필요해 인라인됐으나,
  확장 CSP 가 인라인을 금지. 웹앱은 CSP 가 느슨해 통과.
- ② build.mjs 가 다크 아이콘 복사 라인을 추가하지 않음 (라이트만 복사).

## 3. 수정 방향 (구현계획서에서 확정)

- ① 인라인 테마 스크립트를 **외부 파일**(예: `rhwp-studio/public/theme-init.js`)로 분리하고
  `index.html` `<head>` 같은 위치에 `<script src="/theme-init.js"></script>`(동기, module
  아님)로 로드 → CSP `'self'` 충족. vite 가 public/ 을 그대로 복사하고 확장 build 가 viewer
  자산을 복사하므로 양쪽 동작. FOUC 방지(번들·스타일 전 실행) 유지.
- ② `build.mjs` 에 `icon_small_ko_dark.svg` 복사 추가(라이트와 동형).

## 4. 목표와 완료 조건

1. 확장 viewer 에서 CSP 인라인 위반·다크 아이콘 404 미발생.
2. 다크/라이트 테마 전환 + FOUC 방지 동작 (웹앱·확장 공통).
3. rhwp-studio 웹앱 회귀 0.
4. chrome/edge/firefox 확장 공통 정상.
5. 0.2.5 zip 재생성.

## 5. 단계 개요 (3단계 예정 — 구현계획서에서 확정)

1. 인라인 테마 스크립트 외부 분리(theme-init.js) + index.html 참조 + build.mjs 다크 아이콘 복사.
2. 확장 재빌드 + viewer CSP/404 미발생 확인 + 웹앱 회귀 0 + e2e(theme-mode).
3. 0.2.5 zip 재생성 + 문서·보고서.
