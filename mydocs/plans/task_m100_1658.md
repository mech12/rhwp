## 설계 (하단 예약)

- out-of-flow(높이 0) ✗ — 1개 문서 −1 폭증(검증됨). 문서순 flow ✗ — 2개 문서 +1.
- **정답**: 해당 표 높이를 **페이지 하단에서 예약**(available 축소) + 위치는 layout 하단 배치.
  - 1개: 소비 높이 = flow 와 동일 → 무회귀.
  - 2개+: 하단에 함께 스택 → 1쪽 수용 → over 해소.

## 구현

- `TypesetState` 에 `bottom_anchored_reserve: f64` 추가(페이지별, 새 페이지/단 전환 시 0 리셋).
- `available_height()` 에서 `bottom_anchored_reserve` 차감.
- `typeset_table_paragraph` 표 dispatch 에서 페이지 하단 고정 표 감지 시:
  `bottom_anchored_reserve += 표 높이(+host_spacing)` + `PageItem::Table` push + `continue`
  (typeset_block_table flow 누적 회피). 단단·비-oversized 한정.
- 감지 조건: `!treat_as_char && TopAndBottom && VertRelTo::Page && VertAlign::Bottom`.
