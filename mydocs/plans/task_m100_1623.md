# Task M100 #1623 수행 계획서

## 이슈

- GitHub Issue: #1623
- 제목: rhwp-studio 셀 테두리/배경 모달 UI를 한컴 기준으로 정합하고 탭 전환 시 크기 고정
- 기준 브랜치: `upstream/devel`
- 작업 브랜치: `task_m100_1623`

## 문제 요약

`rhwp-studio`의 `셀 테두리/배경` 대화상자는 한컴 2024의 대화상자와 다음 점에서 다르다.

- 탭 전환 시 모달 외곽 크기가 안정적으로 고정되지 않는다.
- `대각선` 탭이 한컴처럼 `\ 대각선`, `/ 대각선`, `+ 중심선`의 다중 아이콘 그룹과 미리 보기를 제공하지 않는다.
- 중심선 UI가 `BorderFill.center_line`, HWPX `centerLine`, HWP5 `BORDER_FILL` 중심선 비트와 충분히 연결되어 있지 않다.

작업지시자가 추가한 샘플 기준으로는 UI뿐 아니라 렌더링도 함께 다뤄야 한다.

- `samples/대각선샘플.hwp`
- `samples/대각선샘플.hwpx`
- `pdf/대각선샘플-2024.pdf`

Hancom 2024 PDF와 현재 rhwp SVG 렌더링을 비교하면, HWP/HWPX에는 `centerLine`과 `cellzone` 정보가 존재하지만 rhwp 출력은 일부 대각선/중심선을 한컴처럼 표시하지 못한다.

## 현재 분석

### 샘플 데이터

HWPX `Contents/header.xml`에는 다음 BorderFill 정보가 들어 있다.

- `borderFill id=10`: `centerLine="VERTICAL"`, 대각선 선 속성 `SOLID`, `5.0 mm`, `#0000FF`
- `borderFill id=11`: `slash=CENTER`, `backSlash=CENTER`, 대각선 선 속성 `THICK_SLIM`, `3.0 mm`, `#000000`

HWPX `Contents/section0.xml`의 표에는 다음 `cellzoneList`가 들어 있다.

- `row 1..2, col 0..1, borderFillIDRef=9`
- `row 2..2, col 2..3, borderFillIDRef=11`

`rhwp dump` 기준으로 HWP/HWPX 모두 위 cellzone과 cell borderFill 참조가 IR에 보존된다. 따라서 1차 원인은 파서 데이터 손실이 아니라 렌더링/편집 API 적용 경로에 있다.

### 렌더링 경로

현재 `src/renderer/layout/table_layout.rs`는 cellzone 배경은 zone 전체 영역에 한 번 렌더링하지만, cellzone의 대각선/중심선은 렌더링하지 않는다.

개별 셀 대각선은 `render_cell_diagonal`을 통해 셀 `border_fill_id` 기준으로만 그려진다. 이 때문에 Hancom 2024에서 zone 단위로 보이는 굵은 X와 중심선이 rhwp 출력에서 누락된다.

또한 `src/renderer/layout/border_rendering.rs`는 HWP5 `BORDER_FILL.attr`의 slash/backSlash 방향 비트는 처리하지만, HWPX에서 들어온 `Crooked`, `isCounter` 조합 및 zone 단위 bounding box 적용은 아직 충분히 반영하지 못한다.

### UI/API 경로

`rhwp-studio/src/ui/cell-border-bg-dialog.ts`는 대각선 탭을 이미 갖고 있지만, 현재는 `\`, `/`, `+` 단일 버튼 수준이다.

`src/document_core/commands/table_ops.rs`의 `getCellProperties`/`setCellProperties` JSON은 테두리 4방향과 배경 중심이며, 대각선/중심선 필드가 없다. `create_border_fill_from_json`도 새 BorderFill을 만들 때 `diagonal`과 `center_line`을 기본값으로 초기화한다.

## 작업 범위

1. 샘플 기준 렌더링 결함을 회귀 테스트로 고정한다.
2. 표 렌더러가 `cellzone`의 대각선/중심선을 zone bounding box 기준으로 그리도록 보강한다.
3. `render_cell_diagonal`이 Hancom 2024 샘플의 slash/backSlash, 중심선, 굵은 대각선 조합을 더 정확히 처리하도록 보강한다.
4. `getCellProperties`/`setCellProperties` JSON에 대각선/중심선 필드를 추가한다.
5. rhwp-studio `셀 테두리/배경` 모달의 대각선 탭을 한컴 2024 구조에 가깝게 정렬하고, 탭 전환 시 외곽 크기를 고정한다.
6. WASM 빌드와 studio E2E/시각 확인 산출물을 준비한다.

## 제외 범위

- 한컴 2024 UI의 모든 아이콘/패턴을 픽셀 단위로 완전 복제하지 않는다.
- 저장 파일에 없는 대각선 의미를 추정해서 새 스펙으로 확장하지 않는다.
- Hancom 2024 시각 판단은 작업지시자 확인을 최종 권위로 둔다.

## 검증 계획

### 로컬 구조 검증

- `target/release/rhwp dump samples/대각선샘플.hwpx`
- `target/release/rhwp dump samples/대각선샘플.hwp`
- `target/release/rhwp export-svg samples/대각선샘플.hwpx -o output/poc/issue1623_diagonal_sample/after_hwpx`
- `target/release/rhwp export-svg samples/대각선샘플.hwp -o output/poc/issue1623_diagonal_sample/after_hwp`

### 이미지 비교 준비

- `pdftoppm -png -r 96 -singlefile pdf/대각선샘플-2024.pdf output/poc/issue1623_diagonal_sample/pdf/hancom2024`
- `rsvg-convert`로 SVG를 PNG 변환 후 Hancom 2024 PDF 렌더와 비교

### Rust 검증

- 관련 단위 테스트 추가 후 focused test
- 변경 범위 확정 후 `cargo fmt --check`
- 변경 범위가 렌더링 공통 경로라면 `cargo test --release --lib`
- 통합 테스트가 필요하면 `cargo test --profile release-test --tests`

### WASM/studio 검증

- `wasm-pack build --target web --out-dir pkg`
- `cd rhwp-studio && npm run build`
- 필요 시 셀 테두리/배경 모달 E2E 또는 Playwright 스크린샷 확인
