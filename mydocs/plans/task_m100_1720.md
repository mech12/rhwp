# Task #1720 수행계획서 — 개체 단위 시각/geometry 회귀 인프라

## 배경
#1718 잔여 under-pagination(rhwp 42 vs 한글 48, −6쪽)은 후반부 중첩표·그림 개체의 누적 배치
차이로 규명됐으나, 개체별 정밀 정렬·회귀 검출 수단이 없다(현재 page/PI 레벨
`verify_pi_page_vs_hangul.py` 만 존재). 개체(표·그림) 단위로 rhwp vs 한글(OLE)을 대조하고
rhwp 버전 간 개체 회귀를 검출하는 인프라가 필요하다.

## 목표
`tools/object_visual_regression.py`:
1. rhwp 개체 geometry — `export-render-tree` 의 depth≥1 중첩 개체(Table pi/ci/rows/cols/bbox,
   1×1=그림 프레임).
2. 한글 권위 렌더 — COM→PDF→PyMuPDF: 96 DPI 페이지 래스터 + 이미지 bbox + `find_tables` 표 bbox.
3. rhwp 래스터 — `export-png`(native-skia).
4. 개체 매칭(크기기반) + 개체별 side-by-side 크롭 HTML 갤러리 + geometry TSV.
5. baseline 저장/비교 — rhwp 버전 간 개체 이동/리사이즈 회귀 검출(CI 게이트).

## 검증 게이트
- 대표 파일(승강기 [별표27]) 전 파이프라인 동작(render-tree 42p, export-png 42p, 한글 48p).
- baseline 회귀: 동일 baseline → 0건(오탐 없음), 변형 → 정확 검출.
- 좌표계 정합: render-tree/export-png/한글 래스터 모두 96 DPI px.
