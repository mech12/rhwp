# Task M100 #1634 수행 계획서 — 선택 셀 Transpose

## 목표

선택한 표 셀 범위의 내용을 행/열 전치 형태로 다른 위치에 채우는 기능을 구현한다.

작업지시자가 제공한 참고 그림은 스프레드시트의 `TRANSPOSE(B2:C5)`처럼 원본 범위는 유지하고,
대상 시작 셀부터 `행×열`을 `열×행`으로 바꿔 채우는 사용 흐름을 가리킨다. 따라서 1차 범위는
동적 수식이 아니라 정적 전치 복사/붙여넣기로 정의한다.

## 범위

- F5 셀 선택 범위를 전치 원본으로 사용한다.
- 전치 명령을 실행하면 내부 transpose 버퍼에 선택 범위의 셀 내용을 복사한다.
- 이후 대상 시작 셀에서 붙여넣기 명령을 실행하면 `r,c -> c,r` 매핑으로 셀 내용을 채운다.
- 원본 셀은 유지한다.
- 셀 내용은 `paragraphs` 단위로 복사한다.
- 대상 범위가 표 크기를 벗어나면 실패 처리한다.
- Ctrl 제외 셀이 있는 비직사각형 선택은 1차 범위에서 제외한다.
- 병합 셀이 포함된 범위는 1차 범위에서 제외한다.
- 중첩 표 경로 기반 전치는 1차 범위에서 제외한다.
- `={TRANSPOSE(...)}` 같은 동적 수식 참조는 1차 범위에서 제외한다.

## 작업 순서

1. Rust 표 모델에 전치 가능성 검증과 셀 내용 복사/붙여넣기 로직을 추가한다.
2. `DocumentCore` native API와 WASM 바인딩을 추가한다.
3. `WasmBridge`와 Studio 표 명령에 `table:transpose-copy`, `table:transpose-paste`를 연결한다.
4. F5 셀 선택 모드에서 명령 실행 가능 조건을 추가한다.
5. Rust 단위 테스트와 Studio 타입/빌드 검증을 수행한다.

## 검증 계획

- `cargo test -p rhwp model::table::tests::...` 범위의 신규 단위 테스트
- 필요 시 `cargo test -p rhwp table_transpose` 또는 해당 테스트명 직접 실행
- `npm run build` 또는 TypeScript 검증
- `git diff --check`

## 리스크

- 셀 `paragraphs`에는 텍스트 외 컨트롤, 필드, 그림이 들어갈 수 있으므로 단순 텍스트 복사는 기능 목표에 부족하다.
  1차에서는 문단 구조를 그대로 clone한다.
- 병합 셀은 전치 후 span 축이 바뀌어야 하므로 구조 변경 범위가 커진다. 1차에서는 명시적으로 거부한다.
- 비정사각형 전치 붙여넣기는 원본 선택보다 다른 크기의 대상 범위를 덮어쓴다. 대상 범위가 표 안에 있을 때만 실행한다.
