# Task #1567: HWPX 표 셀 pic 드롭 해소 — 수행계획서

## 목표
HWPX 저장 시 표 셀 내 `<hp:pic>` 가 드롭되는 결함을 해소한다(roundtrip IR diff
`tbl.cell.p[0] controls: expected=[pic] actual=[]`). 실문서 최다 IR_DIFF 클래스
(fidelity v4 1844건 중 907건).

## 근본원인 (수행계획 선행 조사)
1. 원본 셀 pic 다수가 `<hc:img binaryItemIDRef="">`(빈 값, 이미지 참조 없는 placeholder).
   파서가 `bin_data_id=0` 으로 매핑. (실증 36385464: hp:pic 2개 중 하나가 빈 ref)
2. `write_img`(`src/serializer/hwpx/picture.rs:256`) 가 `resolve_bin_id(0)→None` 으로
   `SerializeError` 반환.
3. 호출자 `section.rs:701`(및 Shape::Picture 1173)가 에러를 **로그만 찍고 pic 드롭**
   → SERIALIZE_FAIL 아닌 IR_DIFF(소실).

## 범위
**포함(In)**
- 빈/미해결 `binaryItemIDRef` pic 보존: `bin_data_id==0`(빈 ref) 시 `binaryItemIDRef=""`
  를 verbatim 방출(에러·드롭 금지). 비-0 미해결은 종전대로 진단.
- `section.rs:701`·`1173` 의 조용한 드롭 방지(보존 경로 확립).
- 표 셀 pic 드롭 roundtrip diff 감소 검증.

**제외(Out)**
- char_shape 8유닛 시프트(F3 #1556) 직접 수정 — 단 pic 슬롯 보존의 후속 효과는 측정.
- HWP5·secCnt·시각 픽셀 diff.

## 검사·판정 기준
- 대표 케이스(36385464·36388571 등) roundtrip IR diff 의 `expected=[pic] actual=[]` 해소.
- opengov 고정 말뭉치(#1564) 스냅샷: 해당 파일 status 개선 → 스냅샷 승격(갱신).
- `hwpx_roundtrip_baseline`(samples/hwpx) 회귀 없음.
- fidelity 재측정: 표셀 pic 드롭 건수 감소 정량화.

## 위험/주의
- `bin_data_id==0` ⟺ 빈 ref 가정 확인(HWPX bin id 는 1-based, 0=무참조) — Stage 1 검증.
- 비-0 미해결(진짜 누락 BinDataContent)은 보존이 아니라 진단 유지(데이터 손실 은폐 방지).
- HWP3 분기 금지. HWPX serializer/parser 범위.
