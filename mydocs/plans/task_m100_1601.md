# Task M100 #1601 수행 계획서

## 개요

- 이슈: #1601
- 주제: `mydocs/**` 전용 PR fast-pass 확장
- 기준 브랜치: `upstream/devel`

## 문제

PR #1580에서 `mydocs/pr/**`와 `mydocs/orders/*.md` 전용 PR은 preflight 기반 fast-pass로 heavy CI를
skip할 수 있게 보강했다. 그러나 `mydocs/manual/**` 같은 운영 문서 변경은 아직 allowlist에 포함되지 않아,
코드 실행 결과에 영향이 없는 문서 PR도 `Build & Test`와 CodeQL heavy job을 다시 실행할 수 있다.

## 목표

- `mydocs/**`만 변경한 PR을 문서 전용 변경으로 판단한다.
- CI, CodeQL, Render Diff에서 preflight는 유지하고 heavy job만 skip한다.
- `render-diff.yml`도 `mydocs/**` 변경 시 preflight check가 나타나도록 trigger path를 맞춘다.
- PR 리뷰 워크플로우 문서에 `mydocs/**` fast-pass 기준과 한영 병기 코멘트 규칙을 기록한다.

## 범위

- 수정 대상:
  - `.github/workflows/ci.yml`
  - `.github/workflows/codeql.yml`
  - `.github/workflows/render-diff.yml`
  - `mydocs/manual/pr_review_workflow.md`
- 문서 대상:
  - `mydocs/orders/20260628.md`
  - `mydocs/plans/task_m100_1601.md`
  - `mydocs/plans/task_m100_1601_impl.md`

## 검증 계획

- `actionlint .github/workflows/ci.yml .github/workflows/codeql.yml .github/workflows/render-diff.yml`
- `git diff --check`
- 변경 범위가 workflow와 운영 문서에 한정되는지 확인
