# Task #1831 수행계획서 — 다쪽 표 continuation 페이지 상수 오프셋 (v2, 원인 재규명)

## 배경

hwpdocs 5,200 표본 조사(`mydocs/report/survey_pipage_20260703.md`)에서 페이지·PI 는
MATCH 인데 continuation 페이지 전체가 30~54pt 위로 이동하는 위치 결함 확인.

## v1 진단 폐기 (중요)

v1(오전)은 "한글이 p2 상단에 반복 제목행을 렌더하는데 rhwp 가 누락 → repeat_header
폴백(제목셀 전무 시 row0 반복)"으로 진단했으나, **판정 기준으로 쓴 r2.pdf 가 한글
산출물이 아니라 rhwp 자체 출력**(PDF producer=`rhwp`)이었음을 확인했다. 진짜 한글
2022 정답지 2종(신규 COM 생성본 + survey `h_10.pdf`, 둘 다 producer=Hancom PDF)은
서로 정확히 일치하며 **반복 제목행이 없다**:

- 한글 p1: 제목문단 + 표1(11행)에서 끝남 (끝줄 y=631pt).
- 한글 p2: **표2(12행) 전체가 p2 에서 시작** (머리행 y=99pt) — p2 첫 줄
  "평가항목/평가지표"는 반복이 아니라 표2 자신의 row0.
- rhwp: 표2의 row0 만 p1 하단에 조각으로 남기고(rows=0..1, cont=false) p2 는
  rows=1..12 로 시작 → p2 전체가 한글보다 위로 밀림(−54.3pt)과 머리행 누락의 정체.

### 인과 실험으로 확정한 사실 (재현물 `output/poc/task1831/`)

1. **한글의 표2 전체 밀기는 제목 줄 반복과 무관**: COM 으로 두 표 RepeatHeader=0
   저장(`2448877_norepeat.hwp`, attr 0x06000002 확인) 후에도 동일하게 전체 밀기.
2. **repeat_header 폴백은 한글과 불일치**: 18151945 별표7(306행, repeat_header=true,
   제목셀 전무)의 한글 PDF 39쪽 전수 스캔 — row0(캡션)·머리행(구분/분야) 모두
   p1 에만 출현, continuation 38쪽 전부 반복 없음. 행 경계에서 시작하는 페이지
   (p4/p8/p24 등)에도 없음 → 제목셀 플래그 없이는 반복하지 않는다.
3. **제목셀 있는 표의 반복은 기존 경로가 정합**: rowbreak-problem-pages 25×7 표
   (row0~1 제목셀 hdr=true)는 한글 PDF(pdf/rowbreak-problem-pages-2024.pdf)에서
   p3·4·5 반복 ✓ — rhwp 기존 has_header_cells 경로와 일치.
4. 셀 LIST_HEADER bytes6-7 상위 바이트(b7)에 제목셀 유사 비트(0x0400/0x0100)가
   코퍼스에 대량 존재하나(452 셋 5,639셀), 비트 클리어 플립 실험에서 한글 배치
   불변 — 이 건과 무관 (b6 기반 기존 is_header 파싱이 옳음).
   부산물: HWP5 압축 스트림 꼬리 보존 셀 플래그 패처(`tools/patch_cell_flags.py`).

## 수정 (v2 — 규칙 실측 확정 후 구현)

1. **WIP 폐기 완료**: `leading_header_rows()` 폴백(model/table.rs) + typeset 게이트
   제거(renderer/typeset.rs 미커밋분) 되돌림. dump `hdr=` 진단(main.rs)은 유지.
2. **같은 문단 float 스택 이월 규칙** (renderer/typeset.rs 분할 진입부):
   **같은 문단에 앵커된 선행 float(표) 아래로 밀린** 다행 자리차지 표가 단 잔여에
   통째로 안 들어가면 첫 조각을 만들지 않고 표 전체를 다음 단/쪽으로 민다.
   임계 규칙 실측 확정 과정 (모두 한글 COM PDF 인과 실험):
   - 최소 2행 아님: row1 축소로 2행(35.9mm)이 들어가도 밀기.
   - 새쪽수용성 아님: 행 10개 추가로 표>1쪽이어도 밀기(다음 쪽 상단에서 분할).
   - 잔여 공간 아님: 표1 삭제+텍스트 필러 스윕(잔여 133~880px) 전 구간 분할.
   - 같은 문단 한정: 36387040 pi=46은 위에 다른 문단 표 3개가 있어도 분할.
3. **전체-배치 2px 허용**: 이월 후 새 단 상단에서 표 전체가 ≤2px 오차로만 넘치면
   통째 배치(행높이 측정 드리프트 흡수, 2448877 p2 = 941.1px 가용 vs 942.9px).
   전체가 들어갈 때만 적용되어 분할 경계 산정에는 무영향.
4. 3114781(한방 검토부표 p2 −34pt)은 한글 정답지 재생성 후 재판정(오전 비교본
   r3114781.pdf 도 rhwp 산출물). 이슈 존속 항목으로 이관.
