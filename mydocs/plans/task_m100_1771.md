# Task #1771 수행계획서 — 중첩 그룹 직렬화 경계 계약 수정 (복합 벡터 대량 소실)

## 배경
복합 벡터(중첩 Group) 문서가 HWP5 직렬화→재로드 후 대량 소실 (이슈 #1771).
대표 3067999: IR children 710→12, 렌더 Path 335→0.

## 원인 (레코드 카운트 + 코드 대조로 확정)
- 직렬화 바이트에는 전 레코드 존재(SHAPE_COMPONENT 503, CONTAINER 217) — 소실은 재파스.
- 파서(parse_container_children)의 자식 경계 = **SHAPE_COMPONENT @ child_level** 이고,
  중첩 그룹은 "경계 SHAPE_COMPONENT + 더 깊은 SHAPE_COMPONENT 존재 시 재귀" 계약.
- 직렬화기(serialize_group_child)는 중첩 그룹을 **CONTAINER(0x56) 레코드만**으로 방출
  (경계 SHAPE_COMPONENT 없음) → 파서가 자식 경계로 인식 못 함 → 하위 전체가 이전
  자식 slice 로 흡수/소실.

## 수정
`serializer/control.rs`:
1. 최상위 그룹의 '$con' SHAPE_COMPONENT 데이터(자식수 u16 + 자식 ctrl_id 목록 + instance_id)
   구성을 helper 로 추출.
2. 중첩 그룹 방출을 CONTAINER 레코드 → **SHAPE_COMPONENT('$con', comp_level)** + 손자
   (comp_level+1/+2 재귀)로 교체 — 파서·한컴 계약 정합.
