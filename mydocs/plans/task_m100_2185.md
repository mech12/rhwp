# 수행 계획서 — Task M100 #2185

## 2. 배경과 확정 근거

#2185의 한컴 통제 실측은 `attr1 bit7=1`이 글자 단위 채움, `bit7=0`이 어절
유지로 동작함을 보여 준다. 현재 공통 라인브레이커는 이 의미를 반대로 해석한다.

115쪽 거대 셀 샘플의 대상 문단은 `section[0]/paragraph[0]/control[2]/cell[2]/paragraph[5]`이며,
문단 끝에 `1`을 입력할 때 저장 `LINE_SEG.text_start`가 다음처럼 바뀐다.

```text
입력 전: [0, 44, 84, 122]
입력 후: [0, 43, 78, 113]
```

현재 구현에서 글자 단위 분기를 강제로 선택한 통제 대조에서는 동일 입력 후에도
`[0, 44, 84, 122]`가 유지됐다. 폰트·셀 폭·측정 엔진은 동일했으므로, 이 재현의 직접
원인은 폰트 매트릭스가 아니라 줄 나눔 단위의 의미 반전이다. git 이력상 `0da18bbc`에서
어절/글자 분기가 뒤집힌 정황도 확인됐다.

## 3. 목표

1. 공통 라인브레이커의 한글 줄 나눔 계약을 한컴 통제 실측과 일치시킨다.
2. 기존 HWP/HWPX 저장값과 roundtrip 계약을 불필요하게 변경하지 않는다.
3. 재현 샘플의 첫 입력에서 저장된 앞선 줄 경계와 후속 문단 위치가 유지되도록 한다.
4. 광역 줄바꿈 변경에 따른 기존 문서의 시각·페이지네이션 회귀가 없음을 검증한다.

## 4. 수행 범위

### Stage 1. 의미 계약과 영향 범위 고정

1. `korean_break_unit`의 모든 생산·소비 지점을 점검해 HWP/HWPX 파서, 직렬화기,
   문단 모양 UI, 스타일 해석기, 토크나이저와 줄 채움 로직의 계약표를 작성한다.
2. HWPX의 `KEEP_WORD`/`BREAK_WORD` 명칭보다 #2185의 한컴 통제 실측을 권위로 삼고,
   저장값을 바꾸지 않은 채 어느 소비 분기를 정정할지 확정한다.
3. `0da18bbc` 전후 동작과 현재 단위 테스트가 반대 의미를 고정한 범위를 확인한다.

### Stage 2. 회귀 핀과 최소 수정 범위 설계

1. bit7=1 글자 단위와 bit7=0 어절 단위를 각각 직접 검증하는 composer 테스트를 정한다.
2. 거대 셀 샘플 끝에 `1`을 입력한 뒤 `LINE_SEG` 시작점, 줄 수, 후속 문단 `vpos`,
   페이지 수를 검증하는 통합 회귀 테스트를 설계한다.
3. HWP/HWPX 파서·직렬화 값은 유지하고 공통 토큰화·줄 채움 의미만 고치는 최소 변경안을
   구현 계획서에 구체화한다.

### Stage 3. 광역 회귀 게이트 정의

1. #2169의 kbu/kbu2 통제 자료와 관련 장문 줄바꿈 코퍼스를 표적 게이트로 사용한다.
2. 기존 composer, HWP/HWPX roundtrip, #937 PUA 및 셀 reflow 회귀 테스트를 포함한다.
3. 공통 라인브레이커 변경이므로 공식 PDF pin, 359 recount와 사용 가능한 대량 서베이에서
   `REGRESSED=0`을 요구한다.
4. 전체 Rust 테스트와 clippy를 최종 코드 게이트로 둔다.

### Stage 4. 결과 정리와 후속 분리

1. 정확성 수정 결과와 대량 게이트 결과를 단계별 완료보고서 및 최종 보고서에 남긴다.
2. 재생성 `LineSeg`의 `column_start`·줄별 tag 보존은 별도 정확성 후속으로 분리한다.
3. 캐럿 page-tree, Canvas 재도장과 전체 pagination 지연은 별도 성능 이슈 후보로 분리한다.

## 5. 완료 조건

- bit7=1은 글자 단위, bit7=0은 어절 단위로 동작한다.
- 재현 샘플 끝에 `1` 입력 후 네 줄의 시작점 `[0, 44, 84, 122]`가 유지된다.
- 마지막 줄만 `하여 적용한다.1`로 바뀌며 후속 문단 `vpos`와 115쪽이 유지된다.
- HWP/HWPX 저장값 및 roundtrip 계약에 의도하지 않은 변화가 없다.
- 표적·전체·대량 회귀 게이트에서 새 회귀가 없다.

## 6. 비목표

- 한컴 줄바꿈·페이지 분할 엔진 전체 복제
- 폰트 메트릭과 글리프 렌더링 정합 개선
- 변경 줄 이후만 계산하는 증분 `LINE_SEG` reflow
- 기존 줄별 `column_start`·tag 보존 방식 변경
- 거대 셀 입력 성능과 전체 pagination 최적화
