## 1. 배경

C1a(#1453) Part B에서 막대 누적(`render_bars`의 stacked/percentStacked)을 구현할 때,
`render_line`도 `c:grouping`을 인식하지 않는 동일 결함이 발견되어 C1d로 분리 등록됐다
(#1431 트래킹). 현재 **누적꺽은선형·표식이있는누적꺽은선형·백프로기준누적꺽은선형(3종)이
독립 선으로 오렌더**된다. 파서는 line plot의 `c:grouping`을 의도적으로 보류 중이며
(`parser.rs:246` "line의 grouping은 C1d 후속" 주석), 고정 테스트
`test_parse_grouping_line_ignored`(parser.rs:710)가 현행 무시 동작을 단언하고 있다.

추가로 정답지 실측에서 **표식(마커) 갭**이 확인됐다: `render_line`은 마커를 아예 그리지
않아, 누적만 구현하면 표식이있는누적꺽은선형이 누적꺽은선형과 동일하게 렌더된다.
**작업지시자 승인(2026-07-09)으로 마커 렌더를 본 작업 범위에 편입** — 비누적
표식이있는꺽은선형도 같은 플래그로 함께 정합된다.

→ 이슈의 현상·원인이 모두 사실로 확인됨. 데이터 추출은 이미 완전하고(값·카테고리·색),
**grouping/marker 채택 + render_line 누적 기하 + 마커 렌더**만 추가하면 된다.

## 3. 목표

`render_line`이 `c:grouping`(stacked/percentStacked)과 plot 레벨 `c:marker`를 인식하여
라인 5종(꺽은선형·표식이있는꺽은선형·누적꺽은선형·표식이있는누적꺽은선형·
백프로기준누적꺽은선형)을 정답지와 정합하게 렌더한다. C1a 막대 누적과 대칭 구현.

## 4. 범위

### 포함

1. **모델** (`mod.rs`): `OoxmlChart`에 `line_grouping: BarGrouping` + `line_markers: bool`
   추가. `BarGrouping` enum 이름 유지, doc만 "막대/라인 공용"으로 갱신.
2. **파서** (`parser.rs`):
   - `b"grouping"` arm: 매핑 1회 계산 후 `cur_plot_type`에 따라 Column/Bar→`grouping`,
     Line→`line_grouping` 분기.
   - `b"marker"` arm 신설: 게이트 = `cur_plot_type==Some(Line) && cur_series.is_none()
     && val 속성 존재` (계열 내부 marker는 val 없어 자연 배제 — 신규 ParseState 불필요).
3. **렌더러** (`renderer.rs`) — `render_line` 단일 함수 내 분기(render_bars 관용구):
   - 값공간 누적 `cum[i] += v.max(0.0)` 후 선형 매핑. percent는 `cum/카테고리합*100`
     (합≤0→0.0 가드).
   - 축: stacked→`nice_axis(0, max category_positive_sum, VERTICAL_AXIS_TICKS)` /
     percent→(0,100,20) 고정 + `render_value_grid` percent 플래그.
   - path 생성을 `points: Vec<(f64,f64)>` 수집 → `polyline_path` 재사용으로 정리
     (마커 좌표에도 필요).
   - 마커 헬퍼 `push_line_marker(svg, si, cx, cy, color)`: `si % 4`로 ◆(r≈3.5)/■/▲/원
     폴백, `<path class="hwp-chart-marker" fill={색} stroke="#fff">`. 정답지 실측 사이클.

### 제외 (Non-goals)

- `render_combo` 누적/마커 — 누적 라인 콤보는 코퍼스에 없음. 콤보의 line은 현행 유지.
- `line3DChart` 라우팅 — 코퍼스 27종에 없음. 미라우팅 갭으로 이슈에 기록만.
- `c:smooth`(코퍼스 전부 0)·계열별 `c:symbol`/`c:size` 세부 fidelity — C2.
- 범례 순서 역전(누적에서 계열3이 위) — C1c에서 관찰 상충으로 **C2 기이관**, 불변.
- `estimate_axis_label_width` 개선 — C1a percent 막대와 동일 한계로 기수용.
- x 배치(양끝 엣지 고정)·`max_len<2` 조기 반환 — 기존 시각판정 통과 동작, 불변.

### 변경 면적

파서(grouping 분기 + marker arm) + 모델(필드 2개) + 렌더러(`render_line`·마커 헬퍼).
`render_bars`·`render_combo`·`render_scatter`·축 헬퍼 무변경.

## 5. 검증 기준

1. **파서 단위 테스트**: line stacked→`line_grouping==Stacked`이면서 `grouping==Clustered`
   불변 / percentStacked 매핑 / 콤보(bar stacked + line standard) 상호 무오염 /
   plot 레벨 marker val=1→true·0/부재→false / 계열 내부 marker 무시.
   기존 `test_parse_grouping_line_ignored`는 신규 동작 단언으로 반전.
2. **렌더러 기하 단위 테스트** (`line_chart(grouping)` 헬퍼 신설 — `bars_chart` 미러,
   3계열×4카테고리 합 12.3 데이터): 누적 축 `>15<` 존재·`>14<` 부재 / 상위 계열 y <
   하위 계열 y / percent `100%`·`20%` 라벨 / 최상위 계열 수평선 / 합 0 카테고리 NaN 부재 /
   마커 개수=계열×점·형상 사이클·기본값 무마커.
3. **회귀 통합 테스트** (`tests/issue_2129_line_stacked.rs`, issue_1882 패턴): 라인 5종
   stem × hwp/hwpx **10파일** — 누적→`>15<`+무마커 / 백프로→`100%` / 표식누적→마커+`>15<` /
   표식(비누적)→마커+개별값 축 `>6<` / 꺽은선형→무마커+`>6<` (무회귀 핀).
4. 기존 차트 테스트 회귀 없음. `cargo test` 전체 통과 + `cargo clippy --all-targets
   -- -D warnings` 무경고. 포맷은 수정 파일만.
5. `export-svg` → `output/poc/chart_c1d/` 산출 → `pdf/chart/라인/` 정답지 대조표 →
   **작업지시자 시각판정 통과**.
