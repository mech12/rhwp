## 배경
#1760 행 분해의 후속 정밀 조사(#1759 razor-thin 계열). 당초 "대형 폰트 줄간격" 재분류를
셀 저장 LINE_SEG 대조로 재정정: 줄 간격은 전부 정확 일치, 실체는 **셀 콘텐츠 높이 측정이
마지막 줄 trailing line_spacing 을 포함**해 선언 행높이를 초과 확장 (이슈 #1763).

## 정량 (2501937 12×16 TAC 표 row0, 5문단 셀)

  (= 한글 find_tables 142.1px).
- rhwp: content 141.5px = 스팬 133.5px + **trailing ls 8px(600HU)** → required 149.1px
  > 선언 142.2px → 행 확장 +7px. 관찰된 상하 7.8px 는 확장 행의 valign=Center 슬랙.

## 원인 코드
`src/renderer/height_measurer.rs` 측정 closure: `include_trailing_ls = !is_cell_last_line
|| cell_para_count > 1` — **다문단 셀은 셀 마지막 줄 trailing ls 포함** (#874/#1086 계열
보존 조건). 주석(825행)의 "cell.height 는 pad+content, trailing ls 미포함" 원칙과 충돌.

## 수정 방향 (보수적 — 기존 보존 케이스 무회귀)
측정값 자체는 유지하되 **선언높이 권위 가드** 추가: trailing 제외 콘텐츠+pad 가 선언
cell.height 안이면 확장하지 않음(required = 선언). trailing 포함이 필요한 기존 케이스
(aift.hwp pi=123, KTX TOC — 콘텐츠가 선언을 실제 초과)는 가드 조건 미충족으로 불변.
