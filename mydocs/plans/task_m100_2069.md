# Task M100 #2069 수행 계획서

## 목표

`samples/한셀OLE.hwp`의 한셀 OLE 개체가 한컴처럼 선택되고, 우클릭으로 `개체 속성(P)...`에 들어가며, 빈 문단의 캐럿이 OLE 오른쪽 가장자리에 놓이도록 보정한다.

## 현재 진단

- 샘플은 `text_len=0` 빈 문단에 비-TAC `ShapeObject::Ole`가 들어 있다.
- OLE 미리보기는 `RawSvg`로 렌더되지만, `RawSvg` 노드에 문서 좌표 메타가 없어 `getPageControlLayout()`에 선택 가능한 개체로 노출되지 않는다.
- `getCursorRect(0,0,0)`은 TextRun/메타가 있는 개체 bbox를 찾지 못해 `CursorState updateRect` 경고를 낸다.

## 구현 계획

1. OLE `RawSvg`/placeholder 렌더 노드에 `section_index`, `para_index`, `control_index` 메타를 보존한다.
2. `getPageControlLayout()`에서 메타가 있는 OLE `RawSvg`를 개체 레이아웃으로 방출한다.
3. `rhwp-studio`에서 OLE를 기존 shape 계열 선택/속성 경로에 연결한다.
4. 빈 문단 + OLE 단독 문단에서 `getCursorRect()`가 OLE 오른쪽 가장자리 rect를 반환하도록 한다.
5. `samples/한셀OLE.hwp`를 7700 앱에 열어 클릭, 우클릭, 속성 대화상자, console warning을 검증한다.

## 검증 계획

- `target/release-test/rhwp info/dump/dump-pages/export-render-tree samples/한셀OLE.hwp`
- focused Rust test 또는 기존 cursor/layout query 테스트
- `wasm-pack build --target web --out-dir pkg`
- `rhwp-studio` 기존 7700 서버에서 Puppeteer 검증
- 필요 시 `rhwp-studio npm test`의 관련 테스트 추가/수행

## 리스크

- `RawSvg`는 OLE chart, EMF preview, native image preview, placeholder 모두에서 쓰이므로 메타가 있는 OLE 렌더 노드에만 선택 메타를 제한해야 한다.
- OLE 전용 속성 UI가 아직 없으므로 이번 단계는 한컴 호환의 최소 사용자 동작인 공통 개체 속성 진입을 기존 shape 속성 대화상자로 재사용한다.
