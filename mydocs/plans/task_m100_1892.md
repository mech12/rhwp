## 증상

대법원 행정예규 서식 hwp(실체는 **HWP3 V3.00**) 계열에서 `render-diff --via hwp`
(HWP3 파스 → `export_hwp_with_adapter` → 재파스) 렌더 위치가 페이지 내 최대
5449px 갈라짐. 대표 2912309 (제적등본2 서식, 2쪽).

## 조사 계획

1. 대표 2912309 재현: `hwp5-roundtrip` (IR_DIFF=1) + `render-diff --via hwp` (5449px STRUCT)
2. 파일 시그니처 확인 → HWP3 여부 판별
3. `dump` / `ir-diff` / `export-render-tree` 로 A(원본 파스) vs B(재파스) IR·렌더 트리 대조
4. 결함 레코드/코드 경로 특정 → 최소 수정
5. 대법원 15건 전수 재측정 + 회귀 게이트 (cargo test, big_hwp/big_hwpx 코퍼스 A/B)

## 사전 판별 (조사 결과 반영)

- 이슈의 IR_DIFF=1 단서는 `hwp5-roundtrip`(어댑터 미경유 plain serialize) 경로의
  SectionPageDef 소실로, 실제 제품 경로(`export_hwp_with_adapter`)와 **다른 축**
  (#1893 함정 노트 (1) 동형). 렌더 변위의 진짜 축은 render-diff 재현으로 특정한다.
- 렌더 변위 축 = ① 빈 묶음('$con' children=0) 재파스 사각형 오분류(Group→Rect 구조
  분기) ② 직렬화기 rendering matrix 폴백의 offset 승격(자식 최대 10485px 이동)
  ③ HWP3 `build_common_obj_attr` 크기 기준 비트 누락(Paper 퍼센트 해석 → 종이비례
  팽창) ④ 탭 "데이터 없음" 마커의 IR 유입(탭 무폭화) — 4중 결함 체인.
