# 수행계획서 — Task #2102

## 제목
쪽 배경 이미지 채우기(page_border_fill)가 모든 쪽에 그려지는 문제 — 구역 첫 쪽에만 적용

## 배경 / 증상
구역의 쪽 테두리/배경(`page_border_fill`)에 **그림(이미지) 채우기** 배경이 설정된 HWP5 문서에서,
표지(구역 첫 쪽)용으로 의도된 배경 이미지가 **2쪽 이후 모든 쪽에 반복 렌더링**된다.
이미지에 상단 가로줄 등 표지 장식이 포함된 경우, 2쪽 이후 본문 텍스트를 관통하는 선이 그려진다.

- 한컴 편집기 실측(작업지시자 확인): 배경 이미지는 **구역 첫 쪽에만** 표시, 2~N쪽 없음.
- rhwp 현재: 1쪽 포함 전 쪽에 배경 이미지 표시.

## 근본 원인 (조사 완료)
- `src/document_core/queries/rendering.rs`: 모든 쪽에 대해 `section_def.page_border_fill`(record[0]=BOTH)을 그대로 사용.
- `src/renderer/layout.rs` `build_page_background`: 페이지 적용 범위(첫 쪽/parity)를 고려하지 않고 배경 노드 생성.
- 문서 구조: 바탕쪽(master page) 없음, BIN0001을 참조하는 쪽-앵커 그림 없음 → 배경은 오로지 `page_border_fill` record[0]의 이미지 채우기(`FitToSize`, `fillArea=종이 전면`).

## 수정 방향 (승인됨)
- 페이지 배경의 **이미지 채우기**는 **구역 첫 쪽에만** 적용.
- 단색/그라데이션 채우기, 쪽 테두리선(border), 쪽번호 배치는 **현행 유지** (회귀 방지).

## 한계 / 리스크 (정직 고지)
- HWP5 파일에서 "첫 쪽 전용"을 인코딩하는 필드는 발견하지 못함. PAGE_BORDER_FILL 3-레코드는 스펙상 `[BOTH,EVEN,ODD]`이고 record[0](BOTH)=이미지이므로 스펙상 "모든 쪽"이 정석 해석.
- 따라서 본 수정은 **한컴 실측을 맞추는 규칙**이며 "이미지 page-fill=첫 쪽"이 범용 한컴 규칙인지는 미확정. → 골든/시각 회귀 검증 필수.
- 저장소 골든/샘플에 "모든 쪽 이미지 배경" 사례는 확인되지 않음 → 회귀 표면 제한적.
