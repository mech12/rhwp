- **브랜치**: `local/task2285` (from `local/devel`)

## 1. 배경 / 목표

rhwp-studio 파일 메뉴에 **"최근 문서" 서브메뉴**를 추가한다. 최근 열었던 문서 목록(최대 8개, 최신 우선)을 표시하고, 항목 클릭 시 재열기한다. 재열기는 IndexedDB에 저장된 `FileSystemFileHandle`을 사용하며, 접근 권한을 재확인한 뒤 문서를 로드한다.

## 2. 범위

| 포함 | 제외 |
|------|------|
| 파일 메뉴 하위 "최근 문서 ▶" 서브메뉴 (최대 8개) | 경로 툴팁 (브라우저가 절대경로 미제공 → 취소) |
| 최근 목록 저장/조회 — IndexedDB에 핸들 + 메타(파일명/형식/시각) | 드롭·파일선택(input)으로 연 핸들 없는 파일 (재열기 불가 → 목록 제외) |
| 클릭 시 권한 재확인 후 재열기 | 상태 표시줄 변경 |
| "최근 문서 목록 지우기" 항목 | 파서/렌더러 로직 |
| 목록이 비었을 때 비활성 처리 | HWP3 전용 로직 |

## 3. 제약 (조사로 확정)

- **절대경로 불가**: 브라우저 File System Access API는 파일명(`handle.name`)만 제공하고 로컬 절대경로를 노출하지 않는다. → 경로 툴팁 기능은 취소(작업지시자 결정).
- **핸들 있는 파일만 재열기 가능**: `FileSystemFileHandle`은 IndexedDB에 structured-clone으로 저장 가능하며, 재접근 시 `queryPermission`/`requestPermission`로 권한 재확인이 필요하다. 드롭·`input[type=file]`로 연 파일은 핸들이 없어 재열기가 불가능하므로 **목록에 기록하지 않는다**(작업지시자 결정).

## 4. 현행 구조 (조사 결과)

- **파일 메뉴**: `index.html:25` `<div class="md-item" data-cmd="file:open">열기</div>`. 서브메뉴는 `.md-sub` + `.md-sub-panel` 패턴(예: `index.html:68~82` 배율).
- **메뉴 동작**: `src/ui/menu-bar.ts:70~87` `setupItemClicks()` — `.md-item[data-cmd]` 클릭 시 `data-cmd`를 읽어 dispatch, `data-*` 속성이 params로 전달. `updateMenuStates()`(133~158)는 드롭다운 open 시 활성/비활성 갱신 → 동적 목록 갱신 훅 지점.
- **파일 열기**: `src/command/commands/file.ts:313~349` `file:open` → `pickOpenFileHandle()` → `readFileFromHandle()` → `open-document-bytes` 이벤트(`fileHandle` 포함).
- **로드 수렴점**: `src/main.ts:902~924` `loadBytes(data, fileName, fileHandle, ...)` — 모든 로드 경로가 여기로 수렴. `open-document-bytes` 핸들러(999~)도 내부에서 `loadBytes` 호출.
- **영속 저장 선례**: `src/recovery/autosave-store.ts`(IndexedDB), `src/ui/symbols-dialog.ts:320~333`(recent 패턴 — 맨 앞 삽입/중복 제거/상한).

## 5. 설계 요지

1. **저장소** — 신규 `src/recent/recent-store.ts`
   - IndexedDB DB `rhwpStudioRecent`, objectStore `recent`.
   - 레코드: `{ id, fileName, sourceFormat, openedAt, handle }`.
   - API: `addRecent(entry)`, `listRecent()`(최신순), `clearRecent()`, `removeRecent(id)`.
   - 중복 제거: `handle.isSameEntry()` 우선, 실패 시 파일명 기준. 최대 8개 유지.
   - autosave-store.ts 패턴 준용, IndexedDB 미지원 환경은 no-op fallback.

2. **기록 지점** — `main.ts:loadBytes()`
   - `fileHandle`이 존재할 때만 `addRecent()` 호출. 핸들 없으면 기록 안 함.

3. **메뉴 UI** — `index.html` + `menu-bar.ts`
   - `index.html` 파일 메뉴에 `.md-sub` "최근 문서" 추가(열기 아래, 구분선 부근).
   - 서브패널은 메뉴 open 시점에 `menu-bar.ts`에서 `listRecent()`로 동적 렌더. 항목 `data-cmd="file:open-recent" data-id="..."`. 하단에 구분선 + "목록 지우기"(`file:clear-recent`).
   - 목록 비면 "(최근 문서 없음)" 비활성 항목.

4. **커맨드** — `src/command/commands/file.ts`
   - `file:open-recent`: `data-id`로 레코드 조회 → `queryPermission`→(필요 시)`requestPermission` → `getFile()` → `open-document-bytes` emit. 실패(이동/삭제/권한 거부) 시 토스트 안내 + 해당 항목 `removeRecent`.
   - `file:clear-recent`: 확인 후 `clearRecent()`.

## 6. 구현 단계 (구현계획서에서 상세화)

- **S1**: `recent-store.ts` — IndexedDB CRUD + 정렬/중복/상한 + fallback
- **S2**: `loadBytes` 기록 훅 + `index.html` 서브메뉴 마크업 + `menu-bar.ts` 동적 렌더
- **S3**: `file:open-recent` / `file:clear-recent` 커맨드 + 권한 재확인·실패 처리
- **S4**: 빌드(tsc+vite) · 개발서버 수동 검증 · 회귀 확인

## 7. 검증

- `npm run build`(tsc 타입체크 통과) + `vite` 개발서버에서:
  - 문서 열기 → "최근 문서"에 등재 확인
  - 재시작(새로고침) 후 목록 유지 확인
  - 항목 클릭 → 권한 재확인 → 재열기 확인
  - 파일 이동/삭제 후 클릭 → 실패 안내 + 목록 제거 확인
  - 드롭/파일선택으로 연 파일은 목록에 안 뜸 확인
  - "목록 지우기" 동작
- 기존 e2e 무영향 확인.
