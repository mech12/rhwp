## 목표

rhwp-vscode 뷰어에 1쪽/2쪽 보기 토글을 추가한다. 2쪽 모드는 두 페이지를 좌우로 나란히 배치한다.

## 구현

- 상태 표시줄 우측에 `#stb-view-mode` 버튼(1쪽/2쪽 토글) 추가.
- `viewer.ts`:
  - `viewMode: "single" | "double"` 상태.
  - `buildPageLayout()`를 모드별 분기: double 시 두 쪽씩 `.page-row`(flex-row)로 묶어 배치. `makePageWrapper(i)` 헬퍼로 페이지 래퍼 생성 공통화.
  - `setViewMode()`: 모드 전환 시 레이아웃 재구성 + 현재 페이지 위치 유지(`scrollToPage`).
- CSS `.page-row { display:flex; flex-direction:row; gap:12px; }`.

## 설계 포인트

- `pageInfos[i].element`는 여전히 각 페이지 래퍼를 가리켜, 가상 스크롤(`updateVisiblePages`)·현재 페이지 추적·`scrollToPage`·줌 로직이 그대로 동작(래퍼가 `.page-row` 안에 있어도 getBoundingClientRect 유효).
- 페어링은 0&1, 2&3… 단순 방식(표지 단독 옵션은 향후).

## 검증

- `npm run compile`(webpack) 통과.
- 1쪽/2쪽 토글, 2쪽 배치, 모드 전환 시 현재 페이지 유지, 줌/스크롤 회귀 없음 (작업지시자 시각 판정).
