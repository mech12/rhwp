## 1. 계측으로 확정된 두 트랙 (2026-07-09, RHWP_TABLE_DRIFT)

### 트랙 B — 59043 인트라셀 컷 진동 (bounded, 우선)
- 종전 "0-footprint near-empty" 판단은 **오해**: pi=160 조각들은 실제 946~958px 소비.
- **진짜 결함**: 컷 진동 — `consumed 946.4 → 22.4 → 946.4 → 22.4` 교대. 컷 좌표(end_cut): 셀A 유닛 `36→78→79→122→165→190`(+42,+1,+43,+43,+25), 셀B `39→85→86→132→133→179`(+46,+1,+46,+1,+46).
- **기전**: 3×3 표의 두 대형 셀(법령 원문 104·77문단) 병렬 인트라셀 컷에서, fragment 경계가 한 셀의 한계로 결정될 때 **다른 셀이 1유닛만 전진하는 sliver fragment**(22px)가 페이지 하나를 통째로 소비. pi=160 rows1..3에 6 fragment(한글은 ~4 추정). 유사 표(pi=98/124/153) 합산 시 +11의 상당분.
- **수정 목표**: sliver fragment(소비 ≪ 페이지, 컷 전진 극소)를 직전/직후 fragment에 병합하거나, 병렬 셀 컷 경계를 동기화. `scan_block_table_split_rows` 내부.

### 트랙 A — 156714340 부동 표 콘텐츠 미측정 (deep)
- pi=42(1×1 RowBreak 부동 표, 셀에 전면 Square 이미지 5장)는 **분할 스캔 미진입**(TABLE_SPLIT_RESULT 없음): 측정 `mt.total_height`=저장 871px가 body에 fit → `PageItem::Table` 원자 배치, 이미지 5장 겹침.
- **수정 목표**: (1) 측정이 셀 내 전면 부동 이미지 스택 높이(≈5×880px)를 반영 → 분할 머시너리 진입, (2) 인트라셀 컷이 이미지들을 fragment별로 분배(스택 셀문단의 유닛화 — 재분류 N분할 + 컷 유닛 정합), (3) 렌더가 fragment별 이미지 1장씩 배치.
- 선행 실증: 측정만(1차 시도) → +1 spurious, 재분류만(3차 시도) → 불변. **(1)+(2) 결합이 필요**.

## 2. 단계
1. **Stage 1**: 기준선 고정(156714340=4/한글8, 59043=48/한글37, 통합브랜치 2946/0) + sliver 진단 계측(RHWP_TABLE_DRIFT 확장: fragment별 컷 전진량).
2. **Stage 2 (트랙 B)**: sliver fragment 병합/동기화 — `scan_block_table_split_rows` 수정. 검증: 59043 48→감소(한글 37 방향), 조각 콘텐츠 무손실(PDF 텍스트 보존), 회귀(2946 + 2500 A/B).
3. **Stage 3 (트랙 A)**: 부동 RowBreak 표 측정에 셀 부동 이미지 스택 반영 + 스택 유닛화 결합. 검증: 156714340 4→8(±) + 이미지 쪽당 1장 시각.
4. **Stage 4**: 통합 회귀 — 전체 테스트 + 2500 A/B + 부동표 문서군.
5. **Stage 5**: 보고서·PR.

## 3. 리스크
- `scan_block_table_split_rows`는 RowBreak 분할 SSOT(#1025/#1022/#993 계층) — sliver 병합이 정상 분할 경계를 옮기면 광범 회귀. **컷 전진 극소(≤2유닛) ∧ 소비 극소(≤40px) ∧ RowBreak** 3중 게이트로 스코프.
- 트랙 A는 측정·컷·렌더 3경로 연동이라 단계 내 미완이면 정직하게 중간 산출(측정 진입까지)로 경계 설정.
