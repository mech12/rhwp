## 현상

첨부 HWPX를 rhwp로 SVG 렌더링하면 1페이지 제목 영역의 작은 이미지가 누락된다.

원본 HWPX 구조상 누락 이미지는 다음 위치에 있다.

- 외부 표 첫 번째 페이지 영역 내부의 내부표 캡션
- `<hp:caption side="TOP">` 내부 문단의 `<hp:pic>`
- `binaryItemIDRef="image2"`
- `BinData/image2.png`는 정상 PNG

현재 로컬 재현 결과, 1페이지 SVG에는 `<image>` 요소가 1개만 생성된다. 이는 첫 셀의 `image1`에 해당하며, 캡션 내부 `image2`는 `ImageNode`로 방출되지 않는다.

## 사전 분석

### 원인 후보 확정

`src/renderer/layout/picture_footnote.rs::layout_caption()`은 캡션 문단을 `compose_paragraph()` 한 뒤 `layout_composed_paragraph()`로 넘긴다.

그러나 현재 호출은 마지막 인자들인 원본 문단 `para`와 이미지 데이터 `bin_data_content`를 `None`으로 고정 전달한다.

`src/renderer/layout/paragraph_layout.rs::layout_composed_paragraph()`의 주석과 구현에 따르면:

- `para`: treat_as_char 이미지 인라인 렌더링에 필요한 원본 문단
- `bin_data_content`: 해당 이미지 payload 조회에 필요한 바이너리 데이터

따라서 캡션 문단 안의 컨트롤은 파싱되어도 paragraph layout의 인라인 이미지 렌더링 분기가 활성화되지 않는다.

### 메인테이너 결정 범위

메인테이너 코멘트 기준으로 이번 PR은 다음만 수행한다.

- `layout_caption()`에 원본 캡션 문단 `para`와 `bin_data_content`를 스레딩한다.
- 캡션 내 `treat_as_char` 인라인 이미지를 depth 1로 렌더링한다.
- 인라인 경로는 `layout_caption()`을 재호출하지 않으므로 별도 재귀 가드 없이 depth 1로 제한된다.
- 캡션 속 그림의 캡션은 이번 범위에서 렌더링하지 않는 것이 의도된 동작이다.
- 플로팅 캡션 이미지, 셀 플로팅 배치 로직의 캡션 영역 일반화는 후속 이슈 범위다.

## 작업 목표

캡션 문단 안에 있는 `treat_as_char=true` 이미지가 SVG/렌더 트리에서 누락되지 않도록 한다.

정정 후 기대 상태:

- 캡션 문단의 인라인 그림이 `ImageNode`로 방출된다.
- 기존 본문/셀/글상자 문단의 인라인 그림 렌더링 경로는 유지된다.
- 캡션 안 플로팅 그림은 이번 정정으로 해결하려 하지 않는다.
- 이슈 #1270은 플로팅 후속 범위가 남으므로 이번 작업 완료 후에도 close하지 않는다.
