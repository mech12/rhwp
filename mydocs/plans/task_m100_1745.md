# Task #1745 수행계획서 — 텍스트-anchor 어울림 표의 후속 문단 wrap 흡수 실패 수정

## 배경
hwpdocs 1000건 rhwp vs 한글(OLE) 페이지·PI 대조(#1744 적용 HEAD 92a204f5)에서 PI_MISMATCH 검출.
대표: `law_downloads/보건복지부 식품의약품안전처/18177503_[별표 2] 업무정지 처분을 갈음하여
부과하는 과징금 산정기준(제33조 관련)(약사법 시행령).hwp` — pi1 rhwp 3쪽 vs 한글 1쪽 (총 3쪽은 동일).

## 원인 (empirical 규명)
- 구조: pi0 = 제목 텍스트 + 48×6 표(wrap=어울림, RowBreak, 1~3쪽 분할)가 **같은 문단**, pi1 = 빈 문단.
- 한글은 pi1 줄을 1쪽 표 오른쪽 잔여 띠에 배치. 저장 LINE_SEG `vpos=2072, cs=45568, sw=2620`
  (cs = 표폭 45002 + 바깥여백 283×2)이 이를 인코딩, 실측 캐럿도 p1.
- rhwp Task #362 흡수 로직(`src/renderer/typeset.rs:2810-2839`)은 wrap zone 기준 cs/sw를
  **표 문단의 첫 LINE_SEG**에서 취함. 표가 제목 문단에 anchor 되면 첫 seg 는 전폭 제목 줄
  (cs=0, sw=48188) → pi1(cs=45568, sw=2620) 매칭 전부 실패(`typeset.rs:2537-2541`) →
  wrap zone 종료 → pi1 을 표 아래(3쪽) 일반 배치.
- 부차 결함: 매칭이 되더라도 흡수는 **현재 column**(분할 표의 마지막 fragment 쪽=3쪽)에 기록되므로
  다쪽 분할 표에서는 한글(첫 fragment 쪽=1쪽)과 어긋남.

## 수정 방향
`src/renderer/typeset.rs` 2곳 (TypesetEngine 이 메인 파이프라인, Paginator 는 env fallback):
1. **wrap strip cs/sw 도출**: wrap zone 활성화 시 anchor 문단이 텍스트+표 혼합(첫 seg 가 전폭
   텍스트 줄)이면, 표 geometry(horz_offset + width + outer_margin 좌우)로 expected_cs 를
   계산해 `wrap_around_cs/sw` 에 저장 — #724 그림 `anchor_image_match` 와 동일 접근.
   기존 정확 일치 매칭은 무변경으로 작동.
2. **첫 fragment 쪽 소급 기록**: 분할 표에서 흡수된 WrapAroundPara 를 `st.pages` 에서 anchor 표의
   첫 PartialTable(cont=false) 이 있는 column 에 소급 push (미발견 시 현행 유지).
