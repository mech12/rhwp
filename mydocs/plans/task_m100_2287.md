# Task #2287 수행 계획서 — 표 밀집 문서 과소분할 (cut 행높이 잔여 누적)

- 이슈: #2287 [layout/table] 표 밀집 문서 과소분할 — cut 행높이 잔여 누적으로 -40~-64쪽
- 브랜치: `local/task2287` (upstream/devel + 제출 PR 7건 적용: #2271, #2273~#2276, #2284, #2286)
- 관련: #2237 (단일 문서 cut 높이 잔여), #2279 (반대 방향 +1 드리프트), 10k 서베이 r14

## 배경

10k 한글 오라클 서베이 r14 의 대형 음수 델타(rhwp 가 한글보다 수십 쪽 적음) 상위 3건:

| Δ | rhwp | hwp | 문서 |
|---:|---:|---:|---|
| −64 | 206 | 270 | 농촌지도직공무원 S-OJT 표준화매뉴얼 (45.6MB) |
| −41 | 88 | 129 | 미래부-국민과학기술정서 분석 (18MB) |
| −40 | 375 | 415 | 범교과 학습주제 연결 맵 (1.3MB) |

이슈의 기전 가설은 "표 cut 행높이 잔여의 선형 누적"이었다.

## 규명 절차 (delta staircase)

1. per-PI 델타 계단 도구(`tools/task2287/delta_staircase.py`)로 문서 내 이탈 지점 특정
2. `RHWP_TABLE_DRIFT` / `RHWP_DIAG_SCAN` / `RHWP_DIAG_BLKCUT` 로 표 분할 회계 분해
3. 한글 PDF(COM 생성)와 행 앵커 텍스트 검색으로 한글 측 행→쪽 배치 복원
4. `tools/task2195/rows_by_pi.py` 로 한글 행높이 걷기 교차 확인

## 규명 결과 — 단일 기전이 아니라 2개 결함의 복합

### 결함 1 — RowBreak rowspan 블록 연속 조각의 잔여 증발 (−40 문서 지배)

- 위치: `typeset.rs` 블록 컷 경로의 `rowbreak_use_row_offsets` 분기(#1486)
- 병인: 연속 조각(start_cut 보유)의 `block_fragment_height` 가
  `row_block_cut_row_content_height`(row_span==1 셀만 집계) 기반이라, 걸친
  rowspan 셀의 잔여 유닛(교육부 연결맵 47×9 셀[13]: 123유닛 중 97유닛 ≈ 1700px)이
  **0 으로 평가** → 블록이 즉시 "fits" 종료 → 선언 잔여 1904px/표 증발
- 파급: 쪽수 과소분할 + 렌더러는 잔여를 그대로 그려 **페이지 밖 오버플로**(SVG
  text y=3026 > 페이지 718px 실측)
- 실측: 47×9 표 rhwp 8쪽 vs 한글 11쪽 (cut 합 6016.8px 인데 조각 소비 합 4078px)

### 결함 2 — 저장 LINE_SEG 없는 TAC 그림 anchor 문단 높이 0 붕괴 (−64/−41 지배)

- 위치: `typeset.rs format_paragraph` / `height_measurer.rs measure_paragraph`
- 병인: 빈 텍스트 + TAC(글자처럼) 그림 컨트롤 + 저장 LINE_SEG 부재 문단은
  composed lines 가 비고, `empty_paragraph_fallback_line_metrics` 는 컨트롤 보유
  문단에 None → **문단 플로우 높이 0**. 그림(차트 387px/스캔 855px) 수십 장이
  한 쪽에 응축 (미래부 rhwp p65 = 한글 p101~107)
- 한글: 글자처럼 개체는 줄박스로 취급되어 그림 높이만큼 흐름 전진 (이웃 문단의
  저장 vpos 델타 437px 실측과 정합)

## 수정 설계

1. **결함 1**: `rowbreak_use_row_offsets` 분기의 연속 조각(start_cut 비어있지 않음)
   한정으로 `block_h` 와 `split_total` 을 `row_block_content_height`(rowspan 셀
   포함 콘텐츠 잔여)와 `max()` — 첫 조각(#1486 경로)은 불변
2. **결함 2**: 공용 헬퍼 `tac_object_stack_line_metrics`(renderer/mod.rs) 신설 —
   TAC 그림/도형 폭을 가용 폭에 greedy wrap 하여 줄별 (최대 높이, 0) 합성.
   `format_paragraph` / `measure_paragraph` 의 pairs 빈 경우 폴백에 연결
   (저장 LINE_SEG 보유·비-TAC 는 제외해 이중 계상 방지)
