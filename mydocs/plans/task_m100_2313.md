## 1. 목적

Phase A 완료 뒤 repository production/build/package는 legacy `/web` 앱을 직접 소비하지 않는다. 그러나
tracked 앱 코드와 개발 서버, 생성된 WASM glue, font compatibility link가 남아 있고 current CI·metrics·
contract·manual이 해당 경로를 계속 인식한다.

이번 작업은 다음 세 가지를 하나의 제거 단위로 완료한다.

1. 지원되지 않는 legacy 개발 앱과 전용 개발 자산을 저장소 root에서 제거한다.
2. current 운영 문서와 자동 검증이 Studio 및 canonical asset 구조만 설명하도록 정리한다.
3. metrics 감소를 코드 개선이 아닌 모집단 삭제 결과로 분리하고 Phase B 재평가 입력을 만든다.

## 2. 최신 기준선과 제거 대상

### 2.1 tracked `/web` inventory

기준 커밋에는 `web/` 아래 18개 tracked entry가 있다.

| 분류 | 대상 | 처리 |
|------|------|------|
| legacy 앱 | `app.js`, `char_shape_dialog.js`, `editor.js`, `font_substitution.js`, `format_toolbar.js`, `text_selection.js`, `editor.css`, `style.css`, `editor.html`, `index.html` | 삭제, Studio로 이동·복사하지 않음 |
| 개발·테스트 | `https_server.py`, `clipboard_test.html`, `certs/localhost-{cert,key}.pem` | 삭제. Vite 개발 서버가 current 경로 |
| tracked generated | `rhwp.js`, `rhwp.d.ts`, `rhwp_bg.wasm.d.ts` | 삭제. fresh output authority는 ignored `pkg/` |
| compatibility | `web/fonts -> ../assets/fonts` | 삭제. canonical `assets/fonts`와 Studio link는 유지 |

metrics의 `legacy-web` 모집단은 위 앱 파일 10개, 6,592 lines(5,795 code lines), AST function
251개다. test/dev/generated/link는 현재 규칙에 따라 complexity 모집단에서 제외돼 있다.

### 2.2 repository 내부 결합 분류

| 분류 | 최신 기준선의 결합 | 판정 |
|------|--------------------|------|

저장소 밖에서 특정 commit의 `/web`을 직접 여는 사용자는 탐지할 수 없다. 다만 이 경로는 public npm API나
release artifact가 아니고 current manual에서도 이미 legacy로 표시돼 있으며 Studio 대체 경로가 존재한다.
따라서 compatibility stub이나 저장소 내 archive를 새로 만들지 않고, 이슈·PR과 Git history를 제거 근거와
복구 경로로 사용한다.

## 3. 수행 범위

### 3.1 legacy tree 제거

- tracked `web/` 18개 entry를 전체 제거한다.
- Studio 또는 다른 package에 legacy source 복사본을 만들지 않는다.
- self-signed certificate와 private key, Python HTTPS server, clipboard 전용 page를 제거한다.
- tracked generated glue 대신 repository Docker/CI가 생성하는 `pkg/`를 유일한 current WASM output으로
  취급한다.

### 3.2 current tooling과 contract

- frontend impact detector에서 존재하지 않는 `web/` prefix를 제거한다.
- metrics의 `legacy-web` include group과 `/web` 전용 제외·판별 코드를 제거한다.
- font inventory metadata에서 `legacyWebLink`를 제거하되 schema v2 function comparison은 유지한다.
- font asset contract에서 Studio canonical link만 검증하고 legacy compatibility assertion을 제거한다.
- 제거 PR 자체는 `.github/workflows/ci.yml`과 `scripts/frontend-*` 변경으로 frontend gate가 실행되므로
  deleted filename 감지에 의존하지 않는다.

### 3.3 current 문서와 guardrail

- 한국어·영어 local web server manual에서 legacy Python HTTPS 절과 `/web` troubleshooting을 제거한다.
- Studio Vite server를 유일한 current local web workflow로 표현하고 `신규` 같은 과도기 표현을 없앤다.
- font fallback 문서의 current ownership 설명에서 legacy compatibility link를 제거한다.
- #2023 contract guardrail을 post-Phase-A/post-removal 구조로 현행화한다.
- dated diagnosis, baseline, migration, report와 changelog의 과거 `/web` 언급은 일괄 치환하지 않는다.

### 3.4 metrics와 Phase B 입력

- 공식 #2124 snapshot은 변경하지 않고 `--compare` 입력으로만 사용한다.
- 동일 base pre/post 결과에서 삭제된 legacy group과 나머지 current group delta를 분리한다.
- Total CC, global Top 20, CC>25 합·개수, CC>100 합·개수, max와 stable function diff를 모두 기록한다.
- legacy 제외 후 current hotspot 순위를 #2022 Phase B 재평가 입력으로 남기되 실행 이슈는 자동 생성하지
  않는다.

## 4. 범위 제외와 금지 목록

### 4.1 범위 제외

- `assets/fonts` 파일 추가·삭제·내용·파일명·fallback 변경
- Studio runtime `fonts/...` URL 또는 `rhwp-studio/public/fonts` link 변경
- `InputHandler`, dialog, `WasmBridge`, `diff-engine`, `core/types.ts` 해체
- Rust/WASM API와 generated `pkg/` 형식 변경
- Chrome/Firefox/Safari/VS Code 기능, manifest, CSP, WAR 정책 변경
- `@rhwp/editor` iframe/MessageChannel/public API 변경
- Safari signing, 배포, Xcode build pipeline 개선
- release version 또는 historical changelog 재작성

### 4.2 확정 금지

- legacy source를 `rhwp-studio` 또는 별도 archive directory로 복제
- `assets/fonts`나 `rhwp-studio/public/fonts`를 `/web`과 함께 삭제
- `@rhwp/editor` runtime dependency 추가
- React/Vue/Svelte 등 runtime UI framework 도입(v1.0까지)
- 확장 CSP 완화, inline script 추가, `web_accessible_resources` 확대
- sender/URL/file signature/size 검증 약화 또는 `publicDir: false` 우회
- source removal과 기능·보안·font 정책 변경을 같은 commit에 혼합
- CC/LOC 감소를 함수 해체 또는 SOLID 점수 개선으로 계상
- #2124 snapshot이나 과거 task evidence를 post-removal 경로로 다시 쓰기
- 필수 gate 미실행을 근거 없이 PASS로 표시
