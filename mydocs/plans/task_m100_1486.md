## 재현 및 분석 결과

`cargo build --release` 후 현재 release CLI 기준으로 확인했다.

```text
./target/release/rhwp dump-pages samples/hwpx_sample2.hwpx -p 8
```

9쪽 주요 PageItem:

```text
PartialTable   pi=74 ci=0  rows=0..1  cont=true  1x1
FullParagraph  pi=75
FullParagraph  pi=76  "■ 모바일 신청방법"
PartialTable   pi=77 ci=0  rows=0..1  cont=false  1x1
```

실제 오른쪽 이탈 대상은 `pi=77` 외곽 표가 아니라 `pi=74` 분할 표 내부의 TAC 중첩 표다.

render tree 기준:

```text
Table pi=74 bbox x=37.8 w=718.1
  nested Table bbox x=613.5 w=639.8  # 오른쪽 1253.3px까지 확장
```

PDF 정답지 9쪽에서는 같은 내부 표가 좌측 본문 안쪽에 배치된다.

## 원인 후보

`src/renderer/layout/table_partial.rs`의 분할 표 셀 내부 `Control::Table` 경로에서 TAC 중첩 표를 앞 텍스트 폭 뒤에 그대로 이어 붙인다. 해당 TAC 표 폭은 약 640px이고, 앞 텍스트 뒤 남은 줄에 들어가지 않는데도 줄바꿈 처리 없이 `x ~= 613px`에 배치된다.

일반 표 경로(`src/renderer/layout/table_layout.rs`)는 TAC 표를 별도 분기에서 더 세밀하게 처리한다. 분할 표 경로는 이 처리와 달라서, 분할된 외부 표의 연속 페이지에서만 내부 TAC 표가 우측으로 튀는 것으로 보인다.

## 작업 범위

1. 분할 표 셀 내부 TAC 중첩 표의 수평 배치 규칙을 일반 표 경로와 맞춘다.
2. 앞 텍스트 뒤 남은 폭보다 TAC 표가 크면 같은 줄에 이어 붙이지 않고 다음 줄/좌측 기준으로 배치한다.
3. `samples/hwpx_sample2.hwpx` 9쪽 render tree에서 대상 내부 표의 오른쪽 경계가 body 영역 안에 들어오는 회귀 테스트를 추가한다.
4. 필요하면 HWP 변환본과 PDF 정답지를 시각 검증 자산으로 함께 보존한다.

## 제외 범위
