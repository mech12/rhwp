# Task M100 #1785 수행 계획서 — micro-grid 셀 width_ref 물질화의 유효 패딩 보존

## 배경

Issue #1785: HWPX/HWP5 파스 경로 간 `cell.apply_inner_margin` 불일치로 micro-grid 표
높이가 9.3px 달라지고 본문 흐름이 밀림 (#1772 수정 후 잔여 9.25px 군집의 원인).
조사 상세: `mydocs/troubleshootings/microgrid_cell_margin_roundtrip.md`.

## 원인 요약

HWPX→HWP 어댑터 `materialize_cell_list_header_contract` 가 micro-grid(col_count≥30)
표의 셀에 LIST_HEADER width_ref bit0 을 강제 세팅(한컴 record 계약)하는데, HWP5 파서는
bit0 을 `apply_inner_margin` 으로 해석한다. 원래 aim=false 셀(유효 패딩=table.padding)이
재파스 후 aim=true(유효 패딩=cell.padding)로 반전되어 렌더가 달라진다.

## 수정 내용 (구현 중 접근 변경)

**1차 안(어댑터 유효 패딩 물질화)은 기각**: 어댑터에서 `cell.padding = table.padding` 로
덮으면, 레이아웃의 실제 규칙(레거시 보존값: aim=false 여도 cell > table 이면 셀 값 사용)과
충돌해 코퍼스 300건에서 OVER 10→121 회귀 발생. 원인 재분석 결과 진짜 결함은
**height_measurer 가 레이아웃(resolve_cell_padding)과 다른 단순 `aim ? cell : table`
규칙을 쓰는 것** — 주석은 "resolve_cell_padding 과 일관성"을 주장하나 레거시 보존값
케이스에서 어긋난다. A(HWPX, aim=false) 측정만 0-패딩이 되어 표 예약 높이가 렌더와
불일치했다.

**최종 수정**: 셀 안 여백 선택 규칙을 단일 출처로 추출하고, 측정에는 하이브리드 적용.

- `Cell::use_cell_padding_axis` / `Cell::effective_padding` 신설 (model/table.rs)
- `resolve_cell_padding`(table_layout.rs)의 축별 규칙을 위 함수로 위임 (동작 불변)
- `height_measurer.rs` 행 높이 사이트: **aim=false 만** `effective_padding`(레거시 보존값
  규칙)으로 통일 — #1785 결함 지점. **aim=true 는 기존대로 cell.padding(0 존중) 유지** —
  전면 통일(2차 안)은 `aim=true && padding=0 → 표 기본 폴백` 의미까지 측정에 가져와
  #493 세로 Shift 리사이즈(셀보호2.hwp 셀[20] pad top/bottom=0)를 깨뜨림(3차 안에서 축소).
- 어댑터(width_ref bit0 계약)는 변경 없음 — record 계약과 렌더 의미가 자연 정합
- 후속 항목: layout 의 `aim=true && 0 → 표 기본 폴백`(코드)과 주석("0도 존중")의 모순,
  측정과의 정합은 별도 조사
