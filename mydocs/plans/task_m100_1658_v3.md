표(결재/서명 박스)를 TypesetEngine 이 문서순 flow 로 끼워넣어 생기는 over-pagination 을

## 확정된 근인·정답 모델 (#1653 RCA — 재조사 불필요)

- **근인**: `float_placement.rs is_para_topbottom_float()` 가 `VertRelTo::Para` 만 인식 →
  `VertRelTo::Page` 하단 고정 표가 flow 로 떨어져 `current_height` 에 전량 누적.
  한글은 이런 표를 페이지 하단에 아래→위로 스택하고 본문은 그 위에 흐름.
- **1차 시도 실패 교훈 (Stage 2b)**: vert=Page 일괄 out-of-flow(높이 0)는 **일치 75→55,
  under 12→34 대규모 회귀** — 이 표들은 실제 본문 공간을 차지한다. 하단 고정 표 **1개**면
  문서 말미 flow ≈ 하단이라 일치, **2개**면 한글은 하단에 함께 스택해 1쪽 수용(rhwp 는 +1쪽).
- **정답 모델**: out-of-flow(0)도 문서순 flow 도 아닌 **하단 예약** — 하단 고정 표 높이를
  해당 페이지 하단에서 예약(가용 높이 축소)하고 본문은 그 위를 채움. 다수면 하단 스택.

## 구현 방향 (구현계획서에서 상세화)

1. **판별**: `treat_as_char=false ∧ wrap=TopAndBottom ∧ vert_rel_to=Page ∧ valign=Bottom`
   (RCA 속성 집합; Paper 포함 여부는 Stage 1 에서 판단).
2. **typeset 회계**: 해당 표를 문서순 위치에서 배치하지 않고 **현재 페이지 하단 예약
   큐**에 등록 — 페이지 가용 높이(available_height)를 표 높이+마진만큼 축소. 같은 페이지에
   다수면 아래→위 스택(예약 누적). 페이지 넘김 시 예약 재산정(anchor 문단이 넘어가면
   표도 다음 페이지 하단).
3. **렌더 배치**: layout 은 vert=Page 를 이미 절대배치하나 valign=Bottom 다수 스택의
   y 산정(겹침 방지)이 예약 큐와 정합해야 함 — PageItem 에 하단 스택 정보 전달.
4. 기존 부동 표 기제(`visible_float_exclusions`, `deferred_table_controls`,
   #1745/#1753/#1755 경로)와의 상호작용을 격리 — 판별 조건을 좁게 유지.
