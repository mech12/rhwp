# Task #1728 v2 수행계획서 — RowBreak 셀-내 continuation 상단 space-before 복원

## 배경
Task #1728(부분)은 footer 쪽번호 세로 위치(`margin_footer/2`) 갈래만 수정했다
(`task_m100_1728_report.md`). 보고서 "범위/한계"에 **남은 갈래 = continuation 상단
space-before 트림**으로 명시되어 있다. 본 v2는 그 갈래를 처리한다.

## 조사 결과 (코드 무수정, 96 DPI PNG 픽셀 측정 vs 한글 2024 PDF)

### 증상 (content top/bottom, dark-pixel row 기준)
| 페이지 | PDF top | rhwp top | Δtop | 비고 |
|--------|--------:|---------:|-----:|------|
| giant p5 (idx4) | 95 | 77 | **−18** | continuation, 본문이 위로 당겨짐 |
| giant p6 (idx5) | 95 | 77 | **−18** | 동일(상수 오프셋) |
| giant **p1** (idx0, 첫 조각) | 81 | 80 | −1 | **정합** — 첫 조각은 정상 |
| scattered p5 | 77 | 75 | −2 | 전역 오프셋(정상) |
| scattered p6 | 77 | 75 | −2 (top) | 상단 정상, 하단 +13 별건 |

- **첫 조각(cont=false)은 정합(−1px)**, continuation(cont=true)만 −18px → continuation 전용 결함.
- Δtop 이 p5·p6 모두 상수(−18) → 문단별 값이 아니라 **고정 인셋 누락**.

### 근본 원인 (코드 규명)
- giant 표 = 5행×1열, 거대 셀(row 4, 654문단)의 **셀-내 텍스트가 페이지 분할**(intra-cell split).
- continuation 페이지의 첫 가시 문단은 셀-상단(`is_column_top`)에 오지만, 셀-상대
  인덱스 `cp_idx > 0` 이다.
- `paragraph_layout.rs:1564-1575` (`layout_composed_paragraph`) 의 spacing_before 규칙:
  - `is_column_top && para_index==0` → `min(spacing_before, vpos0)` 적용
  - `is_column_top && para_index>0` → **spacing_before 전량 트림**(Task #853, 본문 page-break 정합용)
- 따라서 continuation 첫 heading 문단(`5.1.x`, **spacing before=1000 HU=13.3px**)의
  앞 간격이 트림된다. 한컴 PDF 는 이 앞 간격을 유지 → rhwp 가 13px 위로 뜬다.
- scattered 는 **행 단위 분할**(inter-row): continuation 셀이 자기 문단 `cp_idx==0` 부터
  렌더 → `para_index==0` 클램프 분기에 정상 진입 → 상단 정합(−2). 두 샘플 차이의 설명.

### 핵심 코드 위치
- `src/renderer/layout/paragraph_layout.rs:1564-1575` — spacing_before 트림 지점(수정 대상)
- `src/renderer/layout/table_partial.rs:967` — continuation 셀 문단 렌더 호출(플래그 set 지점)
- `src/renderer/layout.rs:959` — 엔진 `Cell<bool>` 토글 필드 정의 패턴

## 구현 방침
`layout_composed_paragraph` 는 12+ 호출부가 있어 위치 인자 추가는 노이즈가 크다.
코드베이스 기존 관용(`use_hwp3_origin_flow_spacing_before: Cell<bool>`)을 따라
**엔진 `Cell<bool>` 토글**로 스코프한다.

- continuation 조각(`cut_units.su > 0` = 이전 페이지에서 이월된 분할 셀)의 **첫 가시 문단**
  이며 `start_line==0` 일 때만 플래그를 set → 그 문단은 `is_column_top && para_index>0`
  이어도 spacing_before **전량** 적용.
- 그 외(첫 조각, 본문 page-break, scattered inter-row `cp_idx==0`)는 무영향.
