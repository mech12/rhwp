# Task #1663 수행계획서 — 빈 host co-anchored 자리차지 float 표 orphan control + 표 뒤 말미 빈 문단 흡수

## 1. 이슈

- GitHub: <https://github.com/edwardkim/rhwp/issues/1663>
- 제목: `빈-host co-anchored 자리차지 RowBreak 표: 후속 표가 페이지 경계에서 행 분할돼 헤더-본문 분리 + 표 뒤 빈 페이지`
- 상태: OPEN
- 관련: #1510/#1535/#1549/#1639 — 빈/visible-host co-anchored TopAndBottom float 표 계열. 본 건은 그 표들의 **페이지네이션(분할 vs 통째 이월)** 영역으로 별개.

## 2. 재현 (확정)

결재 헤더 + 점검표 본문이 한 자리차지(TopAndBottom) 표로 구성된 실문서(비공개 점검표 양식)와 issue1639 구조 Clone and Narrow 합성 fixture 로 확정.

- 빈 host 문단에 co-anchored 자리차지(TopAndBottom, vert=문단) RowBreak 표 A·B 가 있고 선행 표 A 가 page0 를 채우면:
  1. **orphan**: 후속 표 B 가 page0 잔여에 머리 행만 남기고 본문이 page1 로 행 분할된다. 한컴은 한 페이지에 들어가는 자리차지 표를 분할하지 않고 통째로 다음 페이지에 둔다.
  2. **표 뒤 빈 페이지**: 통째 이월된 B 가 page1 을 채운 뒤 문서 말미 빈 문단이 빈 page 를 추가한다. 한컴은 흡수.
- 실문서 export-pdf(작성 시점 base): 3페이지(머리행 orphan + 빈 페이지). 한컴 2022 PDF: 2페이지.
- 합성 fixture `dump-pages`(clean): p0=`Table A` + `PartialTable B(rows 0..1)`, p1=`PartialTable B(rows 1..21)`, p2=빈 page → 3페이지.

> [최신 `devel` 재확인, #1722/#1686 이후] #1722 가 실양식의 trailing 빈 페이지는 없애 실양식은 이제
> 2페이지지만 orphan(머리행+본문 분리)은 잔존한다. 합성 fixture 는 여전히 3페이지(두 증상 모두). orphan
> 이 미해결이라 본 point-fix(통째 이월 + trailing 빈 문단 흡수)는 최신 `devel` 에서도 유효하다.

## 3. 근본 원인 (소스 확정)

`src/renderer/typeset.rs`:

- `typeset_block_table`: 자리차지 표가 현재 페이지 잔여를 초과하면 행 단위(RowBreak) 분할 경로로 진입한다. 선행 co-anchored float 가 채운 후속 표도, 한 페이지(`table_total <= available`)에 들어가는데도 머리 행만 남기고 분할.
- `typeset_section_with_variant`: 말미 빈 문단이 현재 page 잔여를 초과하면 새 page 를 만든다. 기존 빈페이지 차단 가드(`next_will_vpos_reset`/force-break)는 "다음 문단이 reset/쪽나눔"인 경우만 커버하고, **문서 끝 trailing 빈 문단**은 미커버.

## 4. 수정 방향 (옵션 검토 → 채택)

- **옵션 A**(keep-block: 헤더 + 최소 N 데이터행을 함께 유지): "최소 N행" magic 상수가 필요하고 분할 루프·머리행 반복·rowspan 보호와 상호작용해 회귀면이 넓다.
- **옵션 B**(채택): **fits-fresh 통째 이월 + co-anchored 한정**. 자리차지 RowBreak 표가 잔여 초과·fresh 적합(`table_total <= available`)이고 같은 host 에 선행 co-anchored float 가 있으면 `advance_column_or_new_page` 로 통째 이월. fresh 에도 안 들어가는(페이지보다 큰) 표는 이월 안 함 → 무한 push 불가.
  - **co-anchored 한정 근거(실측)**: 단독 anchored 자리차지 표(host 의 첫/유일 float)까지 통째 이월하면 복잡 문서 `hwpx_sample2` 페이지수가 29→33 회귀(한컴 29). clean vs orphan-only `dump-pages` 대조로 단독 anchored 표 4개(pi=32/77/100/127, 모두 `ci=0`)가 과도 이월됨을 확인. 선행 co-anchored float 가 채운 후속 표(`ci>0`, 검증점검표 [3] 형)로 한정해야 한컴 정합 + 무회귀.
- **trailing 빈 문단**: page 마지막 항목이 co-anchored 자리차지 표일 때만 말미 빈 문단 overflow 흡수(동일 신호). 일반 빈 문단 페이지네이션(편집 Enter 누적·본문 trailing 빈 줄)은 불변 — 광범위 흡수는 `test_page_overflow_with_enter`·`hwpx_sample2`(29) 회귀를 유발해 기각.

근거: 한컴은 한 페이지에 들어가는 자리차지 표를 쪼개지 않고, 그 표 뒤 빈 문단을 trailing overflow 로 흡수. 최소 변경(가드 2곳) + co-anchored 신호로 회귀 차단.
