## 배경 (#2211 실측 승계)

행 성장 판정의 pad 이중 계상 정정(`relaxed_pad`)이 **중첩/TAC 표 한정**인 이유는
전면 적용 시 상호 배타 실패가 실측됐기 때문: 레이아웃 단독 → #1858(하단앵커
flush, 측정·렌더 8.5px 불일치) / 측정 미러 추가 → #1748(걸침 rowspan 연속쪽 컷
예산 한 줄 오버 13.6px). 즉 비-tac float/분할 표는 **4개 회계가 pad 포함 규약으로
상호 캘리브레이션**되어 있다: ①layout `resolve_row_heights` ②`HeightMeasurer`
(1-b/2-b) ③`row_cut_content_height`(컷 예산) ④분할 컷 오버라이드(RowCutPolicy
계열). 본 타스크는 네 경로를 **동일 규칙으로 동시 전환**한다.

## 1단계 — 잔여 증상 재현·정량 (신규 기준선)

- 비-tac float/분할 표에서 "저장 ls 콘텐츠가 선언 h에 꽉 찬 행"의 +pad 팽창
  사례를 한컴 PDF 대조로 2건 이상 확보 (#1858 fixture의 알려진 드리프트
  3.7px(=pad 쌍) 포함, rowbreak/1748/exam 계열 스캔).
- /2 안착 후 기준치로 각 사례의 (선언 h, 렌더 h, 한컴 h) 표 작성. 한컴 대조가
  안 되는 사례는 대상에서 제외(정답지 없는 정정 금지).

## 2단계 — 4-경로 회계 지도 + 동시 전환 설계

- 경로별 pad 취급 현황표: 성장 판정 산식·호출 관계·캘리브 근거 커밋을 명세.
- 전환 규칙(#2211과 동일): 저장 LINE_SEG 줄 흐름 pad 미가산 / 개체 기반
  (Square·중첩·flow) pad 유지 / LINE_SEG 부재 셀 기존 유지.
- 컷 예산(③④)은 "유닛 합"이 이미 pad 별도 회계이므로, 행 박스와 유닛 합의
  정의 차이를 먼저 명세한 뒤 전환 지점을 확정한다 (#1748의 13.6px = 컷 예산
  한 줄 오버가 재발하지 않는 설계 증명 포함).
- 설계 확정 후 **구현 계획서(`task_m100_2221_impl.md`) 승인 요청** — 영향
  범위가 전 문서 쪽나눔이므로 단계 승인 게이트를 둔다.

## 3단계 — 동시 전환 구현 + 게이트

- 네 경로 동시 적용 + `relaxed_pad` 가드 해제(전면화).
- 게이트(가중): 전수 `--tests --no-fail-fast` + golden 8 + **페이지 오라클
  tsv 546건**(쪽나눔 이동 전수 검출) + OVR 5샘플 + 표적 스위트(#1748/#1858/
  #1486/#2211/#2220/#2189/#2207) + #2223 핀(307/159).
- 쪽수/개체 이동이 발생하면 건별로 한컴 정답지 대조 → 정합 방향만 수용,
  이동 목록을 보고서에 전수 기록(no silent caps).

## 4단계 — 시각 판정 + 보고

- 이동 발생 문서 중 대표 3건 전/후/한컴 3-way → 작업지시자 시각 판정.
- baseline/골든 현행화는 판정 통과 후 (rowbreak 전례).
- 최종 보고 → devel 반영 → close 승인 요청.

## 위험 관리

- 실패 시 롤백 전략: 경로별 커밋 분리로 이분 가능하게 유지.
- #1748 컷 예산 재발 시 해당 축만 pad 유지로 후퇴하고 사유를 보고서에 명기
