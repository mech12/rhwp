③ 800ms static-layer verification ④ text-edit rAF 배칭/무효화 페이지 특정.

## 절차 (버그 정정형 — 구현계획서 생략, 진단→정정→게이트)

### 1. 브라우저 계측으로 stale 경계 확정 (E2E)

- 기존 Puppeteer 인프라(`rhwp-studio/e2e/`, headless Chrome)로 재현
  스크립트 신설: 문서 열기 → para5 끝 캐럿 → `1` 반복 입력(줄 경계 초과)
  → 입력 직후 / 2 rAF / 800ms / 1.5s / Enter 후 시점별로
  **canvas 픽셀(새 줄 영역)** + `getCursorRectInCell` + 모델 텍스트 계측.
- 후보 ①~④ 중 stale이 처음 나타나는 경계를 이분 확정 (예: overlay 재사용
  off 강제 시 재현 소멸 여부).

### 2. 최소 범위 정정

- 확정된 경계에서 가장 작은 수정 (예: text-edit 재도장 시 새 줄이 기존
  static 층 픽셀에 덮이는 경우 해당 층만 재구성, 또는 verification의
  stale 복원 차단). **매 입력 전체 pagination 우회는 금지** (이슈 완료
  조건) — native 층은 이미 올바르므로 표시 층 수정으로 완료 가능.

### 3. 게이트 + 판정

- 표적 E2E(신설, 수정 전 FAILED 실증) + `npm test`/`tsc`/`vite build`
- native 회귀: `issue_2185`(LINE_SEG/vpos/115쪽 핀), `issue_1949`(전 페이지
  렌더), 전수 `--tests --no-fail-fast`
- WASM 빌드(변경 시) → **작업지시자 실사용 판정**: 이슈 재현 절차 그대로
  (반복 입력 → 시점별 관찰 → Enter → 저장·재열기).
