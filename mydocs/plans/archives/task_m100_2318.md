# 수행계획서 — #2318 바탕쪽 개체가 본문 텍스트를 가림 (studio plane 분류)

- 이슈: [#2318](https://github.com/edwardkim/rhwp/issues/2318)
- 마일스톤: M100 (v1.0.0)
- 브랜치: `local/task2318`
- 유형: 버그 정정형 — **구현계획서 생략** (본 계획서에 단계 포함, #2292/#2293 선례)

## 1. 증상

`samples/basic/shortcut.hwp`를 rhwp-studio로 열면 바탕쪽(master page) 개체가 본문
텍스트 **앞**에 합성되어 글자를 가린다. 한컴 편집기는 바탕쪽을 글자 **뒤**에 배치한다
(작업지시자 실기 확인, 2026-07-17). 네이티브 SVG/skia export는 정상.

## 2. 근인 (착수 정찰 실증)

1. shortcut.hwp의 바탕쪽 컨트롤은 글상자 도형 1개, `attr=0x046A4000` —
   wrap 비트(21~23) = **3 = InFrontOfText(글 앞으로)**.
2. `render_layer_from_master_control`([layout.rs:1525](../../src/renderer/layout.rs#L1525))은
   바탕쪽 컨트롤에 자체 wrap을 그대로 `RenderLayerInfo`로 부여한다. 전지 크기
   배경(paper 대비 ≥95% + paper 기준 원점)만 BehindText로 강등하는 기존 가드
   ([layout.rs:1543](../../src/renderer/layout.rs#L1543))가 있으나 이 Rect(576×448px)는 미해당.
3. paint replay plane 분류기([replay_order.rs:40](../../src/paint/replay_order.rs#L40),
   studio [replay-plane.ts](../../rhwp-studio/src/view/canvaskit/replay-plane.ts))는
   layer의 text_wrap만 보고 plane을 정한다 → 바탕쪽 개체가 **inFrontOfText plane**으로
   분류 → studio 다층 canvas 합성(#516)에서 front overlay canvas(z-index 최상)가
   본문 텍스트 canvas 위에 얹혀 글자를 가린다.
4. SVG/skia는 렌더 트리 문서 순서(PageBg → MasterPage → Body)로 그리므로 plane
   분류와 무관하게 정상.

**한컴 의미론**: 바탕쪽 개체의 wrap은 바탕쪽 **내부** 개체 간 순서에만 적용되고,
바탕쪽 전체는 항상 본문 뒤에 깔린다.

## 3. 정정 방향

**바탕쪽 유래(provenance) 기준 plane 상한(cap)**: paint 파이프라인은 이미 바탕쪽
그룹을 `GroupKind::MasterPage`([layer_tree.rs:342](../../src/paint/layer_tree.rs#L342),
builder.rs:212)로 표시하고 JSON에도 `{"kind":"masterPage"}`로 직렬화한다. 이
컨텍스트를 plane 분류에 반영하여 **MasterPage 그룹 내부의 모든 op를 BehindText
plane으로 상한 고정**한다.

- `render_layer_from_master_control`의 layer(wrap) 자체는 유지 —
  `sort_paper_render_nodes`의 바탕쪽 **내부** 정렬(plane→zOrder→stable_index)은
  현행 그대로 보존된다. 변경은 paint replay plane 분류층에만 국한.
- BehindText plane 내에서 바탕쪽 그룹은 트리 순서상 본문 behind 개체보다 앞서
  재생되므로 더 깊게 깔린다 (한컴 정합).
- 기존 전지 배경 강등 가드는 그대로 유지 (케이스별 명시 가드 원칙).
- 케이스 구조 가드: `GroupKind::MasterPage` 한정. Header/Footer는 이번 범위에서
  제외하되 동일 증상 여부를 4단계에서 점검만 하고 발견 시 별도 이슈.

수정 지점 (rust + TS 양측):

| 층 | 파일 | 내용 |
|----|------|------|
| rust 분류 | `src/paint/replay_order.rs` | 그룹 컨텍스트(masterPage) 인지 plane cap |
| rust 필터 재생 | `src/renderer/web_canvas.rs` (LayerFilter), `src/renderer/canvaskit_policy.rs` (replay plan) | MasterPage 그룹 하강 시 plane cap 전파 |
| studio 분류 | `rhwp-studio/src/view/canvaskit/replay-plane.ts`, `page-renderer.ts`(collectLayerPlaneSummary), `canvaskit-renderer.ts` | 동일 cap을 TS 재생/집계에 반영 |

## 4. 단계

1. **재현 고정**: shortcut.hwp 기반 실패 테스트 작성 — PageLayerTree에서 바탕쪽
   글상자 op가 front plane으로 분류됨을 드러내는 rust 단위 테스트(+ 필터 재생
   플레인 집계). 게이트: 신규 테스트가 현행 코드에서 실패함을 확인.
2. **rust 정정**: replay plane 분류에 MasterPage 그룹 컨텍스트 반영(cap=BehindText)
   — replay_order + web_canvas 필터 + canvaskit_policy replay plan. 1단계 테스트
   통과 + 기존 paint/renderer 테스트 회귀 없음.
3. **studio 정정**: TS 분류·집계·재생에 동일 cap 반영 + studio 단위 테스트.
   WASM 빌드 후 e2e(CDP, `mydocs/manual/e2e-cdp.md` 준수)로 shortcut.hwp 화면에서
   본문 텍스트가 바탕쪽 위에 보임을 스크린샷 검증.
4. **게이트 + 시각 판정**: `cargo test --tests --profile release-test` + fmt --check +
   studio 단위 전체 + WASM 빌드. Header/Footer 유사 증상 점검(발견 시 별도 이슈).
   완료 후 작업지시자 시각 판정 요청 (studio에서 shortcut.hwp — 한컴 실기 대조).

## 5. 위험

- 바탕쪽 내부 z-order 회귀: 내부 정렬은 layout 층에서 이미 완료된 순서를
  건드리지 않으므로 낮음. 1·2단계 테스트로 고정.
- 히트테스트 plane 노출(`get_page_control_layout_native` — paper_node_sort_key
  재사용): 이번 변경은 paint 분류층에 국한되어 영향 없음. 4단계에서 확인.
- 본문 BehindText 개체와 바탕쪽의 상대 순서: 트리 순서(바탕쪽 먼저)로 결정 —
  동일 plane 내 재생 순서 보존 여부를 2단계 테스트에 포함.
