## 1단계 — 방출 지점·프로파일 지도 (정찰 일부 완료)

- 확인됨: 미지정 그림은 layout에서 **Image 노드**로 방출되고 백엔드가 회색
  폴백을 그림 (render tree 실덤프). `RenderProfile { Screen, Print, … }` 기존
  존재, OLE용 `PlaceholderNode` 인프라 존재.
- 남은 지도: ①layout의 bin 미해결 그림 처리 지점(find_bin_data 실패 경로)
  ②export 경로들이 쓰는 렌더러/프로파일(legacy SvgRenderer vs paint
  LayerBuilder) ③4-backend(svg/canvas/paint/json) placeholder 소비 방식.

## 2단계 — 설계·구현

- layout: bin 미해결 그림을 Image 대신 **Placeholder(kind=MissingPicture)**
  노드로 방출 (bbox 유지) — 단일 의미 노드로 4-backend 일관 처리.
- 백엔드: Screen 계열 = 점선 테두리 + 그림-없음 아이콘(내장 벡터 픽토그램,
  외부 자산 없이) / Print(export 계열) = 무방출. export 경로가 Print 프로파일
  을 쓰지 않는 곳은 프로파일 전달 보강.
- studio 편집 뷰는 Screen 유지로 자동 정합. 판정 기준: 미지정 판정은
  bin_id 참조 실패/데이터 부재 — 스펙 필드만 사용(하드코딩 금지).
