# #1858 발현 2 조사 — vert=쪽 valign=Bottom 하단 블록 세로 오프셋 (36389312)

> **후속 결과 (2026-07-04)**: 아래 권고대로 하단앵커 회귀코퍼스(opengov 결재문서 18건 +
> 한컴 오라클)를 먼저 구성해 정량화한 결과, 원인 후보 (2)가 확정되었다 — 한컴은 하단앵커
> 표의 **실측(렌더) 하단**을 anchor 하단에 밀착시키는데 rhwp 는 **선언높이(common.height)**
> 기준으로 top 을 잡아, 선언높이가 실측보다 큰(stale) 문서에서 블록이 (선언−실측)만큼
> 위로 떴다(36389312 pi=6: 선언 357.2px vs 실측 316.7px). **PR #1894 로 수정 완료**
> (동일-상수 클래스 13건 −30.5±1.3pt → −1.4~−2.8pt, 쪽수 전 코퍼스 불변, 게이트
> `tests/issue_1858_bottom_anchor_flush.rs`). 이하는 조사 당시 기록 원문이다.

## 결론 요약

`samples/hwpx/opengov/36389312`(관악소방서 결재문서)의 하단 발신/결재 블록(pi=5·pi=6,
`vert=쪽` `valign=Bottom`) 전체가 한컴 정답지 대비 세로 **~30px 위로** 밀려 렌더된다.
페이지 수는 정합(1쪽, #1611/#1658 튜닝)하나 페이지 내 위치가 어긋난다.

**성숙 코드(#321/#1611/#1624/#1658)의 잔여 오차이며, 근본 원인이 단일 지점으로 확정되지
않고 surgical·검증가능 수정이 도출되지 않았다.** 무리한 수정은 하단 앵커 블록을 쓰는 다수
문서(footer/발신명의/결재틀)를 회귀시킬 위험이 크다. → 별도 전용 조사·회귀코퍼스 필요.

## 코드 경로 (조사)

- 배치: `is_page_bottom_fixed_float`(float_placement.rs:24) → Page+Bottom 제외 경로
  (typeset.rs:11893-11955). 페이지네이션(flow 제외)은 정상 — 페이지 수 정합.
- 그리기 Y: `table_layout.rs:2240-2243` valign=Bottom 공식
  `ref_y + ref_h - table_height - v_offset`. Page 앵커는 `(ref_y,ref_h)=(body_top,body_height)`.
- 실측 정합: body_top+body_height = 18.9+1084.7 = **1103.6px = 용지 하단 여백선(정확)**.
  pi=6 top = 1103.6 − 357.2(선언높이) − 0 = **746.4px** ← render tree bbox 와 일치.

## 원인 후보 (미확정)

1. **셀 내용 수직배치**: 박스 top(746.4)과 선언높이(357.2)는 한컴과 동일할 가능성이 높은데,
   박스 내부 헤딩(현장대응단장)이 rhwp 783(=top+37) vs 오라클 806(=top+60) — 셀 안에서
   내용이 ~23px 위. 즉 박스 위치가 아니라 **셀 내용의 상단 여백/수직정렬** 차이일 수 있다.
2. **높이 과대측정**: pi=5(247.3)·pi=6(357.2)가 각각 ~30px 과대라면 bottom-align 이 상단으로
   밀어올린다. 단 선언높이 기반이라 과대측정 여지는 제한적.
3. **bottom 기준**: 한컴이 용지-가장자리(1122.5) 또는 stored vpos(pi6=795.9≈오라클 헤딩 806)
   기준일 가능성. stored vpos 근접이 단서이나 valign=Bottom 의미와 상충.

세 후보 모두 `table_layout.rs:2240`(모든 하단앵커 개체 공용) 또는 셀 내용배치(광범위) 를
건드려야 하며, 단일 문서로 튜닝하면 #1611/#1658/#321 회귀 위험.

## 권고

- 이 발현은 **전용 타스크**로 분리해, 하단앵커 문서 회귀코퍼스(발신명의/footer/결재틀 다수)를
  먼저 구성한 뒤 원인을 pin down 하고 회귀 게이트와 함께 수정한다.
- 그 전까지 **known-residual** 로 유지(페이지 수는 정합, 페이지 내 ~30px 위치 오차).
