# 렌더링 보정 하드코딩 금지

렌더링, 파서, 레이아웃, pagination 보정은 특정 샘플명, 파일명, 페이지 번호, PR/issue 번호, 임의 계수로 맞추지 않는다.

허용되는 보정 근거는 문서가 실제로 가진 속성과 공개 스펙 필드다. 예를 들면 `LineSeg`, `ParaShape`, `CharShape`, 표/셀 크기와 padding, control 속성, page/section 속성처럼 입력 문서에서 읽을 수 있는 값이어야 한다.

시각 차이를 줄이기 위해 계수나 예외 분기가 필요해 보이면 먼저 그 값이 문서 속성에서 유도되는지 확인한다. 근거 속성이 없으면 구현하지 말고, 기준 PDF/샘플 추가 요청 또는 별도 분석 이슈로 분리한다.

PR 리뷰와 self review 에서는 다음을 반드시 확인한다.

- 샘플 전용 파일명/페이지 번호 분기가 없는가
- 특정 issue/PR 번호만을 위한 코드 경로가 없는가
- `0.88`, `1.12` 같은 임의 계수가 문서 속성 설명 없이 들어가지 않았는가
- 보정 근거가 review 문서에 문서 속성 또는 스펙 필드 이름으로 설명되어 있는가
