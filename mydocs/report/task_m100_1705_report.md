# 최종 결과보고서 — Task #1705: 어울림 표 트레일링 빈 문단 앵커-페이지 귀속

- 마일스톤: M100 (v1.0.0) / 이슈: edwardkim/rhwp#1705
- 작성일: 2026-07-01

## 1. 문제 (재정의)

당초 "작은 표를 rhwp가 2쪽으로 과대분할"로 보고됐으나, #1700+#1706 적용 base 에서 한글
per-문단 페이지를 측정한 결과 **표는 양쪽 같은 페이지수**였고, 실제 불일치는 **어울림
(floating) 표 직후 빈 문단의 페이지**였다.

| 문서 | 어울림 표 | 빈 문단 한글 | rhwp(수정 전, #1700 last-page) |
|------|----------|-------------|------------|
| 14504219 | 9행, 앵커 p1 | **p1 (앵커)** | p2 |
| 18092907 | 59행(8쪽) | **p1 (앵커)** | p8 |
| 2067603 | 258행(10쪽) | **p10 (표 끝)** | p10 ✓ |

## 2. 원인

Task #1700 은 wrap_around(어울림) 빈 문단을 **앵커 표의 마지막 페이지**에 귀속했다. 이는
표 아래로 흐르는 전체폭 빈 문단(2067603)엔 맞지만, **표 옆 wrap zone 에 있는 좁은 폭 빈
문단**(14504219 등)은 한글이 **앵커(첫) 페이지**에 두므로 어긋났다.

## 3. 판별자 규명

빈 문단 LINE_SEG 의 **segment_width(sw)** 가 결정한다:
| 문서 | 빈문단 sw | 한글 페이지 |
|------|----------|-----------|
| 14504219 | 2889 (좁음, wrap zone) | p1 (앵커) |
| 18092907 | 2964 (좁음) | p1 |
| 18067775 | 1885 (좁음) | p1 |
| 18125079 | 2681 (좁음) | p1 |
| 2067603 | 48188 (전체폭) | p10 (표 끝) |

→ **좁은 폭(표 옆) = 앵커(첫) 페이지, 전체 폭(표 아래) = 표 끝 페이지.**

## 4. 해결

`src/document_core/queries/rendering.rs` `dump_page_items()` 의 wrap_around 귀속:
- 표의 **첫(앵커) 페이지** `item_first_page` 를 추가 매핑.
- 각 wrap 문단의 `line_seg.segment_width` 를 px 변환해 `body_area.width * 0.9` 미만이면
  wrap zone(좁음) → **첫 페이지**, 아니면 → **마지막 페이지**.

순수 쿼리 표면화 정정(레이아웃/페이지 수/시각 불변). #1700 의 last-page 고정을 sw 기반으로 교체.

first==last 로 불변. 전체폭 문단은 last-page 유지(2067603 등 불변).
