## 결론

원래 전제(비-tac 상위 float/분할 표의 pad 팽창)는 1단계 실측에서 **정답지 대조
가능 사례 미확보**로 기각(A/B/C 검토 후 작업지시자 B안 결정). 재정의된 범위 —
**측정(HeightMeasurer)·렌더(resolve_row_heights)의 pad 회계 비대칭** — 를
정정해 하단앵커 결재란 표의 3.73px 들뜸을 해소(body 하단 밀착 Δ0.00).

## 1단계 (원 전제 기각 근거)

정답지 보유 30종 스캔: aift 최대 후보 표 렌더 총높이 한컴과 1px 이내 정합
(셀 근사 후보가 행 솔버·clamp에 흡수), exam_kor +0.2~1.3px(측정 한계 이하),
hwpx_sample2 폰트 잡음 매몰. 실발현 사례는 전부 tac/중첩(#2211 정정 완료).

## 2단계 (드리프트 귀속)

pi=6 하단앵커 표(1×1 자리차지 + 중첩 12×41 결재란): 측정 320.42 vs 렌더
316.70 = 3.73px(=pad 상하합 280HU). 원인 = #2211이 layout에만 relaxed_pad를
적용하고 측정은 (#1748 충돌로) 되돌린 비대칭 — 중첩 표는 measured_table 없이
layout이 직접 계산하므로 비대칭이 실드리프트로 발현.

## 3단계 (정정, `93219a6b`)

[height_measurer.rs](../../src/renderer/height_measurer.rs) 단일행·병합행 성장
판정에 `relaxed_pad_mirror = depth > 0 && 저장 LINE_SEG 텍스트 셀` 미러
(줄 흐름 pad 미가산 / 개체 pad 유지). 가드 2회 수렴 실측: `||tac` 포함 시
상위 TAC 표(mt 우선이라 이미 일관)의 실지오메트리 이동으로 6건 실패 →
`depth>0` 단독 확정. `issue_1858` 허용오차 6px→**2px 강화**.
