## 4. 같은 페이지 내 위치 정합 (MATCH 표본 20건, 54페이지)

페이지·PI 가 MATCH 여도 위치까지 같은 것은 아니다:

| 유형 | 규모 | 예 |
|------|------|----|
| 정합 (|Δbaseline| ≤ 1pt) | 표본의 약 절반 | 18145031(0.12pt), 3194769(0.96pt) |
| 표 셀 국소 배치 차 (수~십수 pt 스텝) | 다수 | 3114781 p1 (셀 제목줄 −10~−11pt 스텝) |
| **다쪽 표 continuation 페이지 전체 상수 오프셋** | 지배 패턴 | 2448877 p2 **전체 −54.3pt**, 3114781 p2 **전체 −33.9pt** (p1 은 −1pt 정합) |
| 줄바꿈 상이로 페어링 불가(측정 한계) | 소수 outlier | 156757711 (663pt 는 페어링 아티팩트) |

- **신규 지배 패턴**: 다쪽 표/서식의 continuation 페이지에서 rhwp 콘텐츠 시작 y 가
  통째로 30~54pt 위 — 페이지·PI 매칭으로는 잡히지 않는(같은 페이지) 위치 결함.
  continuation 상단(반복 제목행/이월 잔여 높이) 회계 차이로 추정 → 신규 이슈 등록.
- 측정 방법론: bbox top 이 아닌 **baseline** 대조 필수(#1827 판별 — 폰트 ascender
  차이로 bbox 는 +2~3pt 오탐).

### 정정 (2026-07-03, #1831 처리 결과)

위 "continuation 상단(반복 제목행) 회계 차이" 추정은 **오진**으로 확인되어 정정한다
(인과 실험 재현물 `output/poc/task1831/`, 상세 `mydocs/plans/task_m100_1831.md`):

- **2448877 p2 −54.3pt 의 실원인**: 한글은 p2 에 반복 제목행을 그리지 않는다. 같은
  문단에 앵커된 선행 표(표1) 아래로 밀린 표2가 단 잔여에 통째로 안 들어가면 한글은
  **첫 조각을 만들지 않고 표 전체를 다음 쪽으로** 민다. rhwp 는 row0 한 줄을 p1
  하단에 조각으로 남겨 p2 전체가 위로 밀렸던 것. #1831 에서 규칙 구현으로 해소
  (Δbaseline p2 −54.3 → −2.27pt, oracle 452 무회귀).
- **3114781 p2 −33.9pt 는 continuation 오프셋 아님**: p2 는 별도의 전면 1×1 서식
  표이며, 오프셋은 셀 내부 문단의 **인라인 개체 라인높이 재계산 드리프트**
  (예: p[0] '작성요령' 라인 저장 lh=3401HU=12mm 를 rhwp 가 작게 재계산 → 후속
  블록 갭 압축 −33pt, p[6]/p[9] 내부표 인라인 라인 lh=10858/9095 에서 추가
  −11pt/+15pt 반전). 별개 결함군으로 후속 이슈 등록 대상.
- 오진 경위: 비교에 쓴 r2.pdf 가 한글 정답지가 아닌 rhwp 자체 출력(producer=rhwp)
  이었다. 시각 판정 전 PDF metadata producer(=Hancom PDF) 확인을 절차화한다.
