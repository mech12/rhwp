## 원인 (2단계 — 계획서 a/b/c/d 축 전부 기각, 신규 축 확정)

`resolve_row_heights` 1-b/2-b의 `required = content + pad_top + pad_bottom`.
콘텐츠는 #2112 이후 저장 LINE_SEG lh 신뢰인데, 한컴 저장 h는 콘텐츠에 꽉 맞게
저장되므로(빈 셀 lh=h) 그런 행마다 +282HU 성장. 실증: 초과 행 전부 정확히
+3.76px, 여유 행은 0, 꽉 찬 행 ~8개 × 3.76 ≈ +30px = 관측치.

**성능 축 (승인 시 추가 지시)**: 병리적 루프 기각 — 호출 결정적(p1 렌더 rrh 7회/
cch 96회/17ms). 스튜디오 체감 느림 = 전체 렌더(네이티브 56ms, WASM ~110–170ms) ×
입력마다 전량 재계산(캐시 없음) × #2212 bbox 예외. 렌더 캐시는 후속 이슈 후보.
